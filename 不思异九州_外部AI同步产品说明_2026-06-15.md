## 0. 给外部 AI 的使用说明

如果你是被邀请协作的外部 AI，请先遵守以下规则：

1. 项目暂定名为《不思异：九州》。
2. 当前目标不是做宣传片、官网或完整商业版，而是把第一版 Web Demo 的核心机制跑顺。
3. 不要只给宏观想法；请尽量给可以落地到规则、数据、界面、素材、文案或测试清单的内容。
4. 不要复制或贴近参考游戏的具体 UI、视觉、剧情、文案、角色、图标、音频、商店表述。
5. 可以借鉴通用玩法语法：路线选择、资源压力、随机事件、失败重开、横向旅途表现。
6. 所有新增内容必须服务“山海经神秘九州远行”这个核心气质。
7. 当前第一版优先级是：可玩机制 > 路线选择 > 地点志 > 补给 > 半途路遇 > 小横屏体验 > 音频氛围。

## 1. 产品一句话

《不思异：九州》是一款以《山海经》为灵感的横屏远行生存叙事游戏。玩家率领古代车队从中原出发，进入九州、大荒、雷泽、赤水、归墟等神秘地理区域，在“车轴、粮草、神志”三项资源压力下选择路线、停靠地点、搜集补给、处理半途路遇，并逐渐发现地图、文字和世界本身都不再可信。

核心体验：

```text
一个凡人车队，用有限的器物、粮草和心智，强行穿过一个越来越不像人间的世界。
```

## 2. 类型与参考边界

### 2.1 类型定位

- 山海经题材
- 远行生存
- 叙事 Roguelike
- 节点地图探索
- 资源管理
- 文字选择事件
- 横屏手机游戏界面
- 假动态旅途舞台

### 2.2 可借鉴对象

本项目早期讨论过三个参考方向：

1. `Out There`
   - 可借鉴：路线选择、未知地图、资源压力、失败重开、探索推进。
   - 禁止复制：太空、星图、星球节点、飞船、氧气/燃料/船体对应、科幻 HUD、外星语言系统具体表达。

2. 横向西部远行游戏
   - 可借鉴：横屏旅途、队伍在路上、背景滚动、日夜变化、云和风景的“在路上”感觉。
   - 禁止复制：牛仔、枪、酒馆、口琴、西部马车、荒漠小镇等具体西部符号。

3. `Darkest Dungeon`
   - 可借鉴：队伍承压、精神污染、横向推进的戏剧性。
   - 禁止复制：英雄队列、火把压力条、哥特旁白、回合战斗、职业站位、病症怪癖系统。

### 2.3 商业风险基本判断

玩法机制的抽象组合通常不属于某个游戏独占版权，但具体表达、UI、文本、美术、音频、关卡、角色、商标和商店宣传语都可能有风险。

安全做法：

- 借机制，不借截图级表达。
- 借问题结构，不借答案形状。
- 借“玩家感受”，不借“视觉符号”。
- 对外表达时不要说“中国版 Out There”或“山海经版某某游戏”。

## 3. 当前核心资源

正式资源名：

```text
车轴、粮草、神志
```

UI 短名：

```text
车、粮、神
```

### 3.1 车轴

对应“还能不能继续远行”。

包含车轮、车轴、辇架、绳索、牲口、舟板等远行工具状态。

设计作用：

- 移动消耗车轴。
- 山地、泽地、强渡、荒路会增加消耗。
- 车轴过低时，玩家怕的是走不出去。
- 归零进入断轴危机。

### 3.2 粮草

对应“队伍还能不能撑下去”。

包含粮食、盐、草料、药、薪柴等综合补给。

设计作用：

- 移动、等待、绕路、祭祀、交易都会消耗粮草。
- 部分地点可以补粮。
- 粮草过低时，玩家怕的是撑不过今天。
- 归零进入饥荒危机。

### 3.3 神志

对应“所见世界还可不可信”。

包含队伍认知、精神稳定、对地图和文字的信任度。

设计作用：

- 见异象、读巫文、夜行、听神谕、接近归墟会损耗神志。
- 神志越低，文字越恐怖，地图越不可靠。
- 低神志可能看到假路，也可能看到隐藏真相。
- 归零不是普通死亡，而是认知崩坏或特殊结局。

## 4. 核心玩法循环

基础循环：

```text
查看当前位置
-> 阅读地点志
-> 搜集一次补给
-> 打开九州图
-> 观察可见路线、资源消耗、目的地补给倾向
-> 选择下一站
-> 消耗车轴 / 粮草 / 神志
-> 途中可能半途停下处理路遇
-> 抵达新地点
-> 迷雾散开，显影后续路线
-> 继续前进或陷入危机
```

第一版的核心不是战斗，而是：

- 路线选择
- 资源压力
- 地点志阅读
- 补给差异
- 半途路遇
- 神志污染文本
- 失败或抵达终点

## 5. 地图设计

### 5.1 地图结构

底层是节点网络，不是开放世界。

每个地点是一个 `location`，每条路线是一个 `route`。

路线包含：

- 起点
- 终点
- 消耗
- 风险
- 地貌
- 锁定条件
- 半途事件
- 提示文案

### 5.2 战略地图表现

地图页参考 Out There 的“大地图路线决策”，但视觉必须是：

```text
羊皮纸山海舆图 + 墨线 + 朱砂路线 + 迷雾显影 + 地名签
```

不是：

```text
星图 / 星球 / 发光航线 / 科幻扫描 / 宇宙背景
```

### 5.3 迷雾与显影

当前规则：

- 初始只显露当前地点和少量可走路线。
- 抵达新地点后，后续路线和雾中远影才显现。
- 雾中远影只是提示，不可点击。
- 必经路线不能被随机隐藏。
- 同一局同一地点的半随机显影结果稳定，刷新后不变化。
- 新开局可出现不同支线或雾中轮廓。

### 5.4 半随机稳定显影

当前原型使用 `routePools`：

- `requiredRoutes`：必定出现的保底路线。
- `optionalRoutes`：按本局种子抽取的可选路线。
- `fogPool`：雾中远影，只提示，不可点击。
- `rescueTags`：资源危急时用于保底提示。

当前已验证：

- 14 个地点
- 33 条路线
- 14 个路线池
- 从起点到终点有必经保底路径

保底路径：

```text
central_post
-> old_king_road
-> thunder_marsh
-> red_marsh
-> broken_stele
-> kyushu_rift
```

## 7. 地点与补给

每个地点应有：

- 地点名
- 区域
- 地貌
- 地图坐标
- 地点志
- 可用补给
- 当前地点事件
- 后续显影路线

地点志非常重要，因为这是“山海经亮点”的主要承载。

玩家抵达地点时，应先知道：

```text
这里是什么地方
为什么危险
有什么风俗 / 异象 / 地理异常
能补什么
去哪里更冒险
```

### 7.1 补给规则

每个地点资源不同，不是每个城镇都能全补。

例如：

- 有的偏补车轴。
- 有的偏补粮草。
- 有的偏补神志。
- 有的补一种资源但牺牲另一种。
- 有的本次抵达只能补一次。

当前原型已实现：

- 26 个补给项。
- 每次抵达最多使用一次补给。
- 半途路遇状态不能搜集城镇补给。
- 路线卡会预览目的地补给倾向。

## 8. 半途路遇

### 8.1 设计目的

只在城镇触发事件会让旅程像点菜单。半途路遇让玩家感觉车队真的在走。

当前目标：

```text
行走地图途中可能走到一半停下来，进入具体事件选择。
```

### 8.2 当前实现状态

当前 Web Demo 已实现：

- 33 条路线中 29 条有 `midEvent`。
- 路遇事件总数 13 个。
- 主线前半段有稳定半途事件。
- 处理路遇后继续抵达目的地。
- 同一路线已处理路遇不会重复卡住。
- 半途状态下禁用城镇补给。

### 8.3 当前路遇事件

保留原有：

- 轮声入骨
- 路旁无名祠
- 半途黑云

新增山海异象：

- 文鳐雨
- 当康啼垄
- 旋龟浮甲
- 鴸鸟呼名
- 九尾灯影
- 黑齿鳞价
- 鸣蛇横风
- 骨牛空辙
- 倒芦照人
- 梦蝉蜕壳

### 8.4 路遇强度原则

三类路遇：

1. 低风险提醒
   - 教学和氛围。
   - 不制造死局。
   - 单项损耗较低。

2. 中风险抉择
   - 用车轴、粮草、神志做真实交换。
   - 适合第一章主密度。

3. 高风险污染
   - 赤水、梦图、裂隙前后使用。
   - 压神志和车轴，但必须给可承受选择。

后续建议补：

- 路遇冷却。
- 资源安全地板。
- 连续高污染限制。
- 危机后 2 段保护。

## 9. 神志文本系统

神志不是普通血条。它改变玩家看到的文字、地图和选择。

同一事件应有三档文本：

| 神志状态 | 文本气质 | 说明 |
|---|---|---|
| 高神志 | 地理志 | 冷静、古雅、可判断 |
| 中神志 | 异闻录 | 征兆增多、主观感增强 |
| 低神志 | 疯人书 | 语义断裂、世界开始说话 |

示例方向：

高神志：

```text
前方有泽，水色赤而无波。泽畔多白骨，骨上生细草。
```

中神志：

```text
赤泽无风，却有水声从车底传来。白骨上的草向你低伏。
```

低神志：

```text
水不是水。骨不是骨。车轴转了一圈，又转回昨日。
```

重要原则：

- 低神志不只是惩罚，也可以解锁隐藏路线、异文、真名和特殊结局。
- 文本可以恐怖、断裂、克苏鲁式，但不能脱离山海经和古道远行。

## 12. 素材规范

### 12.1 图片

背景图：

- 不要带车队。
- 不要带现代建筑。
- 不要像竞品截图。
- 横向宽图优先。

车队图：

- 必须单独一层。
- 透明 PNG 最好。
- 如果生成引擎不能真透明，应使用纯绿幕或高对比实色背景。
- 避免黑白格假透明。
- 避免光影边缘虚化、半透明辉光、投影过重、运动模糊。

污染层：

- 黑底更适合 `screen/lighten` 混合。
- 白底污染层暂不适合直接接入。
- 星象类污染只能用于低神志和异常状态，不能作为常态地图视觉。

## 16. 多线程协作结构

当前项目有 6 个长期子线程角色：

| 线程 | 职责 |
|---|---|
| A | 核心规则与数值平衡 |
| B | 山海经世界观与叙事 |
| C | Web Demo 技术原型与工具链 |
| D | 视觉体验与交互氛围 |
| E | 商业合规与竞品风险 |
| F | 音乐音效与音频体验 |

工作制度：

- 所有子线程成果必须提交给主线程。
- 主线程负责统一验收、合并、向用户汇报。
- 子线程不得自行改变总方向。
- 任何素材入库、代码执行、授权判断、范围变化，都必须回到主线程。

## 19. 一句话总结给新 AI

请把《不思异：九州》理解为：

```text
一个横屏手机向的山海经远行生存叙事游戏。
底层是 Out There 式路线资源决策，但外观是羊皮纸九州舆图。
主画面是车队沿横向山海长图行进的假动态舞台。
玩家管理车轴、粮草、神志，在地点停靠、搜集补给、选择路线、处理半途路遇。
神志越低，文字、地图和世界越不可信。
当前第一版目标是让 Web Demo 的机制跑顺，而不是做宣传或完整商业版。
```
