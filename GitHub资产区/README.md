# 《不思异：九州》GitHub 资产区

这个目录用于集中管理需要进入 GitHub 仓库、需要同步、或未来可能发布的项目资产。

## 目录结构

| 目录 | 用途 |
|---|---|
| `00_同步台账` | 仓库绑定、同步记录、提交计划 |
| `01_产品文档` | 产品定义、玩法架构、决策记录的整理版本 |
| `02_设计资产` | 视觉方向、UI 草案、风格参考、资产清单 |
| `03_WebDemo` | Web Demo 相关说明、数据结构、原型资料 |
| `04_竞品研究` | 同类产品研究、可借鉴点、风险边界 |
| `05_发布物料` | 未来 README、商店页、宣传说明、截图说明 |
| `06_音乐音效` | 音乐、音效、环境声、音频生成提示语、授权台账 |
| `07_视频动态` | 短循环视频、转场镜头、神志异象、视频提示语、授权台账 |
| `99_归档` | 过期方案、历史版本、废弃材料 |

## 使用原则

1. 先本地整理，再同步到 GitHub。
2. 原始讨论材料可以保留，但进入发布区前必须经过主线程验收。
3. 不上传密钥、账号、未授权素材、未确认来源的图片或竞品文本。
4. 大文件和生成素材先登记，不直接塞进仓库。
5. 每次同步前检查 `git status`，确认没有无关文件混入。
