## 整体判断

这批图可以先用于 Web Demo 的假动态横向舞台验证。

优势：

- 大多数场景图是 `2508 x 627`，接近标准 `4:1`，适合横向滚动。
- 背景、前景、低神志污染、车队层已经具备基本分层条件。
- 画风接近古籍山海图与矿物色方向，没有明显现代建筑或科幻感。
- 车队层是白底剪影，后续可抠图或用混合模式临时叠加。

主要问题：

- `REVIEW-001` 不是游戏背景，而是素材拼贴/预览图，不能直接进 Demo 场景。
- 车队层目前不是绿幕或纯色抠图底，正式使用前需要抠图；后续不要要求网页生图直接生成透明背景。
- 部分背景偏“完整概念图”，可拆层性还不够强，后续最好继续生成更干净的天空层、前景层和地点剪影。
- 低神志污染层是白底水墨图，适合先用 `multiply` 或透明度叠加验证；后续可生成纯色底版本，不要求透明 PNG。

## 下一轮生图改进点

1. 背景继续强调“无车队、无人物主体”，并留出车队行走空间。
2. 车队层不要要求透明背景，优先要求纯绿色绿幕或单一高对比纯色背景，方便抠图。
3. 需要补三类更干净素材：纯天空层、纯地点剪影层、绿幕/纯色底云雾风沙层。
4. 背景不要生成拼贴图、展示板、清单图、四宫格或多图合集。
5. 低神志层应更像可叠加素材，不要像独立场景插画；避免光晕、柔焦、羽化和复杂半透明边缘。
