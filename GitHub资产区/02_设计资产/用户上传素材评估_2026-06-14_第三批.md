## 整体判断

第三批素材非常贴合“假动态横屏长图 / 横向山海旅途舞台”的方向。

优势：

- 所有图片均为 `2508 x 627`，比例统一，能直接进入 Web Demo 第一屏舞台测试。
- `BG-008` 已经具备从左到右展开的路程感，是当前最适合做主推横向旅途舞台的整合背景。
- 天空层、前景视差层、昆仑远景层可以与既有背景拆组，低成本验证远近层移动。
- 三张绿幕车队比前两批更利于后续抠图，适合作为主行动层候选源图。
- 地点节点素材图能补足“路程中出现多个地点信号”的表达。

主要问题：

- 三张车队仍是绿幕源图，不是透明 PNG，进入 Demo 前需要抠绿或前端 chroma key 测试。
- 低神志污染图是白底，不适合直接以普通透明叠层覆盖；更适合待复核后用正片叠底、柔光或重新生成透明/黑底版本。
- `BG-008` 信息量较高，叠 UI 时要控制事件框、资源条和按钮的底色遮罩，避免遮住画面焦点。
- 地点节点图更像概念组合图，作为节点素材需再裁切或分拆，不宜整张直接当地图。

## 对第四批生图的反向建议

1. 继续保持 `4:1` 横向比例和无车队背景。
2. 绿幕车队方向正确，但要要求纯绿色背景、边缘硬、无绿色反光、无烟雾遮挡。
3. 低神志叠层不要白底，优先生成黑底/透明感/单色高对比叠层，便于前端混合。
4. 地点节点建议单地点一张，或明确左中右三个节点，避免裁切后主体过小。
5. 背景信息量可以分两类：一类像 `BG-008` 做整合主背景，一类像 `BG-010` 留更多 UI 空白。
