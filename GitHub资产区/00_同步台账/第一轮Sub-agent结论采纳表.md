# 第一轮 Sub-agent 结论采纳表

> 本表用于主线程初步筛选 sub-agent 结论。这里不是最终产品定稿，只有“采纳”或“改写后采纳”的内容才会进入主产品文档。

## 1. 可直接采纳

| 来源 | 结论 | 处理 |
|---|---|---|
| G | 核心循环是用车、粮、神在路线与事件之间反复做代价选择。 | 进入核心循环表述 |
| G | 三资源必须有行为差异，不只是三个血条。 | 进入规则验收标准 |
| G | 假动态横屏长图应是可读的风险仪表盘，不是装饰背景。 | 进入视觉与玩法原则 |
| T | 第一版推荐 `React + SVG + CSS`，Canvas 只作可选氛围层。 | 进入技术路线候选 |
| T | 不建议第一版使用复杂游戏引擎。 | 进入技术边界 |
| R | 主线程应按主题合并结论，不直接拼贴子线程报告。 | 进入协作流程 |
| Cmp | Web Demo 保持节点路线、三资源、短事件、结果日志、轻量横向旅途舞台。 | 进入 Web Demo 范围 |
| Cmp | 神志可以污染文本和视觉，但资源数字、按钮位置和日志必须稳定。 | 进入 UI 安全边界 |

## 2. 改写后采纳

| 来源 | 原结论 | 改写方向 |
|---|---|---|
| G | 10 分钟 Demo 建议 6-8 个地点、12-18 条路线、15-25 个事件。 | 作为“第一可玩切片”规模，不替代完整 MVP 规模 |
| T | 1 天/3 天/1 周工作量评估。 | 作为内部技术预估，不向外承诺日期 |
| Cmp | 借鉴组合包括多个竞品。 | 改写为“结构参考”，不在宣传中直接绑定竞品 |
| R | 首轮规模需要用户决策。 | 与其他决策项合并，避免重复提问 |

## 3. 暂存为候选

| 来源 | 候选内容 | 暂存原因 |
|---|---|---|
| G | 第一版结局目标建议“带回记录后归返”。 | 很适合，但需与 B 叙事线程地图设计合并判断 |
| Cmp | 失败游记/复盘日志可作为商业卖点。 | 值得做，但需先确认 Demo 规模 |
| Cmp | 是否加入随从/巫祝建议。 | 可能增加复杂度，第一版暂不默认 |
| R | 低神志体验强度上限。 | 需要用户审美决策 |

## 4. 暂不采纳

| 来源 | 内容 | 原因 |
|---|---|---|
| 无 | 直接进入完整战斗、队伍养成、开放世界、复杂故事库存。 | 与当前 Web Demo 目标冲突 |
| 无 | 直接复制竞品 UI、文案、流程、素材、音效或具体表达。 | 商业和版权风险高 |

## 5. 汇总后的待决策问题

| 决策 ID | 问题 | 建议默认 |
|---|---|---|
| D1 | Web Demo 第一版是否正式采用“节点逻辑 + 假动态横向山海旅途舞台”？ | 是 |
| D2 | 第一可玩切片先做 6-8 个地点，还是直接做 20 个地点？ | 先做 6-8 个地点 |
| D3 | 神志低时是否允许出现误导性路线预览？ | 允许轻度误导，但资源数字和日志必须可信 |
| D4 | 是否把“失败游记/复盘日志”作为 Demo 核心展示点？ | 是 |
| D5 | 第一版是否加入随从/巫祝建议？ | 暂不加入，只做事件文本中的提示 |
