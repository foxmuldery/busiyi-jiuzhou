# 《山野逸事》第三部志怪内容协作者 AI 问答工作台

日期：2026-06-24  
接入项目：《不思异：九州》  
适用对象：《山野逸事》第三部部分内容策划 / 志怪内容协作者、AI 资料采编、主线程协调

## 1. 工作台定位

这份工作台不是让《山野逸事》内容协作者拆素材卡，也不是让他学习游戏策划格式。

如果内容协作者还不熟悉《不思异：九州》，先给他这一份更短的单文档说明：

- `协作/山野逸事内容策划项目交接说明_给不熟悉不思异九州的协作者.md`

内容协作者最有价值的工作是：

- 判断哪些旧设定有灵魂，值得进入《不思异：九州》。
- 解释旧设定背后的典故、人物关系和情绪功能。
- 判断 AI 有没有误读原意。
- 回答会影响游戏设计的关键分歧。
- 指出哪些意象、台词、风俗、禁忌不能丢。
- 补充旧项目里没有写进文档的情节来龙去脉。

AI 最有价值的工作是：

- 阅读旧资料。
- 归纳素材候选。
- 找出不确定点。
- 向志怪内容协作者提出少量高质量追问。
- 把内容协作者回答整理成可游戏化素材包。

主线程负责：

- 判断哪些素材能进入当前 Web Demo。
- 控制《不思异：九州》的玩法、调性和风险边界。
- 把确认素材转给 B/C/D/F 线程继续做事件、图像、音频和接入。

## 2. 一次交接的最小闭环

```text
内容协作者提交原始资料
→ AI 初读，生成摘要与追问
→ 内容协作者短答和纠偏
→ AI 生成内容策划确认版素材包
→ 主线程挑选可转译素材
→ 游戏侧转成地点 / 路遇 / 补给 / 危机 / 行囊物 / 结局伏笔
```

第一轮不要追求全量迁移。建议只提交：

- 3 到 5 个旧设定；或
- 5 到 10 页旧资料；或
- 1 个地点 + 2 个怪事 + 1 个人物关系；或
- 1 集未拍成故事的完整设定。

## 3. 内容协作者提交时只需要写这 8 行

内容协作者不需要填游戏表格，只需要在资料前面加这段。

```markdown
## 本批资料说明

资料主题：
原本在《山野逸事》第三部中的用途：
我最想保留的气质：
绝对不能误读或删掉的东西：
可以大胆改编的部分：
我希望 AI 帮我重点判断的问题：
我只确定这些内容：
我不确定、可以让 AI 先猜再问我的内容：
```

如果时间很紧，可以只回答下面 5 句：

```markdown
1. 这批东西里，哪个情节、故事或典故最值得保留？
2. 它原来想表达什么情绪或主题？
3. 哪些人物关系、因果或禁忌不能改错？
4. 哪个意象、动作、关系或句子绝不能删？
5. 低神志时它应该怎么变坏？
```

## 4. AI 首轮阅读标准

AI 收到资料后，先不要写游戏事件。

首轮只输出：

1. 本批资料一句话价值。
2. 资料摘要。
3. 可游戏化候选。
4. AI 初步判断。
5. 不确定点。
6. 给内容协作者的追问，最多 8 个。
7. 暂不建议进入游戏的内容。

每个追问都必须满足：

- 指向具体素材。
- 说明为什么要问。
- 说明内容协作者回答会影响哪个游戏设计判断。
- 不要求内容协作者填写数值、概率、代码字段或 UI 文案。

## 5. 好的追问长什么样

不好的追问：

```text
请完善无灯墟市的世界观设定。
```

好的追问：

```text
我读到“无灯墟市”这一段，目前判断它适合做成路遇或交易点。

这里有一个关键分歧：
它到底是真实存在的鬼市，还是当地人用来掩护人口交易的谣言？

这会影响游戏设计：
- 如果是真鬼市，玩家可以用神志换粮草。
- 如果是人为骗局，玩家可以揭穿它，但可能损车轴、惹来追杀，或打开一条更危险的路线。

请你判断它在原设定里更接近哪一种？有没有第三种答案？
```

内容协作者可以短答：

```text
这是骗局，但骗局背后借用了真实禁忌。当地人不知道自己正在重复古国祭仪。
不可丢的是“灯不照脸，只照影子”。
玩家最好先以为自己进了鬼市，后来才发现活人比鬼更可怕。
```

## 6. 内容协作者回答后的 AI 输出

AI 收到回答后，生成《内容策划确认版素材包》。

每条素材必须明确：

- 原资料位置。
- AI 初步理解。
- 内容策划确认或修正。
- 世界观定性：真相 / 谣言 / 禁忌 / 骗局 / 风俗 / 古国遗痕 / 暂不确定。
- 情绪功能：敬畏 / 恐惧 / 怜悯 / 诱惑 / 荒诞 / 成就感。
- 玩家欲望。
- 玩家恐惧。
- 不可丢意象。
- 低神志扭曲。
- 推荐游戏用途。
- 资源倾向：车轴 / 粮草 / 神志 / 暂不确定。
- 是否适合当前 Web Demo。
- 是否需要二次追问。

## 7. 主线程验收门槛

一条素材进入游戏转译前，不需要完整剧情，但必须满足：

| 检查项 | 要求 |
| --- | --- |
| 内容策划态度 | 明确保留、暂缓或删除 |
| 世界观定性 | 至少知道是真相、谣言、禁忌、骗局、风俗或古国遗痕之一 |
| 玩家欲望 | 玩家知道为什么要冒险 |
| 玩家恐惧 | 玩家知道会失去什么 |
| 不可丢意象 | 至少 1 个，最好 2 到 3 个 |
| 低神志方向 | 有正常 / 低神志差异 |
| 游戏用途 | 能放进地点、城镇事件、半途路遇、补给、危机、行囊物或结局伏笔 |
| UI 可承载 | 可压成 40 到 70 字正文，选项不超过 4 个 |
| 资源关联 | 能关联车轴、粮草、神志之一 |
| 风险 | 无明显版权、现代表达或项目调性风险 |

## 8. 游戏化转译前禁止事项

- 不要求内容协作者写数值。
- 不要求内容协作者拆事件 ID。
- 不要求内容协作者写选项收益。
- 不把整集剧情硬塞成一个事件。
- 不把 AI 推测写成内容策划确认。
- 不把旧剧集素材直接标成可商用发布素材。
- 不在没有主线程验收前接入 Web Demo。

## 10. 第一轮试运行建议

第一轮建议只完成一个小闭环：

1. 内容协作者提交 3 到 5 个旧设定、故事桥段或典故来源。
2. AI 生成初读报告和最多 8 个追问。
3. 内容协作者短答。
4. AI 生成内容策划确认版素材包。
5. 主线程只挑 2 条转成游戏事件卡。
6. 最多只接 1 条进 Web Demo 测试。

如果内容协作者觉得这个流程省力，再扩大到每轮 20 到 30 条候选。
