## 0. 本轮口径

九州图不是现代地图，也不是把完整路线网提前摊开。它应该像一张羊皮纸残舆图：车队到哪里，墨才写到哪里；城市先在雾中露影，再露残名，最终被记入地点志；路线先像断墨或旧车辙，到达新城后才被朱砂描出。

核心文本节奏：

```text
未显名：只给方位、地貌、物象，不给真名。
半显名：露一两个残字，给风险和诱惑，不给完整确信。
显名：地名写定，地点志录入，可走路线刷新。
低神志：同一真实路线，显示更怪、更不可信的描述。
```

给 C 线程的最小落点：在 `locations` 和 `routes` 上补文本字段即可，不需要先重做世界生成器。

## 1. 迷雾中城市的三层命名 / 文本规则

### 1.1 三层定义

| 层级 | 字段建议 | 玩家看见什么 | 文本口径 |
|---|---|---|---|
| 未显名 | `fogName` / `fogText` | 雾中节点、方向影、地貌气味 | 不出现完整地名，只写“西北有墨痕”“泽上有白羽”“关影压纸”等 |
| 半显名 | `partialName` / `partialText` | 残缺地名、缺字、路引旁注 | 可以露一个核心字，如“某齿市”“赤水某滩”，但仍保留不确定 |
| 显名 | `name` / `revealText` | 正式地名、地点志、到达反馈 | 地名落定，说明它在世界中的用途、风险或资源倾向 |

三层文本不要写成“距离 3 格”“北方路线已解锁”。建议写成残卷、地志、舆图旁注：

```text
未显名：西北纸雾下，有一处车辙不肯干。
半显名：残卷露出“故……道”二字，墨痕向西。
显名：故王道已显。旧贡车的石基半埋在沙下，深辙仍把车队往西牵。
```

路线文本分两层：地图上只给 `routeHintText`，路线卡再给完整 `name / hint / risk / cost`。地图提示必须像舆图残注，不要像现代导航。

## 6. 低神志时的显影文本变体

低神志文本只改变玩家读到的描述，不改变真实路线、节点、消耗和锁定条件。C 线程可按阈值选择文案：

```text
sanity >= 60：使用正常 `revealText / routeHintText`
35 <= sanity < 60：使用 `uneasyRevealText / uneasyRouteHintText`
sanity < 35：使用 `madRevealText / madRouteHintText`
```

### 6.1 写法原则

1. 更怪：物象像活物，地名像在等人补全。
2. 更不可信：方向可以写得矛盾，但 UI 真实可走路线不变。
3. 不误导规则：不要把不可走路线写成可走，也不要把代价写反。
4. 不覆盖关键信息：锁定原因、风险等级、资源消耗仍按真实数据展示。

### 6.2 当前地点低神志样例

## 7. 给 C 线程可直接映射的数据建议

第一版建议只增文本字段，不改现有 `locations / routes` 主结构。若字段缺失，回退到现有 `name / detail / hint`。

### 7.1 `locations[id]` 字段建议

| 字段 | 类型 | 用途 |
|---|---|---|
| `fogName` | string | 未显名节点标题，如“泽上伏雷” |
| `fogText` | string | 未显名节点短说明，如“泽气压住纸角，白羽三枚浮在未写之水上。” |
| `partialName` | string | 半显名，如“雷泽……畔” |
| `partialText` | string | 半显名说明，给一点风险/资源倾向 |

| `fogClearText` | string | 雾散文本，可单独做反馈 |

| `uneasyRevealText` | string | 中低神志显影变体 |
| `madRevealText` | string | 低神志显影变体 |
| `poolTags` | string[] | 抽卡池标签，如 `["泽国", "水路", "粮草", "神志风险"]` |

示例：

```js
thunder_marsh: {
```

### 7.3 渲染优先级建议

```text
当前 / 已见：显示 name + revealText 或 detail
可达：显示 name；路线卡显示 route.name + route.hint
雾中：显示 fogName + fogText；路线显示 routeHintText
半显名：显示 partialName + partialText，适合一跳外或锁定目标
低神志：在不改变真实路线的前提下，替换为 uneasy / mad 文案
```

### 7.4 C 线程最小回退规则

1. 若没有 `fogName`，用 `terrain` 生成临时雾名：`road=旧道影`、`market=市影`、`water=泽影`、`rift=裂影`。
2. 若没有 `partialName`，用 `name` 的前两个字 + `……`，但主线地点建议手写。
3. 若没有 `routeHintText`，暂用现有 `route.hint`，但前端显示时避免加现代说明词。
4. 若没有低神志字段，继续使用现有 `events.texts.mad` 体系，不额外编造。
