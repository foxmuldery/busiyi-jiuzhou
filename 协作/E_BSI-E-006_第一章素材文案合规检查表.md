> 子线程：E 商业风险、素材授权与原创边界  
> 项目：《不思异：九州》Web Demo 第一章  
> 日期：2026-06-15  
> 任务：BSI-E-006  
> 范围：只写发布前检查表，不修改代码，不替代律师意见  
> 结论性质：产品侧风险判断；正式商业发布、平台上架、宣传投放前仍需按素材和文案实际版本复核

## 0. 发布前总原则

第一章可以借鉴“路线选择、资源压力、未知探索、横屏旅途、失败复盘”这些通用玩法语法，但正式进入游戏和宣传的表达必须回到本项目自己的组合：

```text
羊皮纸山海舆图
+ 古道车队
+ 车轴 / 粮草 / 神志
+ 雾散、城显、路显
+ 山海地名与自研地点志
+ 自研图标 / 自研音效 / 自研 UI
```

合并前先问三件事：

1. 玩家截图里是否一眼是“九州荒外舆图 / 古道车队”，不是星图、太空、哥特远征或西部车队。
2. 文案是否都是自写地点志和事件文本，没有直接拿现代译文、百科、图鉴设定或竞品口吻。
3. 素材是否能回溯来源和状态；不能回溯的素材只能标 `Demo 临时` 或 `阻断禁用`，不能标正式可用。

## 1. 地点名、地点志、事件文案检查

### 1.1 允许方向

| 检查项 | 绿色做法 | 合并前看点 |
|---|---|---|
| 地点名 | 使用自研地名，或从古籍母题中抽象转译，如“废关”“雷泽浅畔”“白羽淖”“巫咸断碑” | 名称服务路线、资源或神志功能，不像现代百科词条 |
| 雾中名 | 用短、含混、有画面感的自研称呼，如“关影压纸”“泽上伏雷” | 不直接解释设定，不照搬译文句子 |
| 地点志 | 一句气氛 + 一句危险 / 风俗 / 资源理由 | 手机横屏三行内能读完 |
| 事件文案 | 车队、路引、残卷、祭所、墟市、异族交易等本项目语汇 | 选择结果清楚，不只堆神秘感 |
| 低神志文本 | 可不可靠、可怪，但规则信息不能反向误导 | 玩家仍知道会损什么、得什么、去哪 |
| 宣传摘要 | 说“山海经题材远行生存叙事游戏”“以《山海经》神秘地理为灵感” | 不绑定竞品，不声称还原某译本 / 图鉴 |

### 1.2 禁止方向

| 红线 | 为什么危险 | 替代做法 |
|---|---|---|
| 直接使用现代白话译文 | 现代翻译本身是独立表达 | 只理解母题，正式文本重新自写 |
| 直接使用百科、公众号、资料站段落 | 编排、解释和措辞都可能是现代作者表达 | 用“玩法功能转译”：它影响路线、资源、神志还是事件 |
| 继承现代图鉴、影视、漫画、游戏里的异兽 / 异族设定 | 容易继承他人二创设定 | 只保留公共母题，再自定义语言、交易、代价和资源作用 |
| 文案像竞品旁白或失败评语 | 会形成可识别表达相似 | 使用地点志、游记、路引、残卷口吻 |
| 宣传写“山海经版 Out There / 中国版 Darkest Dungeon / 西部远行换皮” | 竞品依附和商业宣传风险高 | 写玩法事实：选路、消耗、路遇、神志变文本 |

### 1.3 第一章地点逐项检查

| 地点 | 重点风险 | 通过标准 |
|---|---|---|
| 中原驿 | “旧王朝 / 驿站”容易写成通用古风设定 | 必须服务教学和“人间秩序将失效”的原创开局 |
| 青丘外邑 | 青丘、狐灯容易套现代狐妖设定 | 只写边邑交易、假名、路引，不继承现代狐族世界观 |
| 故王道 | 王道、旧都容易像百科解释 | 以车辙、迷路、路线选择表达玩法功能 |
| 废关 | 关塞文案容易变成历史注释 | 保持“不得西行”的自研旧令和补车代价 |
| 鸟鼠夹道 | 古名可用，但不能搬现代释义 | 用兽穴、窄路、车轴 / 粮草 / 神志取舍重写 |
| 雷泽浅畔 | 雷泽母题常见，不能抄译注 | 用地底雷、泽地补粮和损神志形成原创表达 |
| 白羽淖 / 羽民渡 | 羽民设定容易继承现代图鉴 | 只写轻重、渡口、名字代价，不照搬图鉴形象 |
| 黑齿市 / 昼骨墟 | 异族市场容易落入现代设定书 | 交易规则、货币、代价全部自研 |
| 无名祠 / 巫咸断碑 | 巫咸、祭所容易照搬注释 | 用碑文、正名、解锁裂隙等玩法功能转译 |
| 赤水外滩 | 赤水母题常见，图像风险高 | 文案自写，视觉不用现代山海插画构图 |
| 归不得林 | 幻路林是通用母题 | 与车队回撤、绳结、神志压力绑定 |
| 白民盐界 | 白民母题容易被现代图鉴固化 | 只写盐界、影子、保粮和路引规则 |
| 梦图驿 / 九州裂隙 | 容易写成抽象玄学 | 必须清楚承担裂隙前哨和章节收束功能 |

## 2. 地图 UI 避免 Out There 星图影子

### 2.1 地图绿色标准

| 组件 | 安全表达 | 合并前看点 |
|---|---|---|
| 地图底图 | 羊皮纸、旧纸、山海舆图、墨线、水渍、朱印 | 不出现深空背景、星云、星点散布 |
| 节点 | 地名签、城印、祠印、关隘符号、泽地墨痕 | 不像星球 / 行星 / 发光空间站 |
| 路线 | 古道墨线、车辙线、断续路引、纸纹显影 | 不像星际航线、jump line、发光网格 |
| 显影 | 雾散、墨晕、纸背渗出、朱印落下 | 不用扫描圈、雷达、科幻 HUD |
| 路线卡 | 路引签 / 驿牒 / 残卷批注 | 不做太空面板、星球资源卡 |
| 资源图标 | 车轴裂纹、粮袋、神志残纹 | 不对应燃料 / 氧气 / 船体三件套 |

### 2.2 地图红线

| 红线 | 处理 |
|---|---|
| 星空、星球节点、星系航线、太空蓝紫发光 | 阻断，替换为舆图纸面和地名签 |
| `jump / galaxy / planet / scan / radar` 等公开 UI 命名 | 改为 `雾中 / 已显 / 可达 / 锁定 / 已行` 等中性或九州语汇 |
| 全图一次性摊开，像星图路线网 | 只显示当前、已见、可达、锁定、雾中；裂隙前不泄露全貌 |
| 路线卡排版接近竞品商店图或截图 | 重排为路引签、消耗、补给、锁定原因 |
| 宣传截图以“星图探索”作为主视觉 | 改为“羊皮纸山海舆图上路线显影” |

## 3. 横屏行走避开西部车队、FAR、Darkest Dungeon

### 3.1 通用旅途语法可用

| 可用语法 | 九州转译 |
|---|---|
| 横向旅途舞台 | 山海古道从左到右推进 |
| 轻微视差和假动态 | 云气、泽雾、纸雾、车辙尘 |
| 日夜与天气变化 | 天色、风沙、赤水、低神志异象 |
| 队伍承压 | 车轴、粮草、神志共同下降 |
| 半途事件打断 | 路遇、祭所、墟市、异象停路 |
| 失败复盘 | 荒外游记、残卷、前队遗物 |

### 3.2 横屏行走红线

| 对象 | 禁止具体表达 | 安全替代 |
|---|---|---|
| 西部车队 | 西部马车、牛仔、枪、酒馆木牌、口琴主题、西部配色 | 古代辇车 / 车队剪影、荒泽、废关、赤水、古驿 |
| FAR 类载具 | 大型独特载具剖面、锅炉、引擎内部、拉杆、实时驾驶、碰撞解谜 | 车队只是旅途前景和状态载体，不是操作机器 |
| Darkest Dungeon | 哥特硬阴影、英雄队列、职业头像、火把压力条、怪物战斗站位、讽刺旁白腔 | 山海墨线、旧舆图、低神志污染、地点志和路引口吻 |
| 横屏镜头 | 直接照竞品截图摆地平线、车体、天空、UI | 自定画幅：舆图 / 路引信息为主，旅途层为氛围和状态 |
| 音效音乐 | 马蹄、缰绳、枪声、口琴、西部酒馆、哥特战斗鼓点 | 车轮、纸墨、风雾、木轴、祭所低频、异族市声 |

### 3.3 截图自检

合并前截一张地图、一张横屏行走、一张事件选择，逐张问：

1. 遮掉游戏名后，是否仍像《不思异：九州》，不是某个竞品换皮。
2. 最显眼元素是否是舆图、车队、地名、三资源，而不是载具剖面、英雄队列或星球网。
3. UI 是否在表达“选路和资源”，不是“实时驾驶 / 战斗站位 / 星系跳跃”。

## 4. AI 图片 / 视频 / 音乐素材入库记录

### 4.1 入库状态定义

| 状态 | 含义 | 是否可合并 |
|---|---|---|
| `Demo 临时` | 内部原型可用，来源和提示词已记录，但未做正式授权 / 相似性复核 | 可进内部 Demo，不能用于商店页、宣传图、正式发布包 |
| `后置复核` | 来源基本可追溯，但授权、平台条款、相似性或字体 / 音频权利尚未确认 | 可作为候选，不标正式 |
| `正式候选` | 来源、生成记录、授权状态、相似性初筛齐全，等待发布前总复核 | 可进入发布候选包 |
| `阻断禁用` | 来源不明、点名竞品 / 画师 / 影视 / 品牌，或结果明显相似 | 不进 Demo，不进宣传，不进素材库 |

### 4.2 最小入库字段

| 字段 | 必填内容 |
|---|---|
| 资产 ID | 如 `AI-IMG-CH1-MAP-001`、`AI-VID-CH1-FOG-001`、`AI-MUS-CH1-ROAD-001` |
| 文件路径 | 本地素材实际路径，不写模糊目录 |
| 类型 | 图片 / 视频 / 音乐 / 音效 / 字体 / UI 图标 |
| 使用位置 | 地图底图、横屏背景、地点插图、按钮图标、BGM、环境声等 |
| 生成或取得日期 | 年月日；多人并行时方便追溯版本 |
| 来源方式 | 自制 / AI 生成 / 用户上传 / 第三方库 / 古籍影印参考 / 其它 |
| 工具或平台 | 生成工具、模型、素材库、录音来源；未知就不能标正式 |
| 提示词摘要 | 记录核心提示，不点名竞品、画师、影视、品牌、角色 |
| 输入素材 | 是否使用参考图、参考视频、旋律、采样；必须记录来源和授权状态 |
| 许可证 / 条款状态 | 已确认 / 未确认 / 禁止商用 / 需署名 / 平台限制 |
| AI 披露要求 | 目标平台是否要求标注 AI 内容；未知标后置复核 |
| 相似性初筛 | 是否像竞品截图、现代图鉴、具体画师、影视游戏画面或音乐 |
| 当前状态 | `Demo 临时` / `后置复核` / `正式候选` / `阻断禁用` |
| 处理人和复核人 | 谁生成、谁看过、谁批准进入候选 |
| 阻断原因 | 仅红色素材填写，说明为什么不能用 |

### 4.3 AI 提示词红线

| 红线 | 处理 |
|---|---|
| 点名 Out There、FAR、Darkest Dungeon、DREDGE、Sunless Sea 等竞品 | 重写提示词，使用抽象玩法和本项目语汇 |
| 点名现代画师、影视、游戏、品牌、角色、音乐人 | 重写提示词 |
| 要求“某图鉴风”“某游戏 UI 风”“某电影镜头” | 重写为“古舆图、墨线、纸雾、朱印、山海异象”等抽象描述 |
| 使用来源不明参考图 / 参考视频 / 旋律 | 阻断禁用，除非补齐来源和授权 |
| 生成结果明显像某竞品、现代图鉴或具体作者风格 | 阻断禁用或重生成 |

### 4.4 首章素材特别提醒

1. 地图和 UI 素材优先使用自研符号、纹样、剪影和纸面效果，不用现代山海图鉴构图。
2. 横屏背景可用 AI 临时素材，但必须硬边、少虚化、无竞品影子；用于抠像或后期合成的素材要记录纯色背景版本。
3. 音乐和音效不能从影视、游戏、音乐平台、短视频平台截取；AI 音乐也必须记录工具、提示、输入旋律和商用条款。
4. 字体文件不清楚来源时不入库；优先系统字体或 OFL 字体，正式宣传图再单独复核授权。

## 5. 主线程合并前红黄绿检查表

### 5.2 黄灯：可合并内部 Demo，但必须登记

| 类别 | 黄灯项 | 合并条件 |
|---|---|---|
| AI 临时图 / 视频 / 音乐 | 来源和提示词已记录，但未完成正式授权与相似性复核 | 标 `Demo 临时`，不进宣传 |

| 视觉气质 | 截图有一点竞品影子，但未命中具体表达 | D/C 线程调整构图、颜色、图标、动画节奏 |

| 低神志文本 | 气质合适，但规则信息不够清楚 | 合并前补消耗、收益、锁定原因 |

### 5.3 绿灯：可进入本次合并

| 类别 | 绿灯标准 |
|---|---|
| 文案 | 地点名、雾中名、地点志、事件、低神志文本均为自写，且能映射路线 / 资源 / 神志功能 |
| 地图 | 舆图、纸雾、墨线、朱印、地名签成立；没有星图、太空和扫描影子 |
| 横屏 | 古道车队、山海远景、纸雾 / 泽雾 / 赤水氛围成立；没有西部、FAR、DD 的可识别组合 |
| 素材 | 每个入库素材至少有资产 ID、路径、类型、来源一句话、当前状态 |
| AI | 提示词不点名竞品 / 画师 / 影视 / 品牌；结果通过相似性初筛 |
| 宣传 | 只描述自有玩法和灵感来源，不绑定竞品名号 |

## 6. 给主线程的最短验收口令

每次合并前，主线程可直接按下面口令问一遍：

```text
1. 文案是不是全自写？有没有现代译文、百科、图鉴设定？
2. 地图是不是羊皮纸山海舆图？有没有星图 / 航线 / 扫描感？
3. 横屏是不是古道车队？有没有西部马车、FAR 载具、DD 队列？
4. 素材有没有 ID、路径、来源、状态？AI 有没有提示词和相似性初筛？
5. 宣传和 UI 里有没有竞品名、某某版、同款、换皮这类话？
```

任一项答不清楚，就先标黄；任一项命中红线，就暂停合并。
