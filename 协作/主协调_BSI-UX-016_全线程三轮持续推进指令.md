# BSI-UX-016 全线程三轮持续推进指令

> 游戏名：《不思异：九州》  
> 主线程原则：先把第一章网页版试玩跑顺；不讨论宣传，不扩大战斗/养成/商业化系统。所有交付仍回到主线程验收。

## 1. 本轮总目标

UX-016 的目标不是“再加一堆新功能”，而是把当前 Web Demo 从“机制能跑”推进到“玩家能玩懂、愿意继续点”。

本轮验收只看 6 件事：

1. 5 分钟试玩路径是否顺畅。
2. 玩家是否能看懂：城镇停靠、补给、九州图选路、半途路遇、抵达。
3. 三资源压力是否形成选择，而不是纯数字消耗。
4. 小横屏界面是否仍有遮挡、按钮不清、信息过多。
5. 状态反馈、音效和视觉提示是否能互相支撑。
6. 临时素材、AI 生成、竞品借鉴边界是否诚实记录。

## 2. 主线程工作

主线程本轮只做调度、验收和整合，不抢各线程专业工作。

主线程立即执行：

- 建立 UX-016 指令与 A-F 任务卡。
- 更新任务交接台账，标记 A-F 已批准继续执行。
- 收到子线程交付后，只做三类判断：
  - 可直接进入 Demo。
  - 需要 C 线程小改落地。
  - 需要用户拍板，进入统一决策收集区。

主线程禁止：

- 不直接把未验收素材标为正式可用。
- 不把竞品具体 UI/图标/音效/文本当作参考落地。
- 不要求 C 线程重构工程。
- 不因为某线程未回执而阻塞其它线程继续推进。

## 3. A 线程：数值与随机平衡

- 任务 ID：BSI-A-010
- 交付物：[A_BSI-A-010_试玩版数值复核与死局检查.md](/Users/yuanzhe/Documents/game/协作/A_BSI-A-010_试玩版数值复核与死局检查.md)
- 目标：复核当前 Web Demo 的第一章资源节奏、随机路遇密度、保底与死局防护。

必须产出：

- 3 条推荐试玩路径，每条 5 分钟内可走完核心循环。
- 当前数值风险：太松、太紧、随机太吓人、补给太慷慨。
- 可直接给 C 线程的微调表，最多 12 条。
- 最低可接受的死局防护规则。

禁止：

- 不写完整数值系统设计。
- 不新增复杂属性。
- 不要求 C 线程做模拟器大工程。

## 4. B 线程：地点志、事件与成就感文本

- 任务 ID：BSI-B-017
- 交付物：[B_BSI-B-017_试玩版地点志与成就感文本补强.md](/Users/yuanzhe/Documents/game/协作/B_BSI-B-017_试玩版地点志与成就感文本补强.md)
- 目标：让山海经亮点更强，让玩家每到一处有“见识到了”的成就感。

必须产出：

- 8 个优先地点的“抵达一句话 + 地点志 2-3 句 + 补给发现一句话”。
- 6 个半途路遇的正常/低神志短文本。
- 6 条可作为浮层/日志的成就感短句。
- 哪些文本可以直接替换进 Demo，哪些只是方向稿。

禁止：

- 不写百科腔。
- 不写长段剧情。
- 不使用现代解释口吻破坏气氛。
