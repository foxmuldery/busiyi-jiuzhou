# BSI-E-002 同类产品趋势与商业风险持续研究

> 子线程：E 商业合规与竞品研究顾问  
> 任务：同类产品趋势、可借鉴边界、禁止复制清单、商店/试玩节表述建议  
> 调研日期：2026-06-14  
> 结论性质：产品侧风险判断，不构成法律意见  
> 边界：不修改主产品文档，不修改 A/B/C/D/F 文件，不复制竞品表达

## 0. 执行摘要

同类产品给《不思异：九州》的安全启发不是“照着某个游戏做”，而是几条抽象趋势：

1. 路线资源生存：玩家愿意接受“每一步都付出代价”，但代价必须清楚、可复盘、不是随机惩罚。
2. 文字冒险商业化：文字量不是问题，问题是首个 Demo 是否能在 10-15 分钟内让玩家理解核心选择压力。
3. 横向旅途表现：横向舞台适合强化“在路上”，但只能作为氛围层和状态层，不能复制 FAR 系列的载具结构、镜头和关卡。
4. 低神志/认知污染：可借鉴“不可靠信息”“夜间压力”“感官错位”的体验目标，不可复制具体恐怖符号、音效、文案和怪物表达。
5. 独立游戏 Web/Steam Demo：Steam Next Fest 等试玩场景强调可玩 Demo、反馈、愿望单和社区；宣传要诚实说明“开发中/玩法验证”，避免把 Demo 包装成完整发行质量。

总建议：Web Demo 阶段用“自研规则 + 自研文本 + 横向山海旅途舞台 + 清晰资源反馈”建立差异化；外部作品只作为体验目标和风险边界参考。

## 2. 可以借鉴的内容

### 2.1 抽象机制

| 可借鉴项 | 来源类型 | 如何转译 |
|---|---|---|
| 路线选择有代价 | 80 Days、Roadwarden、Sunless Sea 等通用结构 | 每条路线显示车轴、粮草、神志和风险词，不使用竞品路线结构。 |
| 时间/天数压力 | 80 Days、Roadwarden、The Pale Beyond | 用“日数、天色、粮草消耗、归期/王命压力”转译。 |
| 队伍状态压力 | The Pale Beyond | 可做“随从信任、饥荒、争执、恐惧”，不要复制极地船员投票和具体事件。 |
| 恐惧/理智资源 | Sunless Sea、DREDGE、心理恐怖类 | 用“神志影响文本可信度、幻路、选项模糊”转译，不使用 Lovecraftian 商标化话术。 |
| 死亡后记忆点 | Sunless Sea、Roguelike 通用设计 | 用“荒外游记、残卷、地图残片、前队遗物”转译。 |
| 短 Demo 钩子 | Steam Next Fest 常见要求 | 10-15 分钟让玩家看到一次资源崩坏或神志文本变化。 |

### 2.2 体验目标

- 玩家感到“我必须选路，但每条路都不舒服”。
- 玩家在第一次神志下降时感到世界表达变了。
- 玩家失败后能复述“我因为某个选择死在某地”。
- 玩家愿意重开，因为不是被系统坑死，而是想试另一条路线。
- Demo 结束时留下“九州还没看懂”的欲望，而不是解释完设定。

### 2.3 通用 UI 模式

| UI 模式 | 可用程度 | 边界 |
|---|---|---|
| 顶部资源条 | 可直接用通用模式 | 视觉风格、图标、排版自研。 |
| 地点/事件文本主区 | 可直接用通用模式 | 不复制 Roadwarden 等文本分栏比例和插图风格。 |
| 路线卡片/路线列表 | 可直接用通用模式 | 资源消耗表达自研，避免同款标签和颜色系统。 |
| 横向旅途舞台 | 可控借鉴 | 只借“横向旅途感”，不复制 FAR 的载具横截面和关卡镜头。 |
| 行旅日志 | 可直接用通用模式 | 用本项目自己的“游记/残卷/队伍记事”语气。 |
| Demo 结束页 | 可直接用通用模式 | 说明开发中、邀请反馈和愿望单/预约，不夸大内容量。 |

### 2.4 资源压力曲线

建议 Web Demo 的压力曲线参考“可理解的早期失败”，不是“完整商业版长期运营”：

| 阶段 | 玩家感受 | 风险曲线 | 建议 |
|---|---|---|---|
| 0-3 分钟 | 明白自己在路上 | 低压 | 给一次低风险移动，展示车轴/粮草扣除。 |
| 3-8 分钟 | 第一次做困难选择 | 中压 | 一条省车轴但耗粮草，一条省粮草但伤神志。 |
| 8-12 分钟 | 第一次世界不可信 | 中高压 | 触发神志文本变化或幻路提示。 |
| 12-15 分钟 | 看到后果 | 高压但可控 | 给一次濒死、失败或带代价的抵达。 |
| Demo 结束 | 想再走一次 | 留钩子 | 显示未见地点、未解巫文、路线残片。 |

## 3. 不能碰的内容

### 3.1 具体表达禁止清单

| 类型 | 禁止内容 | 风险 |
|---|---|---|
| UI 布局 | 复制 Roadwarden 的左右分栏比例、地图/文本/状态排列细节 | 高风险 |
| 横向镜头 | 复制 FAR 的载具横截面、路障解谜、车内站位、机械操作流程 | 禁止 |
| 文案 | 复制 Steam 页短句、事件文本、失败文本、角色口吻 | 禁止 |
| 图像 | 使用竞品截图、商店图、图标、角色、场景构图、色彩识别系统 | 禁止 |
| 音效/音乐 | 模仿具体游戏的主题旋律、船/车声、恐怖音效、环境声组合 | 禁止 |
| 关卡结构 | 复制 FAR 的障碍顺序、DREDGE 的群岛结构、Sunless Sea 的港口循环 | 禁止 |
| 恐怖桥段 | 复制具体怪物、夜间追逐、精神污染表现、台词反转 | 高风险 |
| 商店话术 | “山海经版 Out There / Sunless Sea / FAR” | 禁止 |
| 历史/神话资料 | 复制现代译文、百科条目、现代图鉴设定 | 高风险 |

### 3.2 对各线程的边界提示

- A 线程：可参考资源压力曲线，不复制竞品数值、资源名称、死亡继承规则。
- B 线程：可参考“失败有记忆点”，不复制事件结构、文案语气、结局揭示方式。
- C 线程：可实现通用 UI 和 Demo 反馈机制，不引入未经复核的运行时、素材包或模板。
- D 线程：可做横向山海旅途舞台，不复制 FAR/DREDGE/Sunless Sea 的构图、镜头、色彩识别。
- F 线程：可做“远行、危险、认知错位”的声音目标，不模仿竞品旋律、音效和混音特征。

## 4. 面向 Steam / TapTap / 试玩节的表述建议

### 4.1 Steam 页面

Steam 页面适合直接说明玩法，不要只堆世界观。

推荐短描述：

```text
一款山海经题材的远行生存叙事游戏。管理车轴、粮草与神志，在节点路线与文字事件中穿越九州；越深入荒外，地图、文本和选择越不可靠。
```

推荐要点：

- 写清楚：文字事件、路线选择、三资源压力、神志影响文本。
- 写清楚：Demo 是开发中版本，内容为第一张地图纵切片。
- 可以使用标签：Interactive Fiction、Resource Management、Choices Matter、Atmospheric、Roguelite、Story Rich。
- 谨慎使用标签：Lovecraftian、Psychological Horror、Survival Horror。若使用，要确保 Demo 真的有对应体验。

应避免：

- “中国版某某游戏”。
- “最真实还原山海经”。
- “完整九州开放世界”。
- “上百小时内容”或其他尚未完成的量级承诺。
- 使用 AI 生成素材但未做 Steam 所需披露。

### 4.2 TapTap 页面

TapTap 页面更适合短句、可讨论点和清楚的测试范围。

推荐短描述：

```text
这不是打怪升级的山海经游戏。你带着一支车队往荒外走，车轴会断，粮草会空，神志低了以后连地图和文字都未必可信。
```

推荐要点：

- 第一屏文案要说清“玩什么”：选路、扣资源、遇事件、神志变文本。
- 测试公告要写清楚“这是玩法验证 Demo，不代表最终美术和完整内容”。
- 鼓励玩家反馈：哪条路线看不懂、哪个资源压力不清楚、神志变化是否有效。
- 若使用 AI 临时素材，建议在测试说明里主动标注“内部 Demo 临时素材，正式版将复核授权/替换”。

应避免：

- 用“国产克苏鲁山海经”当主宣传语。
- 用“Steam 爆款同款玩法”吸引点击。
- 把未授权音乐、图像或视频素材作为正式宣传图。

### 4.3 Steam Next Fest / 试玩节

Steamworks 官方文档说明 Next Fest 面向未发售游戏，核心是公开可玩 Demo、玩家反馈和未来发布受众建设。2026 年 6 月场官方文档日期为 2026-06-15 10:00 PDT 到 2026-06-22 10:00 PDT。

建议 Demo 展示目标：

- 10 分钟内让玩家经历一次路线选择和资源压力。
- 15 分钟内让玩家看到一次神志文本变化。
- 结尾给出反馈入口、愿望单/预约引导和“内容开发中”说明。
- Demo 不要过度依赖外部视频和音乐，避免被看成素材展示而不是玩法展示。

试玩节页面建议写：

```text
当前 Demo 展示第一张地图的核心循环：选路、消耗、事件、神志文本变化和失败游记。地图、美术、音频与数值均为开发中版本，欢迎反馈哪些选择最清楚、哪些风险最有记忆点。
```

## 5. 风险矩阵

| 风险 | 等级 | 触发方式 | 避险建议 |
|---|---|---|---|
| 竞品依附风险 | 高 | 宣传页直接对标 Out There、Sunless Sea、FAR、DREDGE | 只讲自家题材和机制，不写“某某版”。 |
| UI 近似风险 | 高 | 路线/文本/资源布局与某竞品高度相似 | 使用自研山海旅途舞台与游记日志结构。 |
| 横向舞台误导风险 | 可控 | 玩家误以为是横版动作或 FAR 式载具游戏 | 商店页写明“节点路线 + 文字事件”，舞台只是表现层。 |
| 恐怖标签错配 | 可控 | 使用 Psychological Horror / Lovecraftian 但 Demo 不明显 | 优先用“认知污染”“不可靠文本”，标签谨慎。 |
| 试玩节承诺过度 | 高 | Demo 页面承诺完整九州、完整美术、完整结局 | 写“开发中纵切片”，列明 Demo 范围。 |
| AI/素材披露风险 | 高 | 宣传图、音频、视频用了 AI 或未授权素材未披露 | 建授权台账；Steam 按内容调查要求披露；TapTap 测试说明主动标注。 |
| 现代资料版权风险 | 高 | 现代译文/百科/插画进入正文或宣传 | 只用原典名词和自研转译；资料台账分开记录。 |
| 音乐音效相似风险 | 高 | 模仿 Sunless Sea、FAR、DREDGE 的氛围音乐和音效 | F 线程使用抽象提示语，禁止具体作品名参考。 |

## 6. BSI-E-002 当前建议动作

1. Web Demo 对外口径统一为“山海经题材远行生存叙事游戏”，不使用“某某版”。
2. Demo 第一版优先验证“路线资源压力 + 神志文本变化”，不要把重点放在大素材展示。
3. 横向旅途舞台在商店页中称为“旅途舞台 / 路线表现”，不要称为横版关卡。
4. 每个外部竞品只保留“可借鉴抽象点”和“不可复制表达点”，不要再继续收集截图做视觉参考。
5. 若参加 Steam Next Fest，需要提前准备公开商店页、Demo、反馈入口、愿望单引导和 AI/素材披露。

## 8. 待主线程/用户决策问题

1. 对外宣传是否完全避开 “Lovecraftian / 克苏鲁”，只用“认知污染 / 不可靠文本 / 荒外异闻”？

4. 是否建立公开版素材披露口径，区分“临时 Demo 素材”和“正式授权素材”？
5. 是否把 Roadwarden、80 Days、FAR、Sunless Sea、DREDGE 列入“只做内部研究，不作为视觉/文案参考”的固定禁区？
