## 0. 文本口径

第一章地点要让玩家每到一处都像翻到一页《山海经见闻录》：地名有来历，资源有理由，危险不是纯数值惩罚。手机横屏展示时建议每个地点只露三行核心文本：

```text
地点志：一句神秘感 + 一句危险或风俗。
资源差异：偏补什么，缺什么。
行动：一次性补给或剧情选择，结果一句话。
```

C 线程可直接映射字段：

```text
id, name, locationType, terrain, fogName, revealText,
resourceBias, scarceResources, supplies[], storyChoice, lowSanityText
```

资源方向沿用当前原型：`车轴 axle / 粮草 grain / 神志 sanity`。以下数值为 B 线程建议区间，A/C 可按平衡再压缩。

## 2. 地点志与资源差异

### 2.1 中原驿

- `id`：`central_post`
- 地点类型：都邑 / 驿城
- 雾中名称：人间最后一灯
- 到达显影文本：驿灯照着旧王朝的里程木牌，西边数字被风沙磨平。这里仍按人间规矩记账、分粮、修车，也因此显得像一处即将失效的旧秩序。
- 资源特点：偏补车轴、粮草；神志不稀缺，但后续再难找到这种稳定感。
- 一次性补给：`借驿钉修轴`，驿卒取出几枚旧铜钉，车轮声稳了一些；车轴 +8，粮草 -2。
- 剧情选择：`问西行旧账`，翻看驿簿可得“没人按时回来”的线索；神志 -2，获得 `old_account`。
- 神志低扭曲文本：驿门没有关。车轴才转半圈，驿卒已经把你们的归期写成旧账。

### 2.2 青丘外邑

- `id`：`qingqiu_outer_city`
- 地点类型：都邑 / 边邑
- 雾中名称：狐灯小城
- 到达显影文本：城墙不高，墙上挂着许多小灯，灯火像人的眼，也像兽的眼。城中人说话温和，交易却总要绕过真实姓名。
- 资源特点：偏补神志和少量粮草；车轴材料稀缺。
- 一次性补给：`换狐灯静夜`，用一袋粟换一盏不灭小灯，夜路少些幻影；神志 +7，粮草 -4。
- 剧情选择：`留假名入册`，可换得一段边邑路引；神志 -3，获得 `false_name_pass`。
- 神志低扭曲文本：灯不是挂在墙上，是挂在你们眼里。每眨一次，城里就少一个人。

### 2.3 故王道

- `id`：`old_king_road`
- 地点类型：古道 / 旧都余脉
- 雾中名称：西北深车辙
- 到达显影文本：半埋石基从纸纹里浮出，深辙仍把车队往西牵。随从说白日它是直道，入暮后会分出不属于任何舆图的岔路。
- 资源特点：偏补车轴；粮草稀缺；神志轻微受扰。
- 一次性补给：`垫旧道石`，撬起半埋王道石垫车底；车轴 +7，神志 -2。
- 剧情选择：`验深浅两辙`，辨出哪道车辙更新；粮草 -3，神志 +3。
- 神志低扭曲文本：车辙不是车留下的，是路睁开的两只眼。它们看向西，也看向你。

### 2.4 废关

- `id`：`abandoned_pass`
- 地点类型：山隘 / 关塞
- 雾中名称：关影压纸
- 到达显影文本：塌关显出门额，旧令上“不得西行”四字未褪。关楼木梁和铁钉能救车，墙上的字却会让人怀疑为什么还要往前。
- 资源特点：强补车轴；粮草少；神志稀缺。
- 一次性补给：`拆门木补车`，锯下一截冷硬门梁补轴；车轴 +10，神志 -3。
- 剧情选择：`拓下旧令`，拓片未干，字像从纸背往外生；神志 -5，获得 `old_order`。
- 神志低扭曲文本：旧令改了三次：不得西行，不得归来，不得看见自己已经过关。

### 2.5 鸟鼠夹道

- `id`：`bird_mouse_pass`
- 地点类型：山隘 / 兽穴山口
- 雾中名称：鸟鼠夹道
- 到达显影文本：两山夹出一线窄路，洞穴密得像被针扎过。白日无声，夜里却像满山都在搬运粮草和骨片。
- 资源特点：可小补车轴，粮草消耗低；神志风险中等。
- 一次性补给：`拾洞口硬枝`，拾取被啃齐的硬枝作轮楔；车轴 +5，神志 -2。
- 剧情选择：`等夜声辨路`，听洞中搬运声判断山腹空处；粮草 -2，获得 `hollow_pass_hint`。
- 神志低扭曲文本：洞里不是鸟鼠，是许多小路在磨牙。它们等车轮瘦到刚好能吞下。

### 2.6 雷泽浅畔

- `id`：`thunder_marsh`
- 地点类型：泽国
- 雾中名称：泽上伏雷
- 到达显影文本：地图先渗出一圈湿痕，随后芦草和白羽显形。雷声不在天上，在车轮将要压过的泥下。
- 资源特点：偏补粮草；车轴和神志容易受损。
- 一次性补给：`采雷泽芦根`，芦根能煮成糊，咬开时有远雷；粮草 +10，神志 -3。
- 剧情选择：`听雷辨浅路`，按地底雷声找水下硬土；神志 -6，获得 `heard_ground_thunder`。
- 神志低扭曲文本：泥下有一面鼓。你们每少一口粮，它就替队伍多记一声。

### 2.7 白羽淖

- `id`：`white_feather_mire`
- 地点类型：泽国 / 异象水地
- 雾中名称：白羽沉水
- 到达显影文本：浅淖里漂着白羽，却没有一枚真正浮起。当地人说轻物在这里欠债，重物反而能渡过去。
- 资源特点：粮草中补，可能给羽民线索；神志稀缺。
- 一次性补给：`捞白羽鱼`，鱼鳞像羽，肉能充饥；粮草 +8，神志 -4。
- 剧情选择：`留重物试渡`，丢下一件旧铁换取羽民渡口方向；车轴 -3，获得 `feather_ford_hint`。
- 神志低扭曲文本：羽毛沉着，水却浮着。渡口藏在上下颠倒的地方。

### 2.8 黑齿市

- `id`：`black_teeth_market`
- 地点类型：墟市 / 异族边市
- 雾中名称：日中有市影
- 到达显影文本：正午的空白处忽然立起棚帐，陶器、兽骨和黑粟围成无声市口。市人不喊价，只把旧铁和明日放在同一张兽皮上。
- 资源特点：偏补粮草、神志；车轴常作为代价。
- 一次性补给：`以旧铁换粮`，换来的粮袋沉得出奇；粮草 +12，车轴 -3。
- 剧情选择：`换取白羽`，买下一枚轻得过分的白羽；粮草 -8，神志 -3，羽民语 +20。
- 神志低扭曲文本：市不是市，是一张合不拢的口。它买明日，找零昨日。

### 2.9 昼骨墟

- `id`：`daybone_ruins`
- 地点类型：墟市 / 骨棚市
- 雾中名称：骨棚无声
- 到达显影文本：墟市只在影子最短时出现，货摊由兽骨支起，摊主把价钱刻在骨背。日影一歪，整座市就会像没来过。
- 资源特点：偏补粮草；神志高风险；车轴材料少。
- 一次性补给：`买骨背干肉`，干肉可食，骨背价钱会自己改字；粮草 +9，神志 -4。
- 剧情选择：`抄下骨价`，记录异族计价方式；神志 -2，获得 `bone_price_note`。
- 神志低扭曲文本：摊棚站在骨头上，骨头站在影子上。影子一短，价钱就开始数人。

### 2.10 无名祠

- `id`：`nameless_shrine`
- 地点类型：祭所
- 雾中名称：小祠无匾
- 到达显影文本：路旁小祠没有匾额，香灰却是新的。祠后有短路，也有几道不像人的脚印。
- 资源特点：偏补神志或路线提示；粮草为代价。
- 一次性补给：`献粟问路`，粟粒落进香灰，灰面显出半截弯路；粮草 -4，神志 +5。
- 剧情选择：`记下供位空名`，供桌空位像缺一个称呼；神志 -2，获得 `nameless_seat`。
- 神志低扭曲文本：祠在路旁，也在路中。你们每走一步，它就近一点。

### 2.11 巫咸断碑

- `id`：`broken_stele`
- 地点类型：祭所 / 巫文界标
- 雾中名称：黑土中缺一笔
- 到达显影文本：断碑半埋在黑土里，碑文像方位，也像人名。懂残笔的人能借它补全西路，看久的人则觉得缺的是自己。
- 资源特点：强补神志或解锁裂隙；粮草、车轴会消耗。
- 一次性补给：`依碑文正名`，众人按碑文重新叫出彼此名字；神志 +6，车轴 -2。
- 剧情选择：`拓碑带走`，拓片卷起时，残画像车队又像裂口；神志 -7，巫咸古辞 +20。
- 神志低扭曲文本：碑不是缺字。碑在等你把名字还给它，补上一笔就少一日。

### 2.12 赤水外滩

- `id`：`red_marsh`
- 地点类型：禁地
- 雾中名称：红水在纸背
- 到达显影文本：赤色水线从地图边缘渗出，岸骨一节节排成路标。白日看是禁地，夜里却像有人在水下铺路。
- 资源特点：少量粮草；神志高压；车轴易损。
- 一次性补给：`采岸草充饥`，岸草能煮成糊，颜色洗不净；粮草 +7，神志 -5。
- 剧情选择：`沿骨行路`，骨路确实指向前方；车轴 -4，神志 -10，获得 `followed_bones`。
- 神志低扭曲文本：骨先到，水后到。有人说夜里才浅，也有人说浅的是你们的心。

### 2.13 归不得林

- `id`：`returnless_wood`
- 地点类型：禁地 / 幻路林
- 雾中名称：林中无归鸟
- 到达显影文本：林木不密，却每棵都像来路。鸟声只从背后传来，随从越想辨方向，越觉得自己已经走过这里。
- 资源特点：可补车轴木料；粮草少；神志稀缺。
- 一次性补给：`伐无纹直木`，直木可作辐条，却没有年轮；车轴 +8，神志 -6。
- 剧情选择：`系绳标归路`，每十步系一次绳，回头时绳结都朝前；粮草 -3，获得 `return_thread`。
- 神志低扭曲文本：你们没有进林，是林把来路长成你们的样子，站在每一棵树后。

### 2.14 羽民渡

- `id`：`feather_folk_ford`
- 地点类型：异族边境
- 雾中名称：羽落成桥
- 到达显影文本：渡口没有桥，只有许多白羽压在水面。羽民以轻物渡水，重物要留下一个名字。
- 资源特点：可省车轴或开启水路；耗粮，可能损神志。
- 一次性补给：`以粮换羽舟`，羽舟轻得像影子，能带车队过浅水；粮草 -7，车轴 +4。

- 神志低扭曲文本：羽不是桥，是许多未说出口的名字。你踏上一枚，就少记得一件事。

### 2.15 白民盐界

- 地点类型：异族边境
- 雾中名称：白盐划界
- 到达显影文本：白盐线横过荒地，线内的人衣色如灰，问路前先问影子是否跟来。盐能保食，也能磨淡地图上的水线。
- 资源特点：偏稳粮草；车轴稀缺；神志小风险。
- 一次性补给：`换盐封粮`，盐封住粮袋霉味；粮草 +8，神志 -2。

- 神志低扭曲文本：盐线不是画在地上，是画在你们脚底。每跨一步，影子都被留下一层。

### 2.16 梦图驿

- `id`：`dream_map_post`
- 地点类型：裂隙前哨
- 雾中名称：梦图自卷
- 到达显影文本：夜里地图自己卷起，卷口像一扇小门。醒来后门还在纸上，只是没人能证明昨夜曾经开过。
- 资源特点：神志可补也可损；粮草稀缺；车轴收益低。
- 一次性补给：`整束梦图`，用绳重新束住残图，队伍短暂确认方位；神志 +7，粮草 -4。
- 剧情选择：`按梦门标记前哨`，地图多出一枚裂隙前记号；神志 -6，获得 `dream_gate_mark`。
- 神志低扭曲文本：地图睡着了。你们醒在它的梦里，驿门开在纸背，不开在人间。

### 2.17 九州裂隙

- `id`：`kyushu_rift`
- 地点类型：裂隙前哨 / 第一章终点
- 雾中名称：纸尽处有裂
- 到达显影文本：地图在此处不再画山水，只留一道折错的破痕。车队抵达时，来路和去路都变薄，梦中地图开始替现实标注下一步。
- 资源特点：偏神志整理；粮草、车轴收益低；适合章节收束。
- 一次性补给：`整束残图`，残图被重新束好，裂隙短暂像可以理解的路；神志 +8，粮草 -4。
- 剧情选择：`照梦图前进`，驶向地图没有画完处；神志 -12，进入 `rift` 结局或下一章。
- 神志低扭曲文本：前方不是下一站，是纸破的声音。车轮只是在裂口边反复打转。

## 3. 推荐先放进 Demo 的 6 个地点

优先建议：

1. `central_post` 中原驿
2. `abandoned_pass` 废关
3. `thunder_marsh` 雷泽浅畔
4. `black_teeth_market` 黑齿市
5. `red_marsh` 赤水外滩
6. `broken_stele` 巫咸断碑

理由：

- 三资源差异最清楚：中原驿稳底，废关补车轴，雷泽补粮，黑齿市交易，赤水高压，断碑补神志和解锁。
- 路线节奏完整：人间起点 -> 关塞 -> 泽国 / 墟市 -> 禁地 -> 祭所解锁。
- 第一章神秘感够用：旧令、地底雷、黑齿交易、赤水岸骨、巫文断碑能直接形成山海见闻。

若 Demo 需要当场闭环结局，再把 `kyushu_rift` 九州裂隙作为第 7 个终点节点接入。

## 4. 给 C 线程的字段拆分建议

地点最小字段：

```js
{
  id: "abandoned_pass",
  name: "废关",
  locationType: "山隘 / 关塞",
  fogName: "关影压纸",
  terrain: "market",
  revealText: "塌关显出门额，旧令上“不得西行”四字未褪。",
  resourceBias: ["axle"],
  scarceResources: ["sanity", "grain"],
  supplies: [
    {
      id: "pass_timbers",
      label: "拆门木补车",
      hint: "车轴 +10，神志 -3",
      result: "锯下一截冷硬门梁补轴；车轮稳了，旧令少了一角。"
    }
  ],
  storyChoice: {
    label: "拓下旧令",
    hint: "神志 -5，获得 old_order",
    result: "拓片未干，字像从纸背往外生。"
  },
  lowSanityText: "旧令改了三次：不得西行，不得归来，不得看见自己已经过关。"
}
```

实现优先级建议：

1. 先加 `locationType / fogName / revealText / resourceBias / scarceResources / lowSanityText`，不影响现有逻辑。
2. `supplies[]` 可继续用现有结构，保证一次性消耗由 `usedSupplies` 控制。
3. `storyChoice` 可先映射到现有 `events[id].choices[]`，后续再拆独立字段。
