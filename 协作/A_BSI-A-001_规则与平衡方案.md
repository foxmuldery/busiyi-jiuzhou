# A_BSI-A-001 规则与平衡方案

> 子线程：A，核心规则与数值平衡设计师  
> 任务：BSI-A-001，三资源与随机保底规则细化  
> 范围：仅服务 Web Demo 第一版测试原型，不扩展到正式版复杂系统  
> 禁止：不写正式剧情文本，不改主产品文档，不写代码

## 1. 设计目标

Web Demo 第一版的平衡目标不是做出最终商业版难度，而是验证三件事：

1. 玩家能在 10 分钟内理解车轴、粮、神志的区别。
2. 随机事件带来压力，但不随机生成死局。
3. 失败可复盘，玩家能说出自己主要败在路线、粮耗、车轴、神志或贪心选择上。

建议第一版单局目标：

| 指标 | 建议目标 |
|---|---:|
| 单局长度 | 18 到 30 次移动或事件 |
| 平均游玩时长 | 20 到 35 分钟 |
| 首轮通关率 | 20% 到 35% |
| 熟悉后通关率 | 40% 到 55% |
| 非自愿死局率 | 低于 1% |
| 失败后能看到明确失败原因 | 100% |

这里的“死局”指玩家没有可理解选择、没有补救机会、只能看着资源归零失败。资源低导致失败可以存在，但必须给最后抉择或复盘原因。

## 2. 三资源初始值与基础规则

### 2.1 初始值

Web Demo 第一版建议三资源都采用 0 到 100 的上限，便于 UI 显示和玩家理解。

| 资源 | 初始值 | 上限 | 低值预警 | 危急线 | 归零状态 |
|---|---:|---:|---:|---:|---|
| 车轴 | 80 | 100 | 30 | 12 | 断轴 |
| 粮 | 90 | 100 | 35 | 15 | 饥荒 |
| 神志 | 85 | 100 | 45 | 20 | 崩解 |

初始值不满 100，是为了让玩家从第一步就知道资源是旅途压力，而不是满状态血条。

### 2.2 资源定位

| 资源 | 玩家理解 | 系统作用 | 主要损耗 | 主要恢复 |
|---|---|---|---|---|
| 车轴 | 能不能继续走 | 移动能力与路线选择 | 路线、恶劣地貌、强行穿越、异兽冲击 | 修复、拆器物、驿站、低风险休整 |
| 粮 | 能不能撑过今天 | 生存倒计时与队伍稳定 | 每次移动、绕路、等待、休整、队伍事件 | 采集、交易、狩猎、补给点、牺牲代价 |
| 神志 | 看到的是否可信 | 文本层级、选项清晰度、隐藏路线 | 神域、夜路、巫文、幻觉、低神志选择 | 休整、祭祀、记录、稳定器物、返程节点 |

### 2.3 基础消耗公式

每次移动至少消耗粮，车轴按路线消耗，神志只在风险、地貌或事件要求时消耗。

```text
移动总消耗 = 路线基础消耗 + 地貌修正 + 事件修正 + 器物修正
```

资源结算顺序建议：

1. 预览路线消耗。
2. 玩家确认移动。
3. 扣除路线消耗。
4. 抵达后抽取事件。
5. 执行事件选项消耗和收益。
6. 检查低值、归零、濒死、结局。

### 2.4 恢复方式

恢复方式必须带选择成本，避免玩家只要反复休息就能满状态。

| 恢复方式 | 触发位置 | 恢复建议 | 代价或限制 |
|---|---|---:|---|
| 修车 | 驿站、废关、工匠事件 | 车轴 +12 到 +25 | 消耗粮 4 到 8，或失去小器物 |
| 采集 | 泽、林、山脚等低中风险地点 | 粮 +10 到 +22 | 可能触发小事件，或神志 -2 到 -6 |
| 交易 | 异族、人间边境 | 粮或车轴 +12 到 +25 | 消耗器物、线索或语言条件 |
| 休整 | 安全点、喘息点 | 神志 +8 到 +18 | 粮 -5 到 -10，不可连续无限使用 |
| 祭祀 | 神祇、遗迹、异常地点 | 神志 +10 或特殊补救 | 可能扣粮、器物或增加后续风险 |
| 记录 | 任意非危机节点，限次数 | 神志 +5 到 +10 | 每 3 步最多一次，不能在崩解中使用 |

## 3. 路线消耗分级

Web Demo 第一版用低、中、高三档路线即可。路线预览可以显示大致消耗和风险，不需要公开所有隐藏权重。

### 3.1 基础路线表

| 路线等级 | 典型含义 | 车轴 | 粮 | 神志 | 事件风险 |
|---|---|---:|---:|---:|---|
| 低风险 | 驿路、平原、已知渡口 | -3 到 -5 | -3 到 -5 | 0 到 -2 | 低 |
| 中风险 | 山路、泽边、异族边境 | -6 到 -9 | -5 到 -8 | -2 到 -6 | 中 |
| 高风险 | 夜路、禁地、神域、强渡 | -10 到 -15 | -7 到 -11 | -8 到 -16 | 高 |

### 3.2 地貌修正

| 地貌 | 车轴修正 | 粮修正 | 神志修正 | 说明 |
|---|---:|---:|---:|---|
| 平原/驿路 | -1 到 0 | 0 | 0 | 适合作为喘息路线 |
| 山地 | -2 到 -4 | -1 | 0 到 -2 | 主要压车轴 |
| 泽地/水路 | -2 到 -5 | -1 到 -2 | -1 到 -4 | 可被涉水器物降低 |
| 流沙/荒漠 | -1 到 -3 | -3 到 -5 | -1 到 -3 | 主要压粮 |
| 异族边境 | 0 到 -2 | 0 到 -2 | -2 到 -5 | 主要压神志和事件风险 |
| 神域/异常地点 | 0 到 -3 | 0 到 -2 | -6 到 -12 | 可能给隐藏收益 |

### 3.3 路线预览规则

玩家应能看到：

- 低风险路线：显示明确消耗，例如“车轴 -4，粮 -4”。
- 中风险路线：显示明确主消耗和模糊副风险，例如“车轴 -7，粮 -6，可能损神志”。
- 高风险路线：显示警告和范围，例如“车轴约 -12，粮约 -9，神志风险高”。

低神志时可以让提示变得不可靠，但不能完全取消可玩信息。最低限度仍要保留路线等级、至少一个主要消耗方向。

## 4. 随机事件抽取规则

### 4.1 事件池分类

每个地点或路线事件池至少分为 6 类：

| 事件池 | 作用 | 资源倾向 |
|---|---|---|
| 风险事件 | 制造损耗和选择压力 | 扣 1 到 2 种资源 |
| 补给事件 | 提供恢复或交换 | 补 1 种资源，可能扣另一种 |
| 线索事件 | 提供地图、语言、隐藏路线信息 | 少量资源代价或无直接收益 |
| 器物事件 | 获得、损坏、交换器物 | 改变后续规则 |
| 神志事件 | 改变文本层级和隐藏认知 | 高风险高收益 |
| 喘息事件 | 低风险、整理状态、复盘节奏 | 小恢复或无损耗 |

第一版不需要复杂剧情链，但每个地点至少要能从风险、补给、线索三类里抽取。

### 4.2 基础权重

按路线等级调整事件池权重：

| 路线等级 | 风险 | 补给 | 线索 | 器物 | 神志 | 喘息 |
|---|---:|---:|---:|---:|---:|---:|
| 低风险 | 20 | 25 | 20 | 10 | 5 | 20 |
| 中风险 | 35 | 18 | 18 | 10 | 12 | 7 |
| 高风险 | 48 | 10 | 12 | 8 | 18 | 4 |

地点本身可以有主题修正。例如神域提高神志事件，边境提高交易和语言线索，但总权重仍应经过死局防护检查。

### 4.3 badLuckMeter 坏运气保险

`badLuckMeter` 是隐藏值，初始为 0，建议范围 0 到 100。

增加规则：

| 情况 | badLuckMeter 增加 |
|---|---:|
| 抽到风险事件且净损耗超过 12 | +12 |
| 连续第二次风险事件 | +10 |
| 连续第三次风险事件 | +20 |
| 任一资源进入危急线 | +15 |
| 玩家没有获得任何正向收益连续 3 步 | +15 |
| 濒死状态被触发 | +25 |

减少规则：

| 情况 | badLuckMeter 减少 |
|---|---:|
| 抽到补给或喘息事件 | -20 |
| 玩家获得器物、关键线索或隐藏路线 | -15 |
| 任一资源从危急线恢复到低值预警以上 | -20 |
| 完成一次濒死补救 | -25 |

权重调整：

| badLuckMeter | 调整 |
|---:|---|
| 0 到 24 | 不调整 |
| 25 到 49 | 补给 +8，喘息 +5，风险 -8 |
| 50 到 74 | 补给 +15，喘息 +10，风险 -15，禁止连续高损耗事件 |
| 75 到 100 | 下一次必须从补给、喘息、低代价线索三类中抽取 |

`badLuckMeter` 不是保送通关，只是防止随机连续重击让玩家失去操作空间。

### 4.4 连续事件限制

Web Demo 第一版建议使用以下硬限制：

1. 不允许连续 3 次只扣资源、不给选择或收益的事件。
2. 不允许连续 2 次让同一资源净损耗超过 15，除非玩家主动选择高风险选项。
3. 玩家三资源都低于预警线时，下一次事件必须包含至少一个补救选项。
4. 玩家刚完成濒死补救后的 2 步内，不抽取高损耗风险事件。
5. 关键路线、关键线索、基础补给不能只依赖一次随机事件。

## 5. 死局防护规则

### 5.1 必须触发补救的情况

以下情况必须提供补救事件、濒死选项或明确的低风险退路：

| 情况 | 必须提供的补救 |
|---|---|
| 车轴低于 12 且所有可见路线车轴需求都高于当前值 | 修车、拆器物、弃重、徒步短行 |
| 粮低于 15 且下一次移动会导致粮归零 | 采集、交易、狩猎、求粮、牺牲换粮 |
| 神志低于 20 且所有可见路线都扣神志 | 休整、记录、祭祀、巫祝代记、退回已知点 |
| 三资源同时低于预警线 | 喘息点或多资源交换事件 |
| 玩家连续 3 步没有正向收益 | 补给、线索或安全路线 |
| 当前节点无可走路线 | 自动生成短退路或困境事件，不直接失败 |
| 关键结局路线被随机事件阻断 | 提供替代线索或第二入口 |

### 5.2 可以失败的情况

以下失败可以接受，因为玩家有明确选择或可复盘原因：

- 玩家明知资源不足仍选择高风险路线。
- 玩家多次跳过补给或休整。
- 玩家把关键器物用于交换，导致后续路线代价升高。
- 玩家低神志时选择幻路，并忽略之前线索。
- 玩家在濒死状态中选择继续冒险而不是补救。

### 5.3 不允许的情况

第一版禁止以下体验：

- 进入节点后没有任何可选路线，也没有事件。
- 资源归零后立刻失败，没有最后抉择。
- 关键路径只靠一次随机抽中。
- 连续多个事件只扣资源、没有选择。
- 低神志给出完全不可判断的路线信息。
- 玩家失败后只显示“你死了”，不显示主要原因。

## 6. 濒死状态方案

濒死状态是 Web Demo 的重要安全阀。它不是奖励，而是最后一次把失败变成选择。

### 6.1 通用规则

1. 任一资源归零时，不立刻结算失败，先进入对应濒死状态。
2. 每种濒死状态每局最多触发 1 次完整补救。
3. 同一局最多触发 2 次濒死补救；第三次资源归零可以直接进入失败或特殊结局。
4. 濒死补救必须付出明确代价。
5. 补救后资源恢复到最低可操作值，不恢复到安全值。

### 6.2 车轴归零：断轴

玩家心理：走不出去。

| 选项类型 | 效果 | 代价 |
|---|---|---|
| 拆器修车 | 车轴恢复到 18 到 25 | 失去 1 件普通器物，粮 -4 |
| 舍弃行囊 | 车轴恢复到 12 到 18 | 粮上限 -10 或失去部分线索 |
| 徒步短行 | 移动到最近低风险节点 | 粮 -8，神志 -10 |
| 等待救援/灾厄 | 抽一次特殊困境事件 | 可能恢复车轴，也可能转入失败 |

若玩家没有器物、没有粮、没有相邻低风险节点，系统必须给“等待救援/灾厄”作为最后出口。

### 6.3 粮归零：饥荒

玩家心理：撑不过今天。

| 选项类型 | 效果 | 代价 |
|---|---|---|
| 采集求生 | 粮恢复到 15 到 22 | 车轴 -4，可能神志 -4 |
| 宰牲/弃车载物 | 粮恢复到 18 到 25 | 车轴上限 -8 或车轴 -8 |
| 向异族求粮 | 粮恢复到 20 到 30 | 需要地点条件；失去器物或线索 |
| 梦中粮 | 粮恢复到 25 | 神志 -15，增加幻觉事件权重 |
| 舍弃随从口粮 | 粮恢复到 12 到 20 | 队伍稳定下降，后续事件风险上升 |

Web Demo 如果暂不做随从系统，可以把“舍弃随从口粮”抽象为“队伍稳定 -1”，只在失败原因和事件权重中体现。

### 6.4 神志归零：崩解

玩家心理：看到的不再可信。

| 选项类型 | 效果 | 代价 |
|---|---|---|
| 强行清醒 | 神志恢复到 12 到 18 | 粮 -8，下一次神志事件权重 +10 |
| 让巫祝代记 | 神志恢复到 20 | 需要相关器物或标记；失去一次记录机会 |
| 接受幻路 | 不恢复神志，进入特殊路线或特殊失败分支 | 路线信息不稳定，但可能获得隐藏线索 |
| 原路逃返 | 回到上一个安全节点，神志恢复到 15 | 车轴 -8，粮 -8，放弃当前线索 |

神志归零不一定是坏结局入口，但必须让玩家知道自己正在选择“更危险的理解方式”，不能伪装成普通安全路线。

## 7. 自动模拟器指标

技术线程未来跑 1000 局测试时，应先实现规则模拟，不要求正式文案。每局记录路线选择、事件类型、资源变化、失败原因和结局。

### 7.1 玩家模型

至少模拟 5 类玩家：

| 玩家模型 | 行为规则 |
|---|---|
| 保守玩家 | 优先低风险路线；任一资源低于预警线就找补给或休整 |
| 冒险玩家 | 优先未探索地点；可接受中高风险；低于危急线才补救 |
| 贪心玩家 | 优先器物、隐藏路线、线索；愿意用资源换收益 |
| 归家玩家 | 资源低于预警线后优先回撤到安全点 |
| 随机玩家 | 在可见选项中随机选择，用于发现规则漏洞 |

### 7.2 每局记录字段

| 字段 | 说明 |
|---|---|
| runId | 第几局 |
| playerModel | 玩家模型 |
| seed | 随机种子 |
| steps | 总移动和事件次数 |
| endingType | 成功、失败、特殊结局、返程 |
| failureReason | 断轴、饥荒、崩解、困荒、贪心、幻路、无路等 |
| resourcesFinal | 结束时三资源 |
| minResources | 本局三资源最低值 |
| deathSavesTriggered | 濒死触发次数 |
| badLuckPeak | badLuckMeter 最高值 |
| consecutiveBadEventsMax | 最大连续坏事件次数 |
| noPositiveStepsMax | 最大连续无正向收益步数 |
| forcedRescueCount | 强制补救事件次数 |
| routeRiskDistribution | 低中高路线选择比例 |
| eventPoolDistribution | 各事件池出现比例 |
| softlockFlag | 是否出现死局 |

### 7.3 1000 局测试建议阈值

第一版可以用以下阈值判断是否需要调参：

| 指标 | 目标区间 | 异常处理 |
|---|---:|---|
| 总通关率 | 25% 到 45% | 低于 20% 降低高风险损耗；高于 55% 提高路线压力 |
| 保守玩家通关率 | 35% 到 60% | 太低说明补给不足或路线过硬 |
| 冒险玩家通关率 | 20% 到 45% | 太高说明高风险收益过强 |
| 随机玩家通关率 | 5% 到 20% | 太高说明选择压力不够；太低正常 |
| 平均存活步数 | 14 到 26 | 过短说明早期压力太大 |
| 非自愿死局率 | 低于 1% | 超过即优先修死局防护 |
| 连续坏事件 3 次以上 | 低于 5% | 超过则增强 badLuckMeter |
| 濒死触发后仍立即失败 | 低于 20% | 超过说明补救力度不足 |
| 三资源失败占比 | 单项不超过 50% | 某资源过高说明该资源压力失衡 |

### 7.4 调参优先级

出现问题时，按以下顺序调整：

1. 先修死局和无选择问题。
2. 再调路线基础消耗。
3. 再调事件池权重。
4. 再调 badLuckMeter 阈值。
5. 最后再改三资源初始值。

不建议一开始频繁改初始值，因为初始值会掩盖路线和事件池本身的问题。

## 8. 给技术线程的实现边界

本方案只定义规则，不要求技术线程一次实现全部复杂表现。Web Demo 第一版最低实现：

1. 三资源 0 到 100。
2. 三档路线消耗。
3. 六类事件池。
4. badLuckMeter 隐藏值。
5. 三种濒死状态。
6. 失败原因记录。
7. 1000 局模拟输出汇总表。

可以暂缓：

- 复杂器物联动。
- 完整队伍成员系统。
- 精细语言系统数值。
- 多层地图生成器。
- 正式版结局分支。
