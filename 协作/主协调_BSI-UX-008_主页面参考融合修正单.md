## 一、用户修正意见

主页面界面其实还是参考“西部横向旅行游戏”和《Out There》。

这里的重点不是再改横竖屏，而是明确主页面的气质来源：

- 西部横向旅行游戏：负责第一眼的“车队正在路上”、远景、昼夜、云、假动态横向行进、停靠和路遇体感。
- Out There：负责路线选择、资源压力、风险比较、下一站判断、补给预期和战略决策感。

当前 BSI-UX-007 已经把页面改成横屏游戏框，但还容易停留在“横屏 UI 壳”。BSI-UX-008 要把主页面进一步收口成“路上画面 + 路线决策”的融合，而不是普通横屏信息面板。

## 二、主页面第一眼目标

玩家打开第一屏时，应该先感到：

```text
我带着车队正在神秘九州的路上。
我有三项资源压力。
我马上要在下一站、补给和事件之间做判断。
```

不应该先感到：

```text
我打开了一个网页仪表盘。
我在看几个信息面板。
我需要先理解按钮和抽屉才能进入游戏。
```

## 三、两套参考的分工

### 3.1 西部横向旅行游戏负责主画面

可借鉴的抽象点：

- 横向路途舞台是主视觉，不是背景插图。
- 车队/辇车位于前景，世界在后方缓慢展开。
- 云、日出日落、风沙/雾气、远山和路面分层移动。
- 玩家每走一段路，画面应反馈“从左到右推进”。
- 半途路遇时，车队停在路中，画面告诉玩家“还没到下一站”。

禁止复制的点：

- 不复制任何具体西部游戏的 UI、镜头、马车造型、配色、字体、音效和事件表达。
- 不把《不思异：九州》做成西部片气质。
- 不引入碰撞、跳跃、操作车队移动、横版关卡等实时玩法。

### 3.2 Out There 负责路线决策

可借鉴的抽象点：

- 每次出发前都要看目的地、路线消耗、风险、可观察资源。
- 三资源常驻，玩家能判断哪条路会让自己陷入危机。
- 地点/路线像战略节点，选择路线后进入旅途反馈。
- 发现新地点、锁定路线、语言/线索解锁路线，形成探索感。

禁止复制的点：

- 不复制 Out There 的星图 UI、图标、太空题材、文本结构、命名、资源图形、商店表达。
- 不在宣传中写“山海经版 Out There”。
- 不把路线决策做成纯太空星图，而应做成“九州图 / 山海图 / 舆图”。

## 四、主页面建议结构

主页面默认是“旅途舞台模式”，不是“地点信息页”。

```text
┌──────────────────────────────────────────────┐
│ 轻 HUD：地点 / 日数 / 车轴 / 粮草 / 神志 / 距裂隙 / 九州图 │
├──────────────────────────────────────────────┤
│                                              │
│      横向山海旅途舞台：天空 / 远山 / 道路 / 车队 / 异象      │
│                                              │
│   小型路标：起点 —— 当前路线/路况 —— 下一站                 │
├───────────────────────────────┬──────────────┤
│ 当前文本：地点志摘要 / 路遇 / 事件正文             │ 行动：
│ 最近反馈：补给、资源变化、日志一句话              │ 补给 / 选择 / 九州图
└───────────────────────────────┴──────────────┘
```

关键调整：

- 舞台视觉应更接近“西部横向旅行”的路上感，少一点卡片边框和网页面板感。
- 下方文本区像游戏行动栏，不像后台表单。
- “九州图”按钮是重要入口，进入 Out There 式战略选择。
- 行囊、日志、设置仍是辅助抽屉，不抢主页面。

## 五、九州图模式建议结构

九州图模式负责 Out There 式决策，但视觉语言必须是九州舆图。

```text
┌──────────────────────────────────────────────┐
│ 轻 HUD：当前位置 / 三资源 / 距裂隙 / 返回旅途        │
├──────────────────────────────────────────────┤
│ 九州图：当前节点、可达下一站、锁定道路、未知远方        │
├───────────────────────────────┬──────────────┤
│ 路线卡：去哪 / 消耗 / 风险 / 半途路遇 / 可观察补给      │ 确认前进
└───────────────────────────────┴──────────────┘
```

路线卡必须承担决策信息，地图承担探索感。不要反过来把地图做得很复杂、路线信息很弱。

## 八、主协调验收标准

- 第一眼有“车队在路上”的感觉，而不是网页 Dashboard。
- 主页面的舞台、车队、远景、路面、昼夜/云气是最强视觉。
- 玩家不用打开任何抽屉，也能知道当前位置、三资源、距裂隙、当前事件和下一步。
- 点击九州图后，路线选择有 Out There 式的消耗、风险、目的地补给和锁定条件比较。
- 事件选择、补给、路线选择仍完整可用。
- 行囊、日志、设置不再抢主页面。
- 不出现“西部游戏换皮”或“Out There 换皮”的视觉风险。
