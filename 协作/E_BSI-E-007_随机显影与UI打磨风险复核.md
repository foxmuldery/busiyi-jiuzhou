# BSI-E-007 随机显影与 UI 打磨风险复核

> 子线程：E 商业风险、素材授权与原创边界  
> 项目：《不思异：九州》Web Demo  
> 日期：2026-06-15  
> 任务：BSI-E-007  
> 范围：复核半随机显影、小横屏路线卡、地图节点、音效钩子、动态云雾的商业表达风险  
> 边界：只写产品侧风险复核文档，不修改代码，不替代律师意见

## 0. 总判断

本轮“固定保底 + 半随机显影”可以继续推进。风险不在“随机抽取路线”本身，也不在“小横屏路线卡”本身，而在这些机制被做成可识别的竞品表达：

```text
Out There 风险：星图、星球节点、航线、扫描、jump、全图网状探索。
西部横屏远行风险：西部马车、口琴、枪、酒馆木牌、同款地平线旅途构图。
Darkest Dungeon 风险：哥特硬阴影、英雄队列、职业头像、火把/压力 UI、旁白腔。
FAR 风险：大型载具剖面、锅炉/引擎/拉杆、实时驾驶和载具维修解谜。
```

安全方向仍是 E-006 已定的项目表达：

```text
羊皮纸山海舆图
+ 古道车队
+ 车轴 / 粮草 / 神志
+ 雾散、城显、路显
+ 路引签 / 驿牒式路线卡
+ 纸、墨、朱印、木轴、风雾的克制音效
```

结论：本轮不需要因机制相似而停工，但 C/D/F 线程必须把“第一眼不是星图、不是西部远行换皮、不是 DD 横向队列、不是 FAR 载具操作”列为合并前硬检查。

## 1. 半随机显影是否更像 Out There 星图探索

### 1.1 风险判断

半随机显影会增加 Out There 联想的地方在于：

| 风险点 | 为什么会靠近 | 当前可控做法 |
|---|---|---|
| 每到一处显出下一批节点 | 和星图探索的“到达后发现新星域”在抽象机制上相似 | 表达成“旧舆图雾散、地名被写出、路引签出现” |
| 路线池/地点池半随机 | 容易被做成“扫描发现星球”或“星域抽取” | 用保底路线 + 少量雾中远影，不做科幻扫描和大爆光 |
| 资源危险时提高补救路线概率 | 和生存探索类游戏的动态救援结构相似 | 这是通用平衡机制；UI 只写车轴、粮草、神志的路引判断 |
| 地图节点逐步增多 | 如果全图形成圆点连线网，会像古风星图 | 节点必须是地名签、关防印、祠印、泽痕，不是圆形星球 |

产品侧判断：抽象机制风险低到可控；截图级表达风险高。只要画面、命名、动效、路线卡都回到“山海舆图”，半随机不会自动构成高风险。

### 1.2 必须避免的做法

| 禁止项 | 处理 |
|---|---|
| 把半随机显影称为 `scan / radar / galaxy / planet / jump` | 改成 `雾散 / 显影 / 路引 / 远影 / 可达 / 锁定` |
| 抵达后用扫描圈、雷达波、发光网格点亮节点 | 改成纸雾退开、墨晕扩散、朱印落下、笔锋描路 |
| 节点使用圆形发光点、星球阴影、环形外圈 | 改成地名签、朱砂印、残卷角、关防印、墨点城影 |
| 路线使用发光弧线、跳跃线、能量线 | 改成朱砂线、淡墨车辙、断墨、裂纸、封条 |
| 一次打开大半张路线网 | 每次只打开当前一跳、必要锁定路线和 1-3 个雾中远影 |
| 把随机表现做成手游抽卡、稀有度、大爆光 | 改成几张路引签淡入，像从残卷旁被抽出 |
| 让玩家刷新页面后随机结果变化 | 固定种子或写入 state；同一局同一地点结果稳定 |

### 1.3 安全实现口径

半随机显影建议对外和 UI 内都按下面口径理解：

```text
车队抵达一处，旧舆图周边的雾才退开。
必经的保底路一定被描出。
若干支路和远影像残卷边角一样显露，但不会摊开全图。
资源危急时，舆图更容易显出能补救车轴、粮草或神志的路引。
```

这套表达与“星图探索”拉开距离的关键不是算法，而是：

1. 保底路线像古道，不像航线。
2. 随机支线像残卷路引，不像新星球。
3. 雾中远影只给方向和地貌，不给完整星系网。
4. 显影反馈是纸墨和朱印，不是扫描和雷达。

## 2. 小横屏路线卡和地图节点如何避免竞品 UI 影子

### 2.1 路线卡风险

小横屏会迫使信息压缩，最容易把路线卡做成竞品式资源面板或太空生存卡。路线卡的安全目标是“路引签 / 驿牒 / 残卷批注”，不是“星球详情卡”。

| 模块 | 红线 | 安全做法 |
|---|---|---|
| 卡片外观 | 深色半透明科技面板、玻璃 HUD、同宽资源仪表 | 旧纸签、残卷边、朱砂标题、淡墨分隔 |
| 标题 | 只写目的地，像星球详情 | 写路线名 + 起点到终点，例如“循旧王道西行 / 中原驿 -> 故王道” |
| 资源 | 图标和颜色接近燃料/氧气/船体 | 固定车轴、粮草、神志顺序；图标自研 |
| 风险 | 用科幻危险等级、稀有度、星级 | 写低/中/高风险，配路况词：旧道、泽气、断墨、红封 |
| 补给预览 | 像商店收益卡或战利品掉落 | 写“可补 / 已取 / 本次已补 / 远看有粮”这类状态 |
| 锁定卡 | 做成现代禁用按钮或设备门槛 | 用红封、断墨、缺路引、未正名等项目语汇 |
| 横滑 | 一排相同圆角面板，像移动端商店卡 | 卡片宽度可一致，但边缘、标签、按钮状态要像路引签 |

### 2.2 地图节点风险

地图节点必须承载“地点”而不是“星球”。小横屏下副文本会被隐藏，所以节点形状和状态差异更重要。

| 节点状态 | 禁止表达 | 推荐表达 |
|---|---|---|
| 当前 | 玩家飞船定位环、雷达环、发光十字准星 | 朱砂大印、车队小旗、地名签压在纸面上 |
| 已见 | 小圆点 + 发光边 | 淡墨地名签、旧印、已录入地点志的小痕 |
| 可达 | 明亮星球、发光外圈 | 朱砂描边、金墨边、路引线连到当前点 |
| 雾中 | 问号星球、灰色未知星域 | 雾中城影、残名、方向词、山影/泽痕 |
| 锁定 | 科技锁、能量门、设备不足图标 | 红封、断墨、裂纸、压暗地名 |

### 2.3 小横屏合并前截图自检

C/D 线程合并前至少截 `844x390` 和 `932x430` 两张九州图，逐张问：

1. 遮掉标题后，第一眼是否是羊皮纸山海舆图，而不是古风星图？
2. 路线卡是否像路引签，而不是星球详情面板或商店卡？
3. 可达、雾中、锁定三种节点是否不用副文也能区分？
4. 地图是否只显示当前周边，不提前摊开完整路线网？
5. 补给预览是否区分“可补、已取、本次已补”，不会诱导玩家反复期待同一补给？

任一项答不清楚，先标黄；出现星球、扫描、航线、科幻 HUD，直接标红。

## 3. 音效钩子和动态云雾如何避开西部远行、Darkest Dungeon、FAR

### 3.1 音效钩子风险

本轮音效钩子包括地图打开、雾散显影、抵达地点、路线选择、补给完成。风险不是“有声音”，而是声源和节奏指向具体竞品或题材。

| 触发 | 禁止方向 | 安全方向 |
|---|---|---|
| 地图打开 | 科幻全息展开、雷达启动、现代 app 弹窗 | 旧纸展开、木轴轻响、纸边摩擦 |
| 雾散显影 | 星图扫描、魔法爆闪、稀有抽卡音 | 纸雾退开、墨水晕散、低风穿纸 |
| 城市显现 | 胜利号角、大锣、任务完成铃 | 朱砂印轻落、墨点晕开、短促木印 |
| 路线显现 | 激光连线、能量流、地图雷达 | 干笔划纸、朱砂线被描出 |
| 路线选择 | 现代 UI click、飞船跳跃确认 | 木签落图、笔点定路 |
| 锁定提示 | 错误 buzzer、警报、恐怖尖刺 | 红封压住、断墨轻裂、短木扣 |
| 补给完成 | 商店购买音、战利品掉落 | 粮袋轻落、木箱轻碰、车轴上钉 |

F 线程新增或生成音效时，提示词不得点名 Out There、FAR、Darkest Dungeon、西部片、具体游戏、具体音乐人、具体影视。内部文件名可以保留项目资产 ID，玩家界面不要出现素材来源名和授权状态。

### 3.2 动态云雾风险

动态云雾可用于横屏旅途和地图显影，但必须避免三类“具体表达”：

| 风险对象 | 不能出现 | 安全替代 |
|---|---|---|
| 西部横屏远行 | 西部地平线、口琴感孤旅、牛仔剪影、西部马车、酒馆木牌、同款黄昏构图 | 山海云气、泽雾、旧道尘线、废关风沙、赤水水纹 |
| Darkest Dungeon | 哥特硬阴影、红黑怪物剪影、英雄队列、火把光压迫 | 低神志污染层、纸面错字、裂纹、淡墨遮蔽 |
| FAR | 大型载具剖面、烟囱、锅炉、引擎内部、拉杆、实时操作机关 | 车队只是前景状态剪影；动态服务路线和氛围，不服务载具驾驶 |

动态素材生成和入库继续沿用 F-007 的硬边绿幕原则：

```text
纯绿幕或纯色高对比背景；主体硬边；边缘清楚；
无虚化；无运动模糊；无阴影；无投影；无光晕；
无现代物品；无文字；无 logo；无水印。
```

补充红线：云雾不能做成“大片西部旅行镜头”的核心卖点。它只是舆图、车队和路线压力的气氛层，不能反客为主。

## 4. 给 C 线程的红线清单

| 类别 | 红线 | 合并前处理 |
|---|---|---|
| 半随机 | 不能因刷新页面改变同一局同一地点的抽取结果 | 使用固定种子或写入 state |
| 保底 | 不能让随机池抽空可走路线 | 保底路线始终先于可抽支线 |
| 显影范围 | 不能一次显示全图节点和路线网 | 只显示当前、已见、可达、锁定、雾中 |
| 命名 | 公开 UI 和可见文案不能出现 `scan / radar / jump / galaxy / planet` | 改为雾散、显影、路引、远影、可达、锁定 |
| 动效 | 不能做雷达圈、扫描线、科幻发光航线、抽卡爆光 | 做纸雾、墨晕、笔锋、朱印 |
| 路线卡 | 不能把全局路线列成卡组，也不能让锁定卡静默无反馈 | 只列当前地点路线；锁定显示短条件和替代路径 |
| 补给 | 不能让已取补给、同次抵达已补给继续像可领取 | 明确 `可补 / 已取 / 本次已补 / 不可补` |
| 音频 | 缺音频不能报错或阻塞玩法 | 音效触发必须可空跑，失败只给短状态 |
| 验收 | 不能只通过语法检查就说风险通过 | 至少提供小横屏截图或说明已按 E 红线自检 |

## 6. 给 F 线程的红线清单

| 类别 | 红线 | 安全方向 |
|---|---|---|
| 竞品引用 | 音效/音乐提示词不能点名 Out There、FAR、Darkest Dungeon 或具体影视/游戏 | 用纸、墨、朱印、木签、旧风、车轴、泽雾描述 |

| 科幻声景 | 不要雷达、扫描、全息地图、飞船跳跃、电子 beep | 纸卷、墨晕、笔锋、朱印、木扣 |
| DD 声景 | 不要哥特战斗鼓点、火把压迫、旁白式失败音、恐怖尖刺 | 低频污染、错相轻层、含混远声，保持克制 |

| 授权 | AI/第三方/临时音频不能标正式可用 | 先标 `Demo 临时` 或 `后置复核`，补资产 ID、路径、工具、条款状态 |

## 7. 主线程合并前最短验收口令

```text
1. 半随机显影是不是稳定且有保底？有没有刷一下就变、抽空可走路？
2. 地图第一眼是不是羊皮纸山海舆图？有没有星球、航线、扫描、科幻 HUD？
3. 路线卡是不是路引签？有没有像星球详情卡、商店卡或太空资源面板？
4. 小横屏 844x390 / 932x430 下，路线名、去向、三资源消耗、风险和锁定短条件是否仍可读？
5. 音效是不是纸墨木印风？有没有马蹄口琴、科幻扫描、哥特战斗、锅炉引擎？
6. 动态云雾是不是山海纸雾/泽雾？有没有西部地平线、DD 硬阴影、FAR 载具剖面？
7. 新增素材有没有资产 ID、路径、来源、工具/提示词、授权状态和当前状态？
```

任一项答不清楚，先标黄；出现竞品截图级表达、来源不明素材、点名竞品生成、公开宣传绑定竞品名，直接标红并暂停合并。
