# BSI-UX-013 主线程验收清单

- 验收对象：小横屏 UI、半随机显影、地图音效钩子、补给预览
- 验收人：主协调线程
- 状态：待 C 线程代码交付后执行

## 1. 基础检查

```bash
node --check /Users/yuanzhe/Documents/game/GitHub资产区/03_WebDemo/prototype/app.js
node --check /Users/yuanzhe/Documents/game/GitHub资产区/03_WebDemo/prototype/data.js
```

必须通过：

2. `window.BSI_PROTOTYPE.validate()` 或等价数据验证 `errors.length === 0`。
3. `locations / routes / events / revealPlan / routePools / fogLabels / supplies` 引用一致。

## 2. 半随机显影验收

必须满足：

1. 同一局同一地点的抽取结果稳定，刷新页面不改变。
2. 新开局可出现不同支线或雾中远影，但不破坏主线可达。
3. 每个地点至少保留一条保底可推进路线，终点除外。
4. 当前地点可见路线不能全部锁死。
5. 从 `central_post` 到 `kyushu_rift` 至少存在一条安全路径。
6. 低资源时不得只给高消耗路线。
7. 雾中节点不进入地点志统计，抵达后才录入。

## 3. 小横屏 UI 验收

重点尺寸：

```text
844x390
932x430
```

必须满足：

1. HUD 能读到地点、第几日、车轴、粮草、神志。
2. 旅途 / 九州图切换按钮可点，不遮挡。
3. 路线卡不撑破容器，文字不互相压住。
4. 地图节点不密到无法点击。
5. 补给按钮能看出主收益、代价和已补状态。
6. 行囊九宫格入口可打开、可关闭。
7. 音乐入口仍可用。

## 4. 音效钩子验收

必须满足：

1. 地图打开可触发音频 key，但缺素材时静默失败。
2. 地图显影可触发音频 key，但缺素材时不报错。
3. 抵达地点可触发音频 key，但缺素材时不阻断 `completeRoute()`。
4. 音频设置 UI 不把缺失素材误报为玩法错误。
5. 已有音乐入口不消失。

## 5. 补给预览验收

必须满足：

1. 路线卡目的地补给预览显示主要可补方向。
2. 已用补给不应继续被当成无限可补资源。
3. 当前地点同次抵达已补给后，补给按钮显示“已补”或等价状态。
4. 补给状态不影响路线推进。

## 6. 风险验收

必须避免：

1. 星图、星球节点、发光航线、科幻扫描。
2. 西部马车、牛仔、枪、酒馆等具体符号。
3. Darkest Dungeon 式英雄队列、压力条、火把系统、旁白腔。
4. FAR 式载具剖面、锅炉、驾驶谜题。
5. 现代山海经译文或图鉴设定直接照搬。
