# D_BSI-D-011 小横屏 UI 返工单

> 子线程：D 手机横屏体验和 UI 验收  
> 项目：《不思异：九州》Web Demo  
> 日期：2026-06-15  
> 边界：只写返工单，不修改 `index.html` / `styles.css` / `app.js` / `data.js`

## 0. 验收结论

844x390 / 932x430 都能打开、能切旅途和地图、控制台没有报错。真正的 P0 不是页面空白，而是主操作层级被压坏：

1. 地图路线卡在小横屏下变成深底深字，玩家看不清“去哪、消耗、风险”，这是最高优先级。
2. 地点页补给区高度不足，第二个补给按钮掉到首屏下方；同时小屏只剩“轴 修轴 / 粮 取粮”，收益和状态不够明确。
3. 地图节点小屏只剩 58x28 的地名块，当前、可达、雾中状态区分偏弱；在路线卡不可读时，会放大迷路感。
4. HUD 可用，但 `距九州裂隙约 5 段路` 在两档尺寸都超出 `distance-board` 实际宽度，应短化。
5. 行囊抽屉首屏能看到 3x3 和关闭按钮，不是本轮 P0；但它不应抢在路线卡、地图节点、补给按钮之前改。

## 1. 844x390 P0 问题和具体修法

| P0 问题 | 当前表现 | 具体修法 |
| --- | --- | --- |
| 路线卡几乎不可读 | 地图页底部路线卡实际是深色底，但文字仍是地图纸面用的深棕色。`844x390` 下每张卡约 `270x86`，文字堆 5-6 行，第一屏看不清消耗和风险。 | 去掉小横屏里 `.map-route-panel .route-card` 的深色覆盖，恢复羊皮纸底；或把该区域文字全部改成浅色。但推荐前者：路线卡应像地图上的路引纸条。卡内只保留 4 行：`路名`、`中原驿→故王道`、`轴-4 粮-4 神0`、`险低 / 路遇：轮声`；长提示移到日志或普通横屏。 |
| 路线卡有纵向裁切风险 | `route-list` 高约 `88px`，路线卡可见内容高度约 `84px`，实际内容高度约 `90px`；底部“目的地可观察/提示”会被压暗或裁掉。 | 小横屏路线卡禁止内部纵向滚动，改为字段压缩。`目的地可观察：偏补车轴 · 车轴 +11` 压成 `补：轴+11`。每张卡固定高度 `78-84px`，内容用 `line-clamp` 或直接不渲染第 5、6 行。 |
| 补给第二项掉出首屏 | 补给区实际约 `164x71`，内容高度约 `107`；第二个按钮底部超出屏幕，只能看到第一项或半项。 | 小横屏下 `.supply-list` 改成能同时露出两项：方案 A：两行紧凑按钮，每个 `30-32px`；方案 B：两列一行，每个 `minmax(74px, 1fr)`。同时把 `.supply-head span` 继续隐藏，标题“补给”保留但高度压到 `16-18px`。 |
| 补给按钮语义被压没 | 当前小屏按钮显示为 `轴 修轴`、`粮 取粮`，隐藏了 `车轴 +8 / 粮草 -2`、`可取/已取`。玩家不能一眼判断收益和是否已拿。 | 小屏不要隐藏全部收益和状态。按钮文案改为 `修轴 +轴8 -粮2`、`取粮 +粮8`；状态芯片压成 `可 / 已 / 本`，放右上角或标题尾部。禁用态保留 `已`，不要只靠透明度。 |
| 地图节点状态弱 | 节点只有地名，最小约 `58x28`；当前、可达、雾中都像小纸签，只靠淡线和透明度区分。 | 当前节点加朱砂点或左侧小印；可达节点加朱砂描边 + 轻亮底；雾中节点改虚线 + 水渍淡底 + `?` 或残影点；锁定节点用断线/封条。小屏节点建议 `min-width: 64-72px`，但只显示地名和状态符号。 |
| 开局地图节点簇太近 | `故王道 / 青丘外邑 / 废关` 三个节点在 844 下纵向挤在一起，点选目标容易混。 | 可在 `data.js` 微调首层坐标：保持 `central_post x=8 y=52`，把 `old_king_road` 拉到约 `x=25 y=34`，`qingqiu_outer_city` 约 `x=25 y=51`，`abandoned_pass` 约 `x=27 y=68`；或者 CSS 小屏下提高节点间的视觉留白。 |
| HUD 距离文案过长 | `distance-board` 客户宽约 `126px`，内容实际约 `230px`，虽然视觉上还能读，但处在隐藏溢出的边缘。 | 844 下把 `距九州裂隙约 5 段路` 改成 `裂隙 5 段`；`西行 1 / 6` 改成 `西行1/6`。保留资源数字和行囊入口，不要再挤进长目标句。 |

## 3. 样式优先级

| 优先级 | 模块 | 要先保证什么 |
| --- | --- | --- |
| P0-1 | 路线卡 | 玩家必须一眼读到去向、三资源消耗、风险/路遇，并且知道整张卡可点。先修颜色冲突，再修字段压缩。 |
| P0-2 | 地图节点 | 当前、可达、雾中、锁定必须不用说明文字也能区分。节点是路线卡的空间确认，不应像同一种按钮。 |
| P0-3 | 补给按钮 | 每个地点 1-2 个补给必须首屏同时可见，按钮自身带收益和状态。不能只显示“轴/粮”这种抽象短词。 |
| P0-4 | HUD | 当前位置、三资源、旅途/地图、行囊入口必须稳定；长距离文案要短化，避免挤压资源和入口。 |
| P1 | 行囊入口/抽屉 | 入口常驻即可。抽屉首屏 3x3 和关闭按钮已成立，后续再微调宽度和语言/困厄区露出。 |

## 4. 给 C 线程的 10 条具体改动建议

1. 在小横屏规则里单独覆盖 `.map-route-panel .route-card`：恢复羊皮纸浅底和深字，或统一改浅字；不要让 `.map-route-panel .route-card` 继承深色按钮底后继续使用 `#2e2418` 文本。
2. 给路线卡增加小屏字段模板：`路名`、`起点→终点`、`轴-4 粮-4 神0`、`险低 / 路遇：轮声`、`补：轴+11`。长 `hint` 不在小横屏路线卡首屏显示。
3. 小横屏路线卡固定高度 `78-84px`，`overflow:hidden`，不要纵向滚动；横向滚动只负责换卡，不负责读一张卡内部被裁掉的内容。
4. 844 下路线卡宽度可从固定 `270px` 调成 `clamp(248px, 31vw, 270px)`，让第一张完整、第二张大半露出、第三张露边提示可横滑；932 仍可自然露出三张。
5. 地图节点补一套手机状态符号：当前 `朱砂点/印`，可达 `朱砂边 + 亮底`，雾中 `虚线 + 淡墨残影`，锁定 `斜封条`。不要依赖 hover。
6. 微调开局节点坐标，拉开 `old_king_road / qingqiu_outer_city / abandoned_pass` 的垂直间距；如果不改数据，至少小屏下给相邻节点加 `min-width:68px` 和更明显描边。
7. 小屏补给列表必须同时露出两项。建议 `.supply-list` 在低高横屏改成两列一行，或两行 `30-32px`；同时把补给区标题压到一行，不占掉按钮高度。
8. 小屏补给按钮不要隐藏收益和状态。把 `formatSupplyDelta` 压成短符号：`+轴8 -粮2`、`+粮8`；状态压成 `可/已/本`，用于区分“可补、已取、本次已补”。
9. HUD 距离文案按低高横屏使用短版：`西行1/6`、`裂隙5段`。`nextStopLabel` 可继续隐藏，但不能让长距离文本造成隐藏溢出。
10. 行囊抽屉只做小修：保持 `行囊/日志/设置` 入口可见，抽屉宽度可降到 `min(320px, 38vw)`，但不要为了抽屉牺牲路线卡和补给区的首屏可读性。

## 5. P0 修复清单

- [ ] 844x390 地图路线卡可读：浅底深字或深底浅字一致，首张卡能读到去向、消耗、风险。
- [ ] 932x430 地图路线卡可读：三张卡露出时文字不再深底深字。
- [ ] 844x390 补给区两项都在首屏可见，不需要先滚动才能看到第二项。
- [ ] 932x430 补给区两项都在首屏可见。
- [ ] 小屏补给按钮保留收益和状态短标，不只显示 `轴 修轴 / 粮 取粮`。
- [ ] 小屏地图节点能区分当前、可达、雾中、锁定；当前和可达不再像同一种纸签。
- [ ] HUD 距离文案短化后不再触发隐藏溢出。
- [ ] 行囊抽屉关闭按钮和 3x3 首屏继续可见，但不作为本轮最高优先。
