# 《不思异：九州》游戏详细介绍

> 版本日期：2026-06-25  
> 项目状态：P0 Web 试玩纵切片开发中  
> 用途：对外协作、内容策划交接、外部 AI 同步、测试者和团队成员快速理解项目  

---

## 1. 一句话介绍

《不思异：九州》是一款以《山海经》气质为灵感的横屏远行生存叙事游戏。玩家作为王朝派出的采风使，率领一支古代车队，从人间边界出发，沿着残缺的九州舆图进入大荒、雷泽、赤水、羽民渡、黑齿市、归墟裂隙等神秘地理，在“车轴、粮草、神志”三项资源压力下选择路线、停靠地点、搜集补给、处理异象事件，采集风土、民谣、异闻与证词，并逐渐发现地图、文字和自己所见的世界都不再可靠。

核心体验可以概括为：

```text
一个凡人车队，用有限的器物、粮草和心智，强行穿过一个越来越不像人间的世界。
```

---

## 2. 类型定位

### 2.1 游戏类型

- 山海经题材远行生存叙事游戏
- 节点地图探索
- 资源管理
- 文字选择事件
- 轻 Roguelike / 多结局重开
- 横屏手机优先的 Web Demo
- 假动态横向旅途舞台

### 2.2 它不是

- 不是传统打怪升级 RPG。
- 不是开放世界跑图游戏。
- 不是横版动作游戏。
- 不是单纯的山海经怪物图鉴。
- 不是靠战斗数值成长驱动的角色养成游戏。

玩家主要做的事情不是“打败怪物”，而是“在不知道前路是否可信的情况下继续往前走”。

---

## 3. 玩家扮演什么

玩家扮演的是王朝派出的采风使，也是一支古代行旅车队的主导者。

表面任务是“采九州之风”：记录各地风土、民谣、祭仪、异闻和灾变，把所见所闻写入采风册。更深一层，采风使也是王朝的眼睛。玩家写下什么、隐去什么、如何上报，会影响王朝如何理解一座城、一个部族或一片异域：可治理、可安抚、可交易、可隐瞒，还是必须镇压、封禁或放弃。

车队携带竹简、舆图、辎车、粮草、祭器、药石、空白采风册和少量随行者，从相对熟悉的人间地理进入越来越陌生的荒外区域。起初，玩家看到的是地点、山川、墟市、渡口和古道；随着旅途深入，玩家会遭遇不合常理的风俗、神祇遗迹、异族语言、不可理解的碑文、会污染感官的地理裂隙，以及越来越像活物的地图。

玩家的核心身份不是英雄，而是奉命记录的人。可怕之处在于：他写下的东西不只是游记，也可能成为一座城的判词。

## 4. 核心资源：车轴、粮草、神志

正式规则名使用：

```text
车轴、粮草、神志
```

界面短名可使用：

```text
车、粮、神
```

### 4.1 车轴

车轴代表车队继续远行的能力，包括车轮、车轴、辇架、绳索、舟板、牲口、桨具和整体通行状态。

它解决的问题是：

```text
我们还能不能走到下一站？
```

设计作用：

- 选择路线会消耗车轴。
- 山隘、泽地、断桥、暴雨、强渡、异兽袭扰会增加车轴消耗。
- 车轴低时，玩家会害怕“走不出去”。
- 车轴触底时进入断轴危机，需要修车、拆器、等待、交易或承担更大代价。

### 4.2 粮草

粮草代表车队还能支撑多久，包括粮食、盐、干肉、草料、药材、薪柴和日常补给。

它解决的问题是：

```text
我们还能不能撑过今天？
```

设计作用：

- 移动、绕路、等待、祭祀、交易、救人都会影响粮草。
- 不同地点提供不同补给，有的偏粮、有的偏修车、有的只能交易。
- 粮草低时，玩家会害怕“人还没到，队伍先散了”。
- 粮草触底时进入饥荒危机，可能导致随从逃散、疾病、神志下降或强迫牺牲。

### 4.3 神志

神志代表车队对世界的认知稳定程度，也代表玩家还能否信任地图、文字和自己的判断。

它解决的问题是：

```text
我们看到的东西还是真的吗？
```

设计作用：

- 见异象、读巫文、夜行、听神谕、接近归墟会损耗神志。
- 神志越低，文本越恐怖，地图越不可靠，选项越暧昧。
- 低神志不只是惩罚，也可能看到隐藏路线、异兽真名和不可见的真相。
- 神志触底不是普通死亡，而是认知崩坏、迷失、异化或特殊结局。

## 5. 核心玩法循环

一局游戏的基本循环是：

```text
抵达地点
-> 阅读地点志
-> 采集风土、民谣、异闻或证词
-> 判断当前车轴 / 粮草 / 神志
-> 搜集一次补给
-> 打开九州图
-> 观察可见路线、消耗、风险和目的地补给倾向
-> 选择下一站
-> 在横向旅途舞台中行进
-> 半途可能遭遇路遇事件
-> 做出选择并承受代价
-> 抵达新地点
-> 迷雾散开，后续路线显影
-> 采风册新增见闻，并可能形成上报、隐瞒、讽谏或谶言
-> 继续深入、抵达结局或陷入危机
```

核心循环可浓缩为：

```text
选路 -> 消耗 -> 遭遇 -> 抉择 -> 采风入册 -> 落账 -> 更深入
```

玩家每一次点击路线，本质上都在回答三个问题：

1. 我愿意花什么资源？
2. 我愿意承担什么未知风险？
3. 我是否还相信地图告诉我的下一站？

玩家每一次记录见闻，本质上也在回答三个问题：

1. 我相信谁的说法：官员、百姓、异族、神祇，还是自己的眼睛？
2. 我要如何写下此地：颂、讽、哀、谶，还是隐去不报？
3. 我写下以后，会救这地方，还是害这地方？

## 6. 地图与行程结构

### 6.1 底层是节点地图

游戏底层不是开放世界，而是节点网络。

每个地点是一个节点，每条路线连接两个节点。路线包含消耗、风险、地貌、半途事件、锁定条件和目的地预览。

示意：

```text
故王道 -> 雷泽浅畔 -> 羽民渡 -> 黑齿市 -> 九州裂隙
        \           \          \
         断碑坡       鸟鼠夹道    赤水
```

### 6.2 表现上是横向旅途

虽然底层是节点地图，但画面表现不是纯点线图，而是横向山海旅途舞台。

玩家会看到一段从左到右缓缓推进的风景：山影、河泽、荒草、墟市、祭所、禁地、云气和日夜变化。车队可以作为前景元素出现，但车队不是动作操作主体，真正的主角是“远方正在靠近的山海世界”。

这套表现方式的目标是：

- 让节点地图不显得单薄。
- 让每次移动都有“在路上”的沉浸感。
- 让风景、天气、远处建筑和异象承担叙事。
- 避免做成真正横版关卡，从而控制技术与美术成本。

### 6.3 大地图与中地图

游戏可以分成两层：

1. 九州战略图  
   玩家选择下一站，查看迷雾、路线、资源消耗、目的地补给倾向。

2. 横向行程图  
   玩家沿路线行进，中途可能停下触发路遇事件，抵达后进入地点页。

这种结构让游戏既有路线策略，又有旅途代入感。

## 7. 地点系统

每个地点不只是一个按钮，而是一个小型叙事单元。

一个地点应包含：

- 地点名
- 地貌类型
- 地点志
- 抵达句
- 风俗或异象说明
- 一次性补给选项
- 地点事件
- 可通往路线
- 低神志版本描述
- 图像或采风记录

### 7.1 地点志

地点志是《山海经》气质的关键。玩家到达一个地方时，应先知道：

- 这里是什么地方？
- 为什么危险？
- 有什么风俗、异兽、禁忌或异象？
- 这里和九州大地图有什么关系？

地点志不应只是背景设定，而应该帮助玩家判断下一步风险。

### 7.2 地点补给

每个地点可以有一次性补给，但资源侧重不同。

例如：

- 驿站：车轴、粮草都能补一点。
- 废关：偏修车，粮草少。
- 雷泽：偏粮草，但可能损神志。
- 断碑坡：偏神志线索，但可能耗粮。
- 黑齿市：可以交易，收益高但代价复杂。

这样玩家不会每站都做同一个选择，而是会根据当前短缺资源决定冒险方向。

## 8. 遭遇与选择

游戏的遭遇包括三类：

1. 地点事件  
   抵达地点后发生，和当地风俗、遗迹、异兽、人物有关。

2. 半途路遇  
   在两地之间行进时触发，通常更短、更突然，用于制造旅途的不确定性。

3. 危机补救  
   当车轴、粮草或神志触底时触发，让玩家有最后补救机会，而不是立刻死局。

每个遭遇应包含：

- 遭遇标题
- 一张对应图像
- 一句诗引或古辞
- 简短场景描述
- 2 到 3 个选择项
- 每个选择的资源变化
- 选择后的结果文本
- 是否影响后续路线、语言、旗标或结局

选择项要让玩家看懂代价，但不必完全透露后果。

示例：

```text
遭遇：文鳐雨

薄云里游过文鳐，鳞光落在车辙旁。粮袋闻到水腥味。

选择一：收鳞作粮
结果：粮草 +6，神志 -2

选择二：覆袋避鳞
结果：粮草 -2，神志 +3

选择三：追鱼入浅泽
结果：车轴 -2，粮草 +9，神志 -4
```

## 9. 神志污染文本

神志系统是本项目最重要的差异化体验之一。

同一个地点或事件，在不同神志区间下会呈现不同文本。

### 9.1 三层文本

| 神志状态 | 文本气质 | 玩家感受 |
|---|---|---|
| 高神志 | 地理志 | 冷静、古雅、可判断 |
| 中神志 | 异闻录 | 征兆增多，世界开始不对劲 |
| 低神志 | 疯人书 | 文字断裂，地图和感官不可信 |

### 9.2 示例

高神志：

```text
前方有泽，水色赤而无波。泽畔多白骨，骨上生细草。随从说此地不可夜渡。
```

中神志：

```text
赤泽无风，却有水声从车底传来。白骨上的草向你低伏，像在听你腹中饥响。
```

低神志：

```text
水不是水。骨不是骨。车轴转了一圈，又转回昨日。草在你牙缝里生长。
```

### 9.3 设计价值

- 数值变化能被玩家真正感受到。
- 恐怖感来自系统，而不是单纯堆怪物。
- 同一内容在不同状态下有重玩价值。
- 玩家会对“低神志是否值得冒险”产生策略判断。

## 10. 采风机制

采风机制是本项目后来确立的重要身份机制。它把“地点志、采风册、事件选择、玩家成就感、王朝政治后果”串成一条线，不只是附属收集系统。

### 10.1 机制定位

玩家不是随便旅行，而是奉命采风。采风使的职责是：

- 记录各地风土。
- 收集民谣、异闻、灾情和传言。
- 观察地方官与百姓说法是否矛盾。
- 判断异族、墟市、祭所和禁地是否可与王朝建立关系。
- 把见闻写成采风册，带回王朝。

采风册既是玩家的旅途档案，也是玩家对世界作出的判断。

### 10.2 采什么

采风内容可以分为六类：

| 类别 | 内容 | 游戏作用 |
|---|---|---|
| 地风 | 地貌、道路、物产、灾变 | 解锁地点志、路线提示、补给倾向 |
| 民风 | 民谣、俗语、传言、怨声 | 揭示真实处境，影响上报判断 |
| 官声 | 官员奏报、安民令、税粮说法 | 和民风形成冲突，制造道德选择 |
| 异闻 | 异兽、神祇、怪病、梦兆 | 推动山海经气质和神志污染 |
| 商路 | 交易规则、禁物、路引、人情债 | 影响后续墟市、补给和路线 |
| 谶兆 | 低神志下看到的诗句、断碑、幻路 | 可能解锁隐藏路线或特殊结局 |

### 10.3 如何采风

采风不是单独开一个复杂小游戏，早期应融入现有流程：

1. 抵达地点后自动收入基础地风。
2. 处理地点事件时，根据选择收入不同异闻、证词或谶兆。
3. 半途路遇可收入短条目，例如民谣、路碑、怪影、商队传言。
4. 低神志时可能采到“谶兆”，内容更危险，也更可能通向隐藏路线。
5. 某些补给行为也可带来采风，例如“向老卒问路”“听渡人歌”“抄断碑”。

最小实现可以先做成：

```text
地点 / 事件 / 路遇 -> 采风条目解锁 -> 写入采风册 -> 结局复盘统计
```

当前 Web 试玩版已经在“地点事件处理后、补给与选路前”接入了第一版轻量采风写法：

```text
处理遭遇 -> 选择颂 / 讽 / 哀 / 谶之一 -> 写入采风册 -> 进入补给与选路
```

当前版本已进一步补入“采什么”的题材层：每个地点有一条专属采风题材，例如地风、民风、官声、异闻、商路、灾情或谶兆。玩家处理地点遭遇后，采风弹窗会先提示本地可采内容，再让玩家决定如何写入王朝册页。

```text
处理遭遇 -> 显示本地采风题材 -> 选择颂 / 讽 / 哀 / 谶 -> 写入采风册
```

为了适配手机横屏，当前采风选项以类别圆标和题材芯片呈现，例如“风 · 地风 · 驿灯与空里程”，长线索只保留一行短预览，完整内容在入册结果和采风册详情中回看。

写入后，玩家可以在“志 -> 采风”的地点卡中看到该地点的题材和写法标签，并在地点详情弹窗里回看本局采到的题材、线索、记录文本和入册日。这让采风不只是一次性选择，而是可复盘的旅途档案。

当前试玩版还会把写法转成一条轻量“王朝案牍”后果，出现在采风册详情、本局复盘、复制复盘和 P0 反馈包里：

- 颂：此地暂作“可安抚”。
- 讽：此地标为“须查访”。
- 哀：此地记为“宜赈济”。
- 谶：此地归入“禁录”。

后续再扩展成：

```text
采风条目 -> 选择上报方式 -> 影响王朝判断 -> 改变地点命运或后续路线
```

### 10.4 采风后的选择

采风机制真正有意思的地方，不是“收集到了”，而是“如何写”。

玩家可以面对四种写法：

| 写法 | 含义 | 可能结果 |
|---|---|---|
| 颂 | 把此地写成可治理、可嘉奖 | 获得王朝信任，但可能掩盖灾情 |
| 讽 | 写出问题，暗中规劝 | 保留余地，可能得罪地方官 |
| 哀 | 写出苦难和求救 | 可能引来救济，也可能引来封禁 |
| 谶 | 写成预兆或禁忌 | 可能触发神秘线索，也可能伤神志 |

当前试玩版的轻量效果是：

- 颂记此地：轻微恢复神志，表现为把此地写成仍可安抚。
- 讽记民声：略耗粮草、恢复神志，表现为多听民声、看见矛盾。
- 哀录灾情：耗粮草、微稳神志，表现为问证苦难。
- 谶写异兆：损神志，表现为把不可解之物写成预兆。

更灰暗的选择还包括：

- 受贿改写。
- 隐瞒异象。
- 夸大灾情。
- 替百姓写下真相。
- 替地方官遮掩。
- 把不可理解的东西写成诗，交给后世误读。

### 10.5 与三资源的关系

采风机制应和车轴、粮草、神志绑定，而不是游离在外。

- 采风可能耗时间，间接耗粮草。
- 抄碑、夜访、入祭所可能损神志。
- 为采风绕路可能耗车轴。
- 成功采到商路、物产、禁忌，可降低后续路线风险。
- 采到民谣或官声，可能解锁特殊选项。
- 采到谶兆，可能打开隐藏路线，但让地图更不可靠。

### 10.6 与神志系统的关系

高神志时，采风条目像地理志：

```text
雷泽多浅水，芦中有废鼓，夜不可击。
```

中神志时，采风条目像异闻：

```text
渡人说鼓声在水底，凡击鼓者，三日后梦见旧雷。
```

低神志时，采风条目可能像谶言：

```text
鼓未响。是你的骨头先响。雷泽记得你来过。
```

同一个地点，不同神志下收入的采风内容可以不同。这会让玩家为了收集“真相”主动承担神志风险。

### 10.7 与内容协作的关系

采风机制也能承接《山野逸事》等旧项目内容。

旧设定不必直接改成完整游戏关卡，可以先转译为采风条目：

- 一段民谣。
- 一个地方传说。
- 一个禁忌。
- 一条官府说法。
- 一个百姓证词。
- 一个低神志谶语。
- 一个地点命运分歧。

这样内容协作者可以发挥自己对典故、情绪和故事因果的判断，不需要直接承担数值、路线和 UI 设计。

### 10.8 成就感与复盘

采风是玩家的长期成就感来源。

每局结束时，复盘不只显示资源和路线，还应显示：

- 采风几条。
- 新收入哪些地点风物。
- 收到哪些民谣或异闻。
- 本局做过几次采风写法判断。
- 哪些条目来自低神志状态。
- 哪些城市因玩家的写法改变了命运。
- 哪些见闻已入册，哪些仍是残页。

它的情绪不是“收集完成 12/20”，而是：

```text
你活着带回了几页不该被看见的九州风声。
```

## 11. 美术方向

### 11.1 总体气质

美术应呈现：

```text
古籍山水 + 荒外地理 + 残卷质感 + 低饱和神秘色 + 局部朱砂异象
```

关键词：

- 山海经
- 古地图
- 羊皮纸
- 水墨设色
- 旧壁画
- 远山与泽国
- 骨、碑、旗、残门、渡口、祭所
- 神秘但不廉价恐怖

### 11.2 画面结构

需要三类核心视觉：

1. 横向路段长图  
   用于旅途舞台和中地图，强调远方、地貌、日夜、云气和移动感。

2. 地点图  
   用于抵达地点、地点志和采风册，强调“这里是什么地方”。

3. 遭遇图  
   用于事件弹窗，建议可用方图或接近方图，突出单个异象、物件、人物或冲突。

### 11.3 重要原则

- 背景长图不应带车队，车队应作为前景单独叠加。
- 事件图不一定要宽屏，方图更适合居中弹窗。
- 临时 AI 图可用于内部 Demo，但正式版本需要授权、统一风格和人工复核。
- 透明素材生成时优先用绿幕或纯色底，不依赖黑白格透明背景。

## 12. UI 与交互方向

目标是更像横屏手机游戏，而不是普通网页。

### 12.1 主界面

主界面应尽量简洁：

- 资源用图标和数字表达。
- 当前目标用短提示表达。
- 文字不要长期占据大面积画面。
- 详情、地点志、遭遇、结果尽量用居中弹窗。
- 选择项跟随弹窗出现，不单独散在页面其他区域。

### 12.2 触屏优先

交互原则：

- 选择项必须一次性显示完整，不依赖滚动条。
- 结果页和说明页可以点击空白继续。
- 有选择项时不能误触关闭。
- 关闭键不是主要交互，轻点继续更适合触屏。
- 低资源、采风入册、路线显影等反馈可用浮动文字和半透明消失效果。

### 12.3 两个核心页面

1. 地点 / 旅途页面  
   展示横向风景、当前地点、地点志、补给、遭遇、结果。

2. 九州地图页面  
   展示羊皮纸地图、当前节点、迷雾、可走路线、消耗和目的地补给倾向。

## 13. 音乐与音效方向

声音不是装饰，而是神秘感和压力感的一部分。

### 13.1 音乐层

应包含：

- 旅途循环音乐
- 地形音乐轮换
- 地点级音乐
- 低神志污染层
- 环境底噪

音乐不宜过满，要留出“荒外空旷”的感觉。

### 13.2 音效层

需要覆盖：

- 点选
- 资源增加
- 资源减少
- 地图显影
- 选路
- 抵达
- 半途路遇
- 危机告急
- 神志污染
- 结局落账

### 13.3 当前状态

当前 Web Demo 已接入可试玩音频切片，但仍属于内部待复核素材，不代表最终授权和最终音色。后续必须通过音频听感验收工作台逐条判断保留、待改或弃用。

当前游戏内设置面板已经加入听感进度条：测试者试听音乐后可直接标记“保留 / 待改 / 弃用”，面板会显示 `听感进度 x/6` 和未评音乐，复制听感模板时也会带入该进度，便于主线程判断音频主观验收是否补齐。

## 14. 内容与世界观来源

项目气质取自古典志怪、《山海经》式神秘地理和中国古代异域想象，但不应机械复刻原典条目。

正确做法是：

- 提取原典意象。
- 转译成玩家能做选择的地点、事件、路线和代价。
- 保留古籍感，但让玩家看得懂。
- 用地点志和遭遇图建立成就感。
- 用采风条目把旧设定转成玩家亲手收入册中的见闻。

例如，一个古典典故不应只变成设定说明，而应拆成：

- 一个地点
- 一种风俗
- 一个风险
- 一个选择
- 一个资源代价
- 一个低神志版本
- 一个可能影响后续的线索
- 一个可收入采风册的风物、民谣、证词或谶兆

## 16. 第一章体验目标

第一张地图不追求内容量巨大，而要验证玩家是否能在 5 到 15 分钟内理解并产生兴趣。

### 16.1 5 分钟目标

玩家应该能明白：

- 我在哪里？
- 我还要往哪里走？
- 车轴、粮草、神志分别代表什么？
- 选路会消耗资源。
- 每个地点能补给，但补给不一样。
- 遭遇选择会改变资源和旅途状态。
- 我是采风使，见闻入册是旅途目标之一。

### 16.2 15 分钟目标

玩家应该经历：

- 至少一次路线选择压力。
- 至少一次半途路遇。
- 至少一次低资源告急。
- 至少一次神志文本变化。
- 至少一次想要查看采风册或复盘的时刻。
- 至少一次“我该如何记录这件事”的采风判断。
- 一个明确的阶段性结局或失败记忆点。

## 19. 后续开发重点

### 19.1 玩法

- 继续打磨第一章路线压力。
- 保证随机显影稳定，不让玩家因为运气直接死局。
- 增强每个地点的资源差异。
- 增强神志低时的文本和地图不可靠感。
- 把采风写法从当前“颂 / 讽 / 哀 / 谶”轻选择扩展为“采什么、是否上报、造成何种地方后果”的中期系统。
- 增强失败游记和结局记忆点。

### 19.2 内容

- 给每个地点补足地点志、风俗、异象和低神志版本。
- 给每个地点补足可采风条目：地风、民风、官声、异闻、商路或谶兆。
- 给每个遭遇配对应图像。
- 把《山野逸事》等已有志怪设定转译为地点、事件、禁忌和选择。
- 建立典故来源与游戏转译的资料台账。
- 建立“旧设定 -> 采风条目 -> 游戏事件/地点”的转译台账。

### 19.3 美术

- 统一横向路段长图风格。
- 补齐地点图和遭遇方图。
- 将车队、云、雾、日月、异象作为可叠加动态层。
- 对 AI 临时素材进行授权和风格复核。

### 19.4 音频

- 完成核心音乐候选听感复核。
- 补齐地图、选路、抵达、资源、危机、神志污染音效。
- 区分内部临时音频和正式可用音频。

### 19.5 测试

- 回收至少 1 份真人 5 分钟试玩反馈。
- 回收至少 1 份主观听音反馈。
- 观察玩家是否知道下一步。
- 观察玩家是否理解三资源。
- 观察玩家是否愿意再走一站。

## 20. 给协作者的工作方式

如果你是内容、编剧、美术、音乐或外部 AI 协作者，请不要直接改动核心方向。优先按以下方式提交材料：

1. 先说明你负责的是地点、事件、视觉、音频、UI 还是测试。
2. 每个新增设定都要说清它会落到哪个游戏模块。
3. 内容策划不要只交世界观描述，要尽量说明玩家会在这里做什么选择。
4. 如果内容暂时还不能变成完整事件，先转成采风条目：民谣、官声、异闻、禁忌、证词或谶兆。
5. 美术提示语要说明图像用途：横向路段、地点图、遭遇图、UI 图标、采风册页面或动态层。
6. 音频需求要说明用途：旅途底乐、地点音乐、神志污染层、资源短音、地图显影、采风入册或危机告急。
7. 所有材料先进入主线程审核，再进入具体实现。
