## 需要 C 线程实现的内容

最小接入建议：

1. `locations[id].detail` 直接替换为本文件的地点志。
2. `routeEvents[id].texts.clear / uneasy` 直接替换为本文件半途路遇两档文本。
3. 若已有抵达 toast，可新增或复用 `locations[id].arrivalText`。
4. 若已有补给结果浮层，可新增或复用 `locations[id].supplyDiscoveryText`。
5. 若没有成就系统，成就感短句先进入日志，不新增复杂成就逻辑。

建议字段形态：

```js
locations.central_post.arrivalText = "驿灯还亮，你们在最后一段官路前停下。";
locations.central_post.supplyDiscoveryText = "驿仓角落还剩一袋干粟，铜钉压在粮袋底。";
```

## 方向稿

以下内容不强制首版接入，但可作为后续 B/C/D 继续推进依据：

- 抵达句应显示在地点名下方，最多一行；小横屏可自动折成两行。
- 补给发现句应跟资源变化同屏出现，让资源收益有“发现感”。
- 成就感短句不要做大弹窗，适合轻浮层、日志插入、或地图显影提示。
- 低神志文本要保留可玩信息，不把风险方向藏掉。

## 风险 / 待确认
