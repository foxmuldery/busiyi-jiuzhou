## 一、用户新增意见

行囊不应继续用纯文字列表展示，应改成九宫格或类似格子背包。

原因：

- 文字列表像调试面板，不像游戏 UI。
- 九宫格能让玩家感觉“我真的带着器物在远行”。
- 后续可以承载器物、线索、符节、地图残片、异族交易物、祭品等。
- 也方便展示空格，让玩家感到还有收集空间。

## 二、第一版实现建议

本轮只做 UI 和状态映射，不扩大道具系统。

### 格子结构

- 默认 3 x 3 九宫格。
- 每格有：
  - 器物名
  - 类型短标，如“器物 / 线索 / 语言 / 空”
  - 简短状态，如“随身 / 已得 / 未得”
- 空格显示为暗格，不要完全消失。

### 当前数据映射

当前 `state.items` 可直接放入前几个格子：

- 辟邪铃
- 旧车具

当前 `flags` 可映射为线索格：

- `old_order` -> 旧令墨拓
- `heard_ground_thunder` -> 地雷听路
- `followed_bones` -> 赤水骨路
- `rift_name` -> 裂隙名
- `accepted_mirage` -> 幻路标记

当前 `languages` 可映射为语言格：

- `feather > 0` -> 白羽语片
- `wuxian > 0` -> 巫咸古辞

如果超过 9 个，第一版只显示前 9 个，并在标题或末格显示“另有若干”。

## 三、界面要求

- 仍放在右侧“行囊”面板里。
- 保留语言、困厄、音乐、动态层等信息，但行囊本体改成九宫格。
- 格子尺寸稳定，不能因为文字长短撑开布局。
- 移动端应保持 3 列或改成 3 列小格，不要横向溢出。
- 不要做拖拽，不要做复杂背包管理，本轮只是展示。
