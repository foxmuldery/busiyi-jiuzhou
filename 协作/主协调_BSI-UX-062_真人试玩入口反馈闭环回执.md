# 主协调 BSI-UX-062 真人试玩入口反馈闭环回执

时间：2026-06-24

范围：Web Demo 试玩入口页、反馈回收模板、QA 守门、浏览器实测。

## 1. 本轮目标

把试玩入口从“可以打开原型的入口”推进为“可以发给真人测试者，并能回收有效反馈的入口”。

## 2. 本轮改动

- `试玩入口.html` 增加三项真人试玩观察目标：
  - 看懂下一步
  - 感到资源紧
  - 愿意再走一站
- 新增“复制测试链接”按钮，默认生成 `index.html?fresh=1` 干净新局地址。
- “复制试玩记录模板”调整为“复制反馈模板”，模板改为更便于判断问题的 8 个问题：
  - 下一步是否清楚
  - 资源变化是否看得懂
  - 哪一刻最像游戏
  - 哪一刻还像网页
  - 第一次卡住在哪里
  - 最想继续看的地点/事件
  - 最不清楚的按钮或文字
  - 音乐/音效是否打扰或加分
- 增加剪贴板失败兜底：如果浏览器不允许复制，入口页会展开可手动复制的链接或反馈模板文本。
- `README.md` 同步说明真人试玩入口与复制失败兜底。
- `qa-check.js` 增加试玩入口反馈闭环守门，防止后续误删三项观察目标、复制链接、反馈模板和手动复制兜底。

## 3. 验证结果

命令验证：

- `node GitHub资产区/03_WebDemo/prototype/qa-check.js` 通过。
- `node GitHub资产区/03_WebDemo/prototype/journey-smoke-check.js` 通过。
- `node GitHub资产区/03_WebDemo/prototype/asset-readiness-check.js` 通过，生成图全量 `59/59 ready`。

浏览器验证：

- 试玩入口页标题：`不思异：九州 试玩入口`。
- 三项观察目标均可见：`看懂下一步 / 感到资源紧 / 愿意再走一站`。
- 主入口仍指向：`./index.html?fresh=1`。
- “复制测试链接”和“复制反馈模板”按钮均存在。
- 当前浏览器未放行剪贴板写入时，页面会展开手动复制区：
  - 测试链接兜底显示 `index.html?fresh=1`。
  - 反馈模板兜底显示完整 5 分钟试玩记录模板。
- 控制台 error/warn：0。

## 4. 结论

当前版本已经更接近可发给真人试玩的内部测试版。下一步不应继续堆新功能，优先做一次真人或模拟真人路径验收：从入口进入、完成一局、打开复盘、回填反馈模板。

## 5. 下一步建议

1. 做 `BSI-UX-063`：真人试玩路径复核。重点验证新玩家是否知道下一步点哪里，以及地图/旅途切换是否迷路。
2. 做 `BSI-AUDIO-012`：音乐/音效主观听感复核。重点收集“加分/打扰/无感”三类反馈。
3. 做 `BSI-VIS-063`：高频首屏审美复核。只看前 5 分钟一定会看到的地点、遭遇、路线、资源状态，不平均检查 59 张图。
