# D_BSI-D-002 视频动态资产使用边界

> 当前状态：等待 D 线程与 C 线程提交开工申请。  
> 本文件暂作交付占位，主线程未验收前不得视为最终方案。

## 任务目标

明确视频动态资产在《不思异：九州》Web Demo 中的使用范围，让视频提升华丽感和沉浸感，同时不破坏核心玩法、阅读效率、性能和授权安全。

## 待 D 线程提交

- 视频适合出现的 UI 层级和场景。
- 背景短循环、路线转场、神志污染叠层的视觉边界。
- 视频如何不遮挡资源、路线、事件文本和按钮。
- 桌面端与移动端的使用差异。
- 第一版视频强度建议。

## 待 C 线程提交

- 视频资产的最小 Web 实现方式。
- 格式、体积、poster、懒加载、移动端降级策略。
- `video`、CSS 假动态和静态图回退之间的取舍。
- 视频默认静音、音频由 F 线程控制的技术边界。
- 加载失败时不影响核心玩法的方案。

## 主线程当前默认判断

第一版建议采用克制方案：只用短循环背景、少量路线转场和神志异象叠层，不做大量事件短片。

理由：

- 能显著提高第一屏气氛。
- 不会把内容生产压力从文本转移到视频。
- Web 体积和移动端性能更容易控制。
- 后续可根据素材质量逐步扩大使用范围。
