# D_BSI-D-002 视频动态资产使用边界

> 子线程：D 视觉体验与交互氛围设计师  
> 协作对象：C 技术原型与工具链工程师  
> 阶段：Web Demo 测试原型  
> 边界：不生成最终视频，不导入大视频素材，不复制竞品镜头，不让视频取代核心玩法

## 1. 任务目标

视频动态资产只能增强“横向山海旅途舞台”的氛围，不能把游戏变成看短片。第一版仍然以资源判断、路线选择、事件文本、行动按钮和日志复盘为核心。

视频出现的正确位置：

```text
资源是否危险 -> 看旅途舞台变化 -> 选路线/行动 -> 读事件 -> 看日志
```

视频不承担：

- 不承担地图点击逻辑。
- 不承担事件正文表达。
- 不承担资源变化说明。
- 不承担车队主行动层。
- 不替代路线、按钮、日志。

## 2. 视频适合出现的 UI 层级和场景

| UI 层级 | 是否适合视频 | 场景 | 使用边界 |
|---|---|---|---|
| 天空层 | 适合 | 云气漂移、天色微变、日月星象轻动 | 最适合短循环，低对比、低速度 |
| 后景层 | 谨慎适合 | 远山雾气、水泽波光、异兽远影一闪 | 动作必须慢，不能抢过节点和文本 |
| 前景层 | 谨慎适合 | 尘土、草木、轻风、路面雾 | 不建议让路面大幅运动，避免误解成动作游戏 |
| 车队层 | 不建议视频化 | 车队、辇车、随从 | 第一版应使用静态图或轻微 CSS 位移，不把车队烘焙进背景视频 |
| 神志污染层 | 适合 | 幻路、重影、第二月亮、血星、倒行云气 | 必须可关闭、可降级、透明度受控 |
| 转场层 | 可选 | 出发、抵达、进入异象节点 | 只用 1-2 秒短转场，不阻塞玩家操作太久 |
| UI 操作层 | 禁止 | 资源条、按钮、日志、事件文本 | 不使用视频作为文字或按钮背景 |

## 3. 背景、转场、污染叠层的分类

### 3.1 只适合做背景的视频

适合：

- 云气横移。
- 天色从白昼到暮色的微弱变化。
- 远处水泽波纹。
- 风沙薄层。
- 星象缓慢偏移。

要求：

- 循环点不明显，时长建议 4-8 秒。
- 画面低运动、低闪烁。
- 不含车队、人物主体、明确交互对象。
- 不含大面积明暗突变。
- 失败时可以被静态背景图完全替代。

### 3.2 适合做转场的视频

适合：

- 路线选择后，一层风沙或墨迹扫过。
- 抵达地点时，远景从雾中显现。
- 夜行或神志事件触发时，星象短暂错位。

要求：

- 时长建议 0.8-1.5 秒，不超过 2 秒。
- 不中断资源变化和日志记录。
- 可以跳过或直接降级为 CSS 淡入淡出。
- 不做大量事件短片，不让每个事件都等视频播完。

### 3.3 适合做神志污染叠层的视频

适合：

- 低神志时出现轻微重影。
- 背景中出现第二月亮、幻路、远处重复车影。
- 云气方向与行进方向相反。
- 星象、山影、水面发生不可信变化。

要求：

- 叠层透明度默认不超过 20%-35%。
- 不覆盖资源条、路线卡、事件文本、按钮、日志。
- 不使用强闪、快速抖动、刺眼红闪。
- 神志恢复或离开异常节点后必须能退场。

## 4. 如何保证视频不遮挡核心信息

第一版应采用“舞台在上，操作在下/侧”的结构。视频只能存在于舞台容器内部或舞台背景层，不进入 UI 面板。

硬性规则：

- 顶部资源栏不放视频背景。
- 路线卡不放视频背景。
- 事件文本容器不放视频背景。
- 行动按钮不放视频背景。
- 底部日志不放视频背景。
- 视频叠层不得拥有点击事件。
- 视频叠层不得覆盖 UI 操作层。

桌面端推荐层级：

```text
背景视频层 z=0
静态 BG/LOC/FG/CAR 层 z=1-4
神志视频叠层 z=5，仅限舞台容器
节点提示 z=6
资源/路线/事件/按钮/日志 z=20+
```

如果画面过亮或过复杂，应在舞台和 UI 之间增加固定遮罩，而不是把 UI 直接压在视频上。

## 5. 桌面端与移动端差异

| 项目 | 桌面端 | 移动端 |
|---|---|---|
| 舞台高度 | 260-320px | 132-180px |
| 背景循环 | 可使用 1 个短循环 | 默认静态图，必要时只用轻量 CSS 动效 |
| 转场 | 可用 0.8-1.5 秒短转场 | 默认禁用或压缩为淡入淡出 |
| 神志叠层 | 可用低透明视频叠层 | 优先静态叠层或 CSS opacity |
| 同屏视频数量 | 最多 1 个背景 + 1 个污染叠层 | 最多 1 个，低端设备为 0 |
| 文本阅读 | 事件区在舞台下方或右侧 | 事件区在舞台下方，避免视频占首屏 |

移动端首要目标是可读和可点，不追求动态完整。若视频导致掉帧、发热、加载慢，应直接降级为静态图。

## 6. 第一版建议视频强度

建议选择：A，只用短循环背景和神志异象。

| 强度 | 内容 | 评价 |
|---|---|---|
| A | 短循环背景 + 神志异象叠层 | 推荐。能验证氛围，又不增加大量素材生产压力 |
| B | 加入路线转场短片 | 可作为第二阶段，前提是 C 线程完成加载和跳过机制 |
| C | 大量事件短片 | 暂不建议。会把核心压力从规则和文本转向视频生产 |

第一版具体范围：

- 1 个背景短循环候选。
- 1 个神志污染短循环候选。
- 0-1 个路线转场候选，只作为可选增强。
- 所有视频必须有静态 poster 或静态图回退。

## 7. 需要 C 线程配合的技术需求

C 线程需要评估并给出实现方案：

1. 视频格式：优先 WebM/MP4 双格式或单格式兼容策略。
2. 体积上限：单个循环视频建议先控制在 2-4MB 内，移动端更低。
3. 加载策略：首屏先显示静态图，视频加载成功后再替换。
4. 降级策略：低端设备、低电量、移动端或加载失败时改用静态图。
5. 控制策略：默认静音、循环播放、不可点击、不抢焦点。
6. 可访问性：避免强闪、快速抖动，支持减少动态效果。
7. 性能监测：记录视频是否加载成功、是否降级、是否影响交互延迟。
8. 资源优先级：视频不得阻塞事件数据、按钮、资源条和日志渲染。

## 8. 禁止事项

- 不复制任何竞品镜头、构图、UI、转场和战斗结构。
- 不使用未授权视频、古籍扫描视频、影视片段、游戏录像。
- 不把车队烘焙进背景视频；车队必须保持独立主行动层。
- 不让视频取代核心玩法、事件文本、路线选择或资源判断。
- 不做泛用仙侠特效：飞剑、法阵、金光、灵气爆发不进入第一版。
- 不为了动态效果遮挡资源、路线、事件文本、按钮和日志。
