# BSI-UX-014 半途路遇与旅途舞台素材推进指令

- 发起时间：2026-06-15
- 发起人：主协调线程
- 当前目标：让第一版 Web Demo 更像“横屏手机游戏在行进”，而不是网页说明页。

## 1. 本轮用户意图

用户已明确：

- 先让机制跑起来，细节后面再补。
- 第一版有音乐，不讨论宣传。
- 大地图战略选择参考 Out There，但表现为羊皮纸山海舆图、迷雾显影和路线资源判断。
- 旅途/城镇页参考横向西行车队的感觉，可以“假动态”，不做真实碰撞平台玩法。
- 行走途中可以走一半突然停下来，进入具体事件选择。
- 背景图不应自带车队，车队应在前景单独行走。

## 2. 本轮 P0 目标

1. 半途路遇节奏
   - 不只在城镇发生互动。
   - 每 1-2 段关键路线至少能遇到一次“半途路遇”。
   - 路遇要真实改变车轴、粮草、神志或旗标。

2. 旅途舞台素材接入
   - 优先使用已入库背景长图、前景视差层、神志污染层、车队层。
   - 背景不带车队；车队层单独放前景。
   - 先用静态图 + CSS 假动态，不急着上视频。

3. 地图音效最小闭环
   - 优先补 `mapOpen`、`mapReveal`、`locationArrive` 三个短音效槽位。
   - 缺素材时仍要静默可玩。
   - 已有音乐入口不能消失。

4. 小横屏真实复核准备
   - 844x390、932x430、1280x720 作为下一轮主要检查尺寸。
   - 本轮代码要为真实浏览器复核留出明确检查点。

## 4. 主线程验收标准

本轮完成后必须满足：

- 至少 6 条路线有半途路遇，且主线前半段能体验到。
- 半途路遇不会连续触发到玩家完全无选择。
- 舞台素材能随地点/地形变化，不再只有一张固定背景。
- 车队仍作为独立前景层，不嵌在背景里。
- 三个地图短音效槽位不再全部 missing，或至少有明确可用占位路径。
- 语法与数据验证通过。
- 台账记录完整。
