## 3. 暂不建议直接使用

- 第四张里的红色“九州”篆字候选存在字形误读风险，不能作为正式标题。
- 过细的分隔线、细小路线在 844x390 小屏下可能糊，需要实际缩放测试后再定。
- 绿色残边已经做了一轮透明清理，但少数图标内部仍有偏绿颜料感，试装时若显脏再单独修。
- 特别复杂的角花和大纹样不要同时上屏，容易让 Web Demo 显得拥挤。

## 4. 对 C 线程接入建议

第一轮只接：

1. 资源图标：车轴 / 粮草 / 神志。
2. 动作小图标：修轴 / 搜粮 / 休整 / 查阅 / 行路。
3. 地图节点：当前 / 可达 / 已访 / 未知 / 危险 / 目的。
4. 路线标记：安全 / 风险 / 封锁。
5. 底板：事件、选择按钮、资源芯片、日志条、提示底板。

接入原则：

- 图标只替换含义，不改变现有布局结构。
- 小屏优先保证文字与按钮可读，不为装饰牺牲信息密度。
- UI 切片先以 24px、32px、48px 三档试缩放，确认边缘不脏再批量替换。
- 面板底板只作为轻纹理，不要用高透明度盖住事件文本。

## 5. 还缺什么

当前 UI / VI 不需要立刻继续大量生成。若还有积分要用，优先补：

1. 更干净的“动作图标专用大图”：只做 32-64px 可读的小图标，不做大面板。
2. 更统一的“按钮状态大图”：普通、悬停、按下、禁用、危险、已完成。
3. 更少装饰的“弹窗底板大图”：事件、危机、结局、奖励、日志五类底板。

不急：

- 正式标题字。
- 完整宣传 VI。
- 大量复杂角花。
- 带文字的任何 AI 字形。
