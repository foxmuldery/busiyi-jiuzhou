## 5. 给 C 线程的音频接入建议

### 5.1 默认策略

- 首次进入页面：默认静音，显示音频开关。
- 第一次用户点击按钮或选项后：解锁 Web Audio / HTMLAudio。
- 默认可开：UI 音效、资源变化音效。
- 默认不开：音乐底层。
- 环境声：可在首次交互后以极低音量淡入。

### 5.2 音量建议

| 通道 | 默认音量 | 最大音量 | 备注 |
|---|---:|---:|---|
| master | 0.70 | 1.00 | 总线 |
| music | 0.18 | 0.35 | 用户主动开启 |
| ambience | 0.28 | 0.45 | 首次交互后淡入 |
| travel | 0.22 | 0.40 | 只在移动时播放 |
| ui | 0.45 | 0.65 | 短促即可 |
| warning | 0.55 | 0.75 | 危急时触发 |
| sanity | 0.08-0.30 | 0.45 | 由神志值控制 |

### 5.3 循环点与淡入淡出

- 循环资产至少保留 150-300ms 淡入淡出，避免爆点。
- 行旅声循环不要有明显第一拍，否则循环会“咔哒”暴露。
- 音乐候选当前只适合测试循环，正式使用前需要人工检查头尾。
- 场景切换建议交叉淡入淡出 800-1500ms。
- 神志层不要突然出现，建议 2000ms 以上淡入。

### 5.4 移动端策略

- iOS/Safari 不要依赖自动播放。
- 第一次点击后再创建或 resume AudioContext。
- 页面隐藏时暂停或降低音量，页面恢复时淡入。
- 移动端最多同时播放 3-4 条循环层：环境、行旅、神志、音乐择一。

### 5.5 失败回退

| 失败情况 | 回退 |
|---|---|
| 音频加载失败 | 游戏继续运行，音频开关显示不可用 |
| 某一层缺失 | 跳过该层，不阻塞场景 |
| 浏览器禁止播放 | 等用户下一次点击再重试 |
| 移动端性能差 | 只保留 UI 音效和一个环境循环 |
| 文件未授权 | 只本地内部测试，不进入发布构建 |
