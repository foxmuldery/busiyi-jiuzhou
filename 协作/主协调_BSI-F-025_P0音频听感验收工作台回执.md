# 主协调 BSI-F-025 P0 音频听感验收工作台回执

日期：2026-06-24

## 1. 目标

把 P0 声音收口从“能播放、能复制模板”推进到“能逐条试听、逐条判断、生成可复制听感摘要”。

本轮只处理内部 Demo 听感验收，不把任何 `review-pending` 或 `demo-temporary` 音频标为正式可发布素材。

## 2. 已完成

- 新增 `P0音频听感验收工作台.html`。
- 工作台直接读取 `data.js` 中的 `audioAssets`，不复制第二份音频清单。
- 支持按全部、音乐、环境、神志、告急、短音、地图音效分组查看。
- 每条音频可试听、停止、标记“保留 / 待改 / 弃用”，并填写备注。
- 新增“分层验收门槛”，分别显示音乐、环境、神志、告急、短音、地图层的已评数量和缺口。
- 新增“顺序试听”流程，显示当前第几条、当前音频名，并提供上一条、听当前、下一条；标记“保留 / 待改 / 弃用”后会自动跳到下一条未评声音。
- 自动生成《不思异：九州》P0 音频听感验收摘要。
- 支持复制摘要、本机保存草稿和清空。
- 试玩入口已新增“音频验收台”入口，声音状态区也已补直达链接。
- QA 已新增 P0 音频听感验收工作台合约。

## 4. 使用边界

- `review-pending`：只表示可用于内部听感复核，不代表授权完成。
- `demo-temporary`：只表示 Demo 临时占位，不应进入正式素材区。
- 听感摘要用于主线程判断“是否加分、是否打扰、是否保留方向”，不替代授权复核。

## 5. 建议验收顺序

1. 先听 6 首核心音乐，判断第一章主音乐和地点轮换是否成立。
2. 再听 `travelAmbience`，判断风景主导横向行程是否更有远行感。
3. 再听轻/重神志层，判断低神志时是否有“不对劲”的感觉。
4. 最后听地图、选路、补给、资源涨跌和告急短音，判断是否打扰。
