# BSI-D-001 主界面信息架构

> 子线程：D 视觉体验与交互氛围设计师  
> 任务：第一版主界面信息架构  
> 阶段：Web Demo 测试原型  
> 边界：不生成最终图片，不引入外部素材，不写代码，不做营销 landing

## 1. 设计目标

第一版主界面要服务玩法验证，而不是展示最终美术。它应当让玩家在第一屏内完成核心判断：

1. 我现在在哪里。
2. 我还能去哪里。
3. 去那里要消耗什么。
4. 当前发生了什么事。
5. 我还能做哪些选择。
6. 我的车轴、粮、神志是否已经危险。
7. 刚才的选择造成了什么后果。

界面气质应是“山海远行工作台”：有古籍、行旅、异闻、污染感，但所有装饰都必须让位于可读、可选、可测试。

## 2. Web Demo 第一屏信息架构

### 2.1 第一屏必须出现的模块

| 模块 | 优先级 | 作用 | 第一版呈现方式 |
|---|---:|---|---|
| 资源状态 | 最高 | 告诉玩家还能撑多久、还能走多远、是否可信 | 顶部常驻三资源条，附状态词 |
| 当前地点 | 最高 | 明确玩家位置和区域 | 地图区与事件区都显示地点名 |
| 可前往路线 | 最高 | 形成空间选择和资源压力 | 地图节点或路线列表显示消耗 |
| 事件文本 | 最高 | 承载叙事、神志文本差异和选择前提 | 中央或右侧主阅读区 |
| 选择按钮 | 最高 | 推动核心循环 | 每个按钮显示行动名、可能代价、风险 |
| 行旅日志 | 中 | 记录资源变化和关键事件，支持复盘 | 底部或侧栏短日志，最多显示最近 5-8 条 |
| 队伍状态 | 中 | 后续扩展随从、疾病、器物 | 第一版可合并为小标签，不抢主流程 |
| 旅途舞台 | 中 | 提供横向远行、昼夜、云影、地貌气氛 | 作为氛围层，不承担动作碰撞和移动操作 |

### 2.2 推荐信息优先级

第一屏从上到下、从左到右的阅读顺序建议为：

```text
资源是否危险 -> 当前地点 -> 可选路线 -> 当前事件 -> 行动选择 -> 结果日志
```

如果屏幕空间有限，优先保留：

```text
资源状态 -> 当前事件 -> 选择按钮 -> 可选路线
```

地图可以在移动端降级为路线列表，但资源、事件、选择不能降级。

### 2.3 主推方向：假动态横屏长图 / 横向山海旅途舞台

主线程已确认第一版倾向采用“假动态横屏长图 / 横向山海旅途舞台”作为主推 UI 方向。它应理解为一段从左到右展开的小路程，不是横版动作游戏。它不负责碰撞、跳跃、即时移动，也不把玩家操作变成控制车队行走。它的任务是把节点选择和文字事件包装成一段正在发生的远行。

它可以借鉴“横向行进的戏剧感”：队伍在前景，世界在后方铺开，路程随选择推进。但不能复制任何现成游戏的美术、UI、战斗结构或镜头组织。第一版只吸收“横向旅途舞台”的产品表达，不吸收战斗玩法。

核心仍然是：

```text
看资源 -> 看横向旅途状态 -> 选节点/路线 -> 读事件 -> 做选择 -> 看结果日志
```

### 2.4 横向山海旅途舞台的信息层级

横向舞台建议放在第一屏上半部或中上部，作为玩家进入游戏后的第一视觉信号。它上承资源状态，下接路线、事件文本和选择按钮。

```text
┌──────────────────────────────────────────────┐
│ 顶部资源：车 / 粮 / 神 / 日数 / 当前区域 │
├──────────────────────────────────────────────┤
│ 横向山海旅途舞台                             │
│ 天空层：日月、云气、星象、天色               │
│ 后景层：山影、远泽、城关、异兽剪影           │
│ 主行动层：当前道路、节点标记、路线方向       │
│ 前景层：辇车、车队、草木、路面、尘土         │
│ 神志污染层：幻路、重影、异象、错位标记       │
├───────────────┬──────────────────────────────┤
│ 路线/节点选择  │ 事件文本与行动按钮             │
├───────────────┴──────────────────────────────┤
│ 行旅日志                                     │
└──────────────────────────────────────────────┘
```

| 层级 | 放置内容 | 玩法作用 | 第一版边界 |
|---|---|---|---|
| 天空层 | 天色、日月、云气、星象 | 表达昼夜变化、时间流逝、异常征兆 | 可以用颜色和简单形状表示，不需要最终天空图 |
| 后景层 | 山影、云雾、远处城关、河泽、异兽剪影 | 提供地域气质和远方风险 | 不做复杂插画，不让剪影变成可战斗对象 |
| 主行动层 | 路线方向、当前节点、可去节点、地貌提示 | 承担“我往哪里走”的空间判断 | 节点和路线必须比背景更清楚 |
| 前景层 | 辇车、车队、草木、路面、旗帜、车辙 | 强化队伍正在路上 | 不做角色动作系统，车队可作为剪影或符号 |
| 神志污染层 | 幻路、重复山口、第二月亮、倒行云气、错位车影 | 表达世界不可信 | 只覆盖氛围层，不遮挡资源、按钮、日志 |
| UI 操作层 | 资源条、路线卡、事件文本、行动按钮、日志 | 承担真实点击和判断 | 永远最高可读性优先 |

### 2.5 低成本假动态方案

第一版的“动态”应是低成本、可复用、可关闭的氛围变化，不是逐帧动画，也不是完整横向卷轴关卡。

建议会动的内容：

- 云气：缓慢横移，表现风和时间。
- 天色：在白昼、黄昏、夜间之间渐变。
- 前景草木或尘土：轻微摆动或透明度变化。
- 车队影子：极轻微上下起伏，暗示行进。
- 远景异兽剪影：偶尔出现、淡出，不参与交互。
- 神志污染：低神志时出现重影、错位、第二月亮、幻路淡入。

建议不动的内容：

- 资源条、数值、状态词。
- 事件文本、行动按钮、日志。
- 当前可选节点和路线消耗。
- 主要车队轮廓，避免玩家误以为能操控角色。
- 地点关键轮廓，避免视觉变化造成地理误解。

假动态可以通过三种低成本方式形成：

1. 分层平移：天空层和云气慢，前景草木快一点，制造远近差。
2. 状态切换：抵达新地点时更换少量地貌层，而不是重画整张图。
3. 污染叠层：神志降低时加一层半透明异象，不改底图结构。

### 2.6 从静态地点图升级为一段路程

不要把每个地点做成一张独立静态图。第一版更适合把“地点”拆成可复用层和路程片段。

静态地点图的做法：

```text
到雷泽 -> 显示雷泽图
到青丘 -> 显示青丘图
到赤水 -> 显示赤水图
```

主推做法：

```text
选择路线 -> 舞台层轻微推进 -> 地貌层替换一部分 -> 车队位置/节点标记更新 -> 事件出现
```

这样可以用较少素材制造“走过一段路”的感觉：

- 地面层换：平路、泽边、山道、荒原。
- 后景层换：远山、城关、泽水、昆仑影。
- 天空层变：清晨、暮色、夜行、异象天。
- 节点标记变：当前点、下一点、未知点。
- 事件区接入：抵达后文本解释真正发生了什么。

最小制作原则：

- 每条路线不需要独立长图，只需要“地貌组合 + 天色状态 + 节点位置”。
- 第一版不表现完整行走过程，只表现“出发前 / 行进中气氛 / 抵达后事件”三种状态。
- 横向舞台不承担精确地图，精确选择仍由路线卡和节点标记完成。

### 2.7 第一版美术最小资产清单

第一版不需要最终资产，但需要明确最小占位资产类型，方便 C 线程做 Web Demo 时不乱用外部素材。

| 资产 | 最小数量 | 用途 | 形式建议 |
|---|---:|---|---|
| 车队/辇车剪影 | 1 | 前景主角，表示队伍 | 简单单色剪影或线稿 |
| 前景地面条 | 3 | 平路、泽边、山道/荒原 | 可横向重复的低细节形状 |
| 草木/石块前景件 | 4-6 | 制造路上经过感 | 小型可复用剪影 |
| 后景山影/城关/水泽 | 4-6 | 区分地貌和远方风险 | 大块剪影，低细节 |
| 天空状态 | 3 | 白昼、黄昏、夜间 | 纯色/渐变色块即可 |
| 云气层 | 2-3 | 假动态横移 | 半透明形状 |
| 日月/星象符号 | 3-5 | 时间与异常征兆 | 简单符号，不用复杂插画 |
| 异兽远影 | 2-3 | 风险和神秘感 | 远景剪影，不做图鉴细节 |
| 神志污染叠层 | 3-4 | 幻路、重影、裂纹、第二月亮 | 半透明覆盖层 |
| 节点/路线标记 | 4-6 | 当前点、可去点、未知点、危险路线 | UI 符号，保持清楚 |

资产授权边界：

- 不引入外部插画、截图、古籍扫描、游戏素材。
- 第一版可用自制几何占位、线稿、剪影、纯色块。
- 后续若使用字体、纹理或参考素材，应交由 E 线程复核授权。

## 3. 桌面端布局方案

### 3.1 推荐桌面布局：横向舞台 + 双栏决策区

适合 1366px 以上宽度。

```text
┌──────────────────────────────────────────────┐
│ 顶部：车 / 粮 / 神 / 日数 / 当前区域          │
├──────────────────────────────────────────────┤
│ 横向山海旅途舞台：天空 / 后景 / 道路 / 车队 / 异象 │
├───────────────┬──────────────────────────────┤
│ 路线节点       │ 事件文本 + 行动按钮            │
│ - 当前节点     │ - 地点名 / 神志文本             │
│ - 可去节点     │ - 行动选择 / 资源预估           │
│ - 路线消耗     │ - 风险提示 / 队伍状态           │
├───────────────┴──────────────────────────────┤
│ 底部：行旅日志，显示最近选择、消耗、获得、异常 │
└──────────────────────────────────────────────┘
```

#### 布局说明

- 顶部资源条必须常驻，不随事件文本滚动消失。
- 横向舞台是主视觉层，但不盖住资源、路线、事件和按钮。
- 路线节点区不追求复杂绘制，第一版可以是节点图或路线卡片。
- 事件文本是决策区视觉重量中心，保证阅读宽度稳定。
- 行动按钮应直接标注代价，例如“夜渡赤水：车轴 -1 / 神志 -12 / 可能发现隐藏路”。
- 底部日志只记录结果，不替代事件文本，避免玩家混淆“当前可选”和“过去发生”。

### 3.2 桌面端尺寸建议

| 区域 | 建议宽度/高度 | 说明 |
|---|---|---|
| 顶部资源栏 | 64-88px | 能同时显示数值、图标、危险状态 |
| 横向舞台 | 220-320px 高 | 提供旅途气氛，不压缩正文 |
| 左侧路线节点 | 28%-34% 宽 | 保证路线关系可读 |
| 右侧事件行动 | 66%-72% 宽 | 正文和按钮同区，减少视线跳转 |
| 底部日志 | 96-140px | 足够显示最近 5 条短记录 |

## 4. 移动端布局方案

### 4.1 推荐移动布局：纵向分层

适合手机竖屏。

```text
┌────────────────────┐
│ 顶部：车 / 粮 / 神紧凑条 │
├────────────────────┤
│ 当前地点 / 日数     │
├────────────────────┤
│ 横向旅途舞台缩略层  │
│ 天色 / 车队 / 远山   │
├────────────────────┤
│ 路线区：可去地点卡片│
│ - 地点名            │
│ - 消耗              │
│ - 风险              │
├────────────────────┤
│ 事件文本            │
│ 神志文本变体        │
├────────────────────┤
│ 行动按钮            │
│ 主行动 / 次行动     │
├────────────────────┤
│ 折叠日志            │
└────────────────────┘
```

### 4.2 移动端交互原则

- 资源条固定在顶部，玩家滚动事件文本时仍能看到危险状态。
- 横向旅途舞台在移动端只保留 120-180px 高的缩略层，不追求完整长图。
- 地图在第一版可降级为“可去地点卡片”，避免小屏节点图难点选。
- 事件文本和行动按钮应相邻，减少玩家读完后寻找按钮。
- 日志默认折叠或只露出最后一条，避免挤占核心内容。
- 如果使用底部固定按钮，只放当前最重要的 1-2 个行动，其余放在事件文本下方。

## 5. 神志高 / 中 / 低界面变化原则

神志变化是玩法反馈，不是单纯滤镜。所有污染效果必须满足：仍能读、仍能点、仍能判断基本风险。

### 5.1 高神志：地理志界面

气质：稳定、冷静、古雅、可判断。

- 地图节点位置稳定，路线消耗明确。
- 文本排版规整，地点、地貌、风险标签清楚。
- 资源条颜色克制，只在低量时警示。
- 选择按钮说明相对可信，例如“低风险”“可能遭遇异族”。

### 5.2 中神志：异闻录界面

气质：征兆增多，玩家开始怀疑信息是否完整。

- 地图边缘可出现轻微墨迹扩散或路线虚线。
- 部分风险提示从明确变成含糊，例如“似有水声”“随从不愿前行”。
- 神志条本身出现轻微污染感，提示它不只是普通数值。
- 事件文本的段落可更短、更主观，但不应影响可读性。
- 隐藏路线或异常节点可以用“未确认”的方式出现。

### 5.3 低神志：疯人书界面

气质：世界开始背叛玩家，但操作仍要可靠。

- 地图允许出现重影、幻路、重复地点名，但真实可选路线必须仍能被系统识别。
- 选择按钮文案可以变得不完全可信，但按钮区域、点击目标和资源变化预估不能乱跳。
- 事件文本可出现断句、重复词、异样标点，但每段长度要短。
- 日志可能夹入异常记录，例如“你记得昨日已经到过此地”，但真实资源变化必须保留清楚数字。
- 不建议使用大面积抖动、强闪烁、低对比文字；这会破坏测试可用性，也有可访问性风险。

### 5.4 横向舞台中的神志污染

如果采用横向山海旅途舞台，神志污染不应只发生在文字层，也可以逐渐影响背景层，让玩家感觉“路上的世界不再可信”。

高神志时：

- 昼夜、云影、山影、道路方向相对可信。
- 前景车队和后景地貌分层清楚，玩家知道自己正在往哪里走。
- 异兽剪影只作为远处传闻或风险暗示，不干扰操作。

中神志时：

- 云影移动方向与车队方向略有矛盾。
- 远处山影出现重复轮廓，像是另一条路。
- 星象或日月位置轻微异常，用来暗示隐藏路线或不可靠风险。
- 后景可出现短暂异兽剪影，但不直接当成可点击目标。

低神志时：

- 后景可出现不可信异象，例如第二个月亮、倒行云气、远处车队影子、重复山口。
- 前景草木或路面可以轻微错位，暗示车队可能绕回旧路。
- 事件文本与背景出现呼应，例如文本说“水不是水”时，后景河泽变成类似道路的形状。
- 真实操作层必须保持稳定：资源条、按钮区域、可选路线、日志数字不能被背景异象遮挡。

## 6. 资源状态风险表达

三资源的第一任务是让玩家一眼知道“我怕什么”。

| 资源 | 玩家恐惧 | 风险表达 | 第一版建议 |
|---|---|---|---|
| 车轴 | 走不出去 | 低于路线最低消耗时强提示 | 显示“可通行 / 勉强可行 / 将困于此” |
| 粮 | 撑不过今天 | 用剩余天数感表达 | 显示“可支撑 X 日”，低粮时标红或加饥荒提示 |
| 神志 | 所见不真 | 状态词比百分比更重要 | 显示“清明 / 多疑 / 失真 / 崩坏边缘” |

### 6.1 风险分层

建议每项资源都有 4 档状态：

| 档位 | 范围建议 | 表达方式 |
|---|---|---|
| 安全 | 70%-100% | 常规显示，低装饰 |
| 紧张 | 40%-69% | 轻微警示，提示即将影响选择 |
| 危险 | 15%-39% | 明确警示，路线和选项显示风险 |
| 崩溃 | 0%-14% | 强提示，触发困境或特殊文本 |

### 6.2 路线选择里的资源表达

路线按钮或路线卡片应直接显示消耗和风险，不只显示地点名。

示例结构：

```text
北路：前往雷泽
车 -8 / 粮 -2 / 神 0
风险：低，路远但稳
```

```text
夜路：前往赤水
车 -1 / 粮 -1 / 神 -12
风险：未知，可能见隐藏路线
```

按钮状态建议：

- 可以承受：正常按钮。
- 勉强承受：按钮可点，但显示危险边框和确认提示。
- 无法承受：按钮禁用或转为“需要修复 / 采集 / 等待”。

## 7. 视觉风格关键词的克制使用

第一版不是最终美术，视觉关键词只作为 UI 语言，不应堆满屏幕。

| 关键词 | 可用方式 | 克制原则 |
|---|---|---|
| 帛书 | 背景纸感、事件文本容器 | 不做强纹理，避免影响阅读 |
| 竹简 | 资源或日志的条目分隔 | 不把所有按钮都做成竹片 |
| 龟甲 | 神志、占兆、异常提示符号 | 只用于关键状态，不当通用装饰 |
| 朱砂 | 危险、标记、当前节点、神志污染 | 只做点睛，避免整屏红色 |
| 墨迹 | 地图线、污染、未知路线 | 不覆盖正文，不降低对比度 |
| 古籍批注 | 日志、风险说明、地点标签 | 文字要短，避免假古风堆砌 |

总体原则：

- 第一版可以用颜色、边框、字重、留白表达气质。
- 暂不需要真实纸张纹理、古籍扫描、插画、商用字体。
- 避免“泛用仙侠”：金光、云纹、飞剑、灵气、华丽法阵不适合本项目。
- 古籍感应来自“记录、标注、缺损、误读”，不是来自装饰堆砌。

## 8. 可选 UI 方向

### 方向 A：横向山海旅途舞台

核心：第一屏上半部或中部是一条假动态横屏长图，像一段从左到右展开的小路程。前景放辇车、车队、草木、路面，后景放山影、云气、日月星象、远处异兽剪影；下方或侧边仍然是可选择节点、事件文本和行动按钮。

它不是横版动作游戏：

- 不做碰撞。
- 不做跳跃、攻击、即时操控。
- 不要求玩家控制车队横向移动。
- 横向画面只表达“旅途正在推进”和“世界正在变化”。
- 不复制任何现成游戏的美术、UI、战斗结构或镜头组织。

优点：

- 远行感、日出日落、白云漂移、路上风景最强。
- 能把节点选择包装成一段连续旅途，而不是纯菜单切换。
- 神志降低时，背景异象可以自然出现，比只改文字更有感染力。
- 适合形成项目辨识度，避免界面过于后台工具化。

缺点：

- 第一版需要明确视觉层级，否则容易挤压文字事件和按钮。
- 如果舞台过重，会被误解成横版动作或跑酷。
- 移动端屏幕较窄，横向舞台高度必须克制，否则正文会被挤出首屏。
- 后续如要动效，需要 C 线程控制实现范围，避免变成美术和动画大工程。

适用判断：主线程已确认它是第一版主推 UI 方向；第一版只做信息架构、分层占位和低成本假动态，不追求最终素材。

### 方向 B：山海图中心工作台

核心：地图在视觉中心，事件和选择围绕地图展开。

优点：

- 远行感最强，玩家一眼知道自己在路上。
- 路线消耗和空间选择直观。
- 后续扩展隐藏路线、幻路、重复地点更自然。

缺点：

- 对第一版实现要求略高。
- 如果地图占比过大，事件文本可能被压缩。
- 移动端需要改造成路线卡片，否则节点难点。

适用判断：如果 C 线程能快速实现基础节点图，这是最能表达空间选择的方向。

### 方向 C：游记卷轴阅读台

核心：事件文本是主视觉，地图作为可选路线辅助。

优点：

- 最适合验证神志文本系统。
- 实现成本低，可快速测试事件选择。
- 失败游记感强，日志可以自然融入界面。

缺点：

- 远行地图感较弱。
- 玩家可能感觉更像普通文字冒险。
- 路线之间的空间关系不如方向 A 清楚。

适用判断：如果第一版重点是验证文本、事件和资源变化，这是最稳妥方向。

## 9. 推荐路线

建议第一版主推“方向 A：假动态横屏长图 / 横向山海旅途舞台”，再吸收方向 B 的路线选择清晰度和方向 C 的文本重心：

- 桌面端在资源条下方设置横向山海旅途舞台，作为主视觉层。
- 横向舞台只做前景、主行动层、后景、天空层、神志污染层的信息层级，不做动作玩法。
- 第一版用假动态：云影、天色、少量前景、异象轻微变化；资源、按钮、文本、路线消耗保持稳定。
- 地图不做复杂美术，只做节点、路线消耗和可选目标。
- 事件文本保持最大可读区域。
- 资源状态常驻顶部。
- 日志放底部，支持复盘。
- 视觉上使用少量朱砂、墨线、帛书底色，不做最终资产。

这样能同时服务 C 线程的 Web Demo、A 线程的资源规则、B 线程的神志文本，并把“从左到右展开的小路程”作为第一版辨识度，同时不提前进入最终美术制作。

## 10. 需要用户决策的问题

1. 横向旅途舞台在桌面端首屏高度应控制在多少？建议默认：约 260px，上限 320px。
2. 移动端第一版是否允许把节点地图降级为路线卡片？建议默认：允许。
3. 神志低时是否允许背景层出现幻路、异象、第二月亮等不可信视觉，但真实资源变化仍保持清楚？建议默认：允许，但第一版只做轻量表现。
4. 第一版车队视觉用“剪影符号”还是“低细节线稿”？建议默认：剪影符号。
5. 第一版是否只做 3 种天色状态：白昼、黄昏、夜间？建议默认：是。
