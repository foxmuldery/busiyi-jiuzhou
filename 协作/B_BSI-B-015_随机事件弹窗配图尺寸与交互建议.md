## 1. 推荐结论

随机事件图不要按全屏 16:9 作为唯一母图。结合新画风参考，建议改为 4:3 装帧事件牌母图：

| 用途 | 推荐比例 | 推荐尺寸 | 说明 |
| --- | --- | --- | --- |
| 生成母图 | `4:3` 横向事件牌 | `2048x1536` 或 `1600x1200` | 最推荐。最接近用户参考图，能保留边框、近景主体和远景叙事。 |
| 弹窗主图 | `4:3` | CSS `aspect-ratio: 4 / 3` | 弹窗里完整展示，适合“事件牌 / 查看详情”。 |
| 紧凑弹窗图 | 不单独生成 | 从 4:3 母图按容器中心裁切 | 生图引擎没有额外中间比例时，直接用 4:3 母图，前端按实际容器裁。 |
| 小横屏缩略图 | `16:9` 中心裁切 | 约 `240x135` 到 `320x180` | 只做预览，不抢文字和按钮空间。 |
| 舞台/横向背景复用 | `16:9` 或更宽裁切 | 从 4:3 母图中心裁 | 只取主体横向氛围，不要求完整展示事件。 |
| 结局/危机大图 | `16:9` 或 `4:3` | 视 UI 决定 | 结局可更大，普通随机事件不默认全屏。 |

## 2. 为什么不是默认 16:9

16:9 很适合横向旅途舞台，但不一定适合事件弹窗：

1. 事件图要讲清“一个瞬间”：车轴、粮袋、碑、手、影子、骨桥等主体需要足够高度。
2. 弹窗不是全屏，16:9 缩小后容易变成一条扁横幅，细节看不清。
3. 小横屏 `844x390` 高度太紧，常驻 16:9 图会挤压事件文本和选择按钮。
4. 4:3 母图可以向下兼容：弹窗完整看 4:3，紧凑弹窗按容器中心裁切，舞台/缩略图裁成 16:9。

## 3. 为什么不是默认 4:3

此前担心 4:3 会偏“插画卡面”，但用户给的新画风本身就是事件牌方向，且更适合随机事件弹窗。因此现在把 4:3 作为唯一推荐生成母图比例；紧凑弹窗只做前端裁切，不再写成生图比例要求。

为避免和横向旅途舞台脱节，提示语必须强调：中心主体清楚，背景仍有横向道路/车队/远景，可安全裁成 16:9 缩略图。

## 4. 弹窗显示建议

### 4.1 桌面 / 普通横屏

推荐：

- 弹窗宽度：`min(840px, 78vw)`
- 弹窗高度：不超过 `82vh`
- 图片比例：`4:3`
- 布局：上图下文，或左图右文

两种可选布局：

| 布局 | 适用 | 建议 |
| --- | --- | --- |
| 上图下文 | 桌面、平板、普通横屏 | 图片完整展示，下面放事件标题、正文、选项。 |
| 左图右文 | 小横屏、低高度横屏 | 左侧图片约 38%-44% 宽，右侧放正文和按钮。 |

### 4.2 844x390 / 932x430 小横屏

推荐默认不是全屏大图，而是：

1. 折叠状态：只显示事件标题、两行正文、选择按钮；图片可显示为小缩略图或隐藏。
2. 展开状态：弹出事件详情层，图片默认用 4:3；若高度吃紧，直接用同一张 4:3 图按容器中心裁切，宽度控制在弹窗 38%-44%。
3. 图片点击：可进入“查看插图”大图层，但这是二级查看，不是默认流程。
4. 不要让图片挤掉主选择按钮，按钮仍是第一优先级。

小横屏建议数值：

| 项目 | 建议 |
| --- | --- |
| 弹窗宽度 | `min(760px, 88vw)` |
| 弹窗高度 | `min(320px, 82vh)` |
| 左图宽度 | 弹窗宽度的 `38%-44%` |
| 左图比例 | 默认 `4:3`，高度吃紧时仍用同图居中裁切 |
| 文字区 | 至少保留标题 + 2 行正文 + 2 个选择按钮 |
| 缩略图兜底 | `16:9`，约 `240x135` |

## 5. 生成提示语需要同步调整

后续所有随机事件配图提示语统一写：

```text
horizontal 4:3 framed event-card composition, safe center crop for compact popup containers and 16:9 stage thumbnails
```

不要再只写：

```text
horizontal 16:9 composition
```

这样生成母图时会更适合弹窗，后续 C 线程仍可裁出 16:9 缩略图或舞台图。

## 6. 给 C / D 线程的落地备注

1. C 线程接图时，建议数据里区分 `imageFull`、`imagePopup` 和 `imageThumb`，即使第一版先指向同一张图。
2. CSS 上事件详情图容器建议用 `aspect-ratio: 4 / 3`，紧凑弹窗沿用同一张图并用容器裁切，缩略图容器用 `16 / 9`。
3. 小横屏下普通事件图默认折叠，危机/结局图可以优先显示。
4. 若事件弹窗采用左图右文，图片主体必须在画面中心 70% 安全区内，避免裁切掉事件核心。
5. B 线程提示语应要求：主体居中、不要文字、不要 UI、不要过多边缘细节。
