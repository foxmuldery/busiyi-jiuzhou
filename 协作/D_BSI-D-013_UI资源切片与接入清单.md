## 0. 本轮结论

用户新一批 UI、地图、结局与资源图已经可以支持 Web Demo 做一次“跑起来”的视觉替换。C 线程本轮不需要等待完整美术体系，可先按下面四类接入：

1. 资源状态：用小图标 + 数字 + 风险色，替代“获取资源”这类纯文字按钮。
2. 地图界面：用大地图底图 + 节点/路线切片，先做可读的可选节点。
3. 事件界面：用事件面板与选择按钮切片，保持文字优先。
4. 开始/失败/结局界面：用整图背景先铺底，文字和按钮仍走 HTML 层。

自动切片候选共有 191 件，其中精选切片 150 件。本轮 C 线程优先使用“精选切片”，不要直接遍历全部候选。

## 3. 资源状态与小图标按钮方案

用户指出“获取资源”如果是文字按钮不够直观。建议首版改成图标化：

| 状态 | 表达方式 | 示例 |
|---|---|---|
| 资源总览 | 顶部或左上角三枚图标 + 数字 | 车轴 3、粮草 8、神志 72 |
| 可获得资源 | 选择按钮右侧显示小图标收益 | `+1 车轴`、`+2 粮草`、`+5 神志` |
| 消耗风险 | 选择按钮右侧显示小图标消耗 | `-1 车轴`、`-2 粮草`、`-8 神志` |
| 危险阈值 | 图标外圈变朱砂，数字加粗 | 车轴 <= 1、粮草 <= 2、神志 <= 35 |
| 获取资源按钮 | 改为图标按钮 + 短词 | 图标 + `修轴`、图标 + `搜粮`、图标 + `定神` |

按钮文字不要写成“获取资源”。资源行为最好直接写动作：`修轴`、`搜粮`、`扎营`、`饮药`、`祭问`。玩家看到图标就知道影响哪条资源。

## 4. 第一屏 UI 布局建议

### 桌面端

```text
+--------------------------------------------------+
| 资源图标条        当前地点/路线         日志入口 |
|--------------------------------------------------|
|                                                  |
|          横向旅途舞台 / 九州图 / 地点显影        |
|                                                  |
|--------------------------------------------------|
| 事件文本面板                         小地图/日志 |
| 选择按钮 1  选择按钮 2  选择按钮 3               |
+--------------------------------------------------+
```

资源条建议固定在上方，不进入旅途舞台图层。事件文本和选择按钮固定在下方或右侧，避免压住地图节点和车队。

### 移动横屏

```text
+-----------------------------------------------+
| 资源图标条         地点名             菜单/日志 |
|-----------------------------------------------|
|      横向旅途舞台或九州图，占画面 58%-64%      |
|-----------------------------------------------|
| 事件文本一行摘要 / 选择按钮横排或两行紧凑排布  |
+-----------------------------------------------+
```

移动端不要把事件面板做成大面积羊皮纸盖住舞台。按钮高度固定，长文案截断或改为两行。

## 6. 地图 UI 接入规则

1. `MAP-002` 作为大地图底图，地图节点使用 `UI切片精选/地图节点路线/` 的节点 PNG。
2. 已知地点优先用金色圆节点，未知地点用灰色雾化节点，危险地点用朱砂针或红圈。
3. 清晰路线用 `MAPROUTE-*` 的线段；红色碎线只用于异常路线、低神志提示或裂隙路径。
4. 雾片只覆盖未显影区域，不覆盖资源条、事件文本和选择按钮。
5. 低神志时可以让地图出现“重复节点”“错误路线”“红色裂纹”，但实际可点击路径仍要清楚。

## 7. 风险与禁用

| 项目 | 风险 | 处理 |
|---|---|---|
| 自动切片候选 | 含碎边、小残片、重复部件 | C 线程只接 `UI切片精选` |
| 绿幕抠图 | 个别边缘可能残留绿色 | 首版可用，但上线前要视觉复核 |
| AI 生成素材 | 需 E 线程继续记录授权与商业使用边界 | Demo 标记为临时素材 |
| 片名参考 | `TITLE-REF-*` 是重绘概念，不等于官方固定台标 | 正式标题优先使用品牌台标目录的固定 PNG |
| 大量纹理叠加 | 会遮挡事件文本和按钮 | 所有雾、裂纹、污染层不得盖 UI 操作层 |

## 8. C线程最小实施清单

1. 把资源条改为三枚图标 + 数字 + 低值风险色。
2. 把“获取资源”类按钮改为图标动作按钮：`修轴`、`搜粮`、`定神`。
3. 事件文本容器使用 `P0_event_main_text_panel.png`；选择按钮使用 `P0_event_choice_button_a/b.png`。
4. 九州图页面接 `MAP-002`，节点优先接金色/灰色/朱砂三类状态。
5. 失败和结局页先接整图背景，文案仍走前端。
