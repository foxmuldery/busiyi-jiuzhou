# BSI-UX-014 主线程验收清单

- 验收对象：半途路遇节奏、旅途舞台素材映射、地图三短音效
- 验收人：主协调线程
- 状态：待 A-F 子线程交付后执行

## 1. 基础检查

```bash
node --check /Users/yuanzhe/Documents/game/GitHub资产区/03_WebDemo/prototype/app.js
node --check /Users/yuanzhe/Documents/game/GitHub资产区/03_WebDemo/prototype/data.js
```

必须通过：

1. 语法检查无错误。
2. `window.BSI_PROTOTYPE.validate()` 或等价数据验证无错误。
3. `routes[].midEvent` 引用的路遇事件均存在。
4. 舞台素材路径能按静态 HTML 相对路径解析。
5. 音频素材缺失时不阻断玩法。

## 2. 半途路遇验收

必须满足：

1. 至少 6 条路线设置 `midEvent`。
2. 主线前半段至少 2 条路线可触发半途路遇。
3. 半途路遇选择必须真实改变资源、旗标、语言或日志。
4. 已处理过的同一路遇不会在同一路线重复卡住玩家。
5. 半途路遇处理后能继续抵达目的地。
6. 玩家在半途时不能继续城镇补给，避免逻辑错位。

建议统计：

```text
路线总数：
midEvent 覆盖：
主线 midEvent 覆盖：
routeEvents 数量：
```

## 3. 旅途舞台素材验收

必须满足：

1. 舞台背景随地点或地形变化，不再只有固定一组背景。
2. 背景层不自带车队，车队仍由独立 `caravan` / `caravan-art` 层负责。
3. 至少包含 road / market / water / rift 四类视觉状态。
4. 低神志污染层可随神志下降增强，但不遮挡主要文字与按钮。
5. 运动关闭时仍能看到清楚静态画面。
6. 缺少某张素材时有 CSS 或默认层兜底。

## 4. 音效验收

必须满足：

1. `mapOpen` 有资产记录；缺素材或待复核时仍静默可玩。
2. `mapReveal` 有资产记录；缺素材或待复核时仍静默可玩。
3. `locationArrive` 有资产记录；缺素材或待复核时仍静默可玩。
4. `routeSelect` 与 `supplyComplete` 旧有交互音不回退。
5. 音频 UI 不把待复核素材误报为正式授权。

建议统计：

```text
mapOpen:
mapReveal:
locationArrive:
routeSelect:
supplyComplete:
musicLoop:
```

## 5. 小横屏验收

重点尺寸：

```text
844x390
932x430
1280x720
```

必须满足：

1. 车轴、粮草、神志可读。
2. 旅途 / 地图切换可点。
3. 半途路遇事件标题、文本和选择不被舞台遮挡。
4. 补给按钮与剧情选择不互相挤压。
5. 行囊九宫格可打开、可关闭。
6. 音乐入口可打开设置。

## 6. 风险验收

必须避免：

1. 宣传片、PV、商店描述进入本轮主线。
2. 背景中出现重复车队或难处理黑白格透明底。
3. 西部牛仔、枪、酒馆、马车驿站等具体西部符号。
4. 星球、星图、发光航线、扫描 UI。
5. Darkest Dungeon 式英雄队列、火把/压力条直接复刻。
6. FAR 式载具剖面、锅炉驾驶谜题。
