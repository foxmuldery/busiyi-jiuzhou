# 主协调 BSI-UX-076 P0 完整试玩包复制入口回执

日期：2026-06-24

## 1. 目标

把 P0 从“本地可以打开试玩入口”推进到“可以直接把一段完整试玩说明发给测试者”。

本轮不改游戏机制，只增强试玩入口的交接能力。

## 2. 已完成

- `试玩入口.html` 新增“复制完整试玩包”按钮。
- 完整试玩包包含：
  - 干净新局链接。
  - 横屏试玩建议。
  - 5 分钟试玩目标。
  - 处理遭遇、补给、开图、选路、半途路遇、抵达下一站的核心循环。
  - 三项观察重点：看懂下一步、资源压力、愿意再走一站。
  - 试玩反馈模板。
  - 音频听感模板。
- 2026-06-24 追加：完整试玩包现在会带出 P0 真人试玩验收台和 P0 音频听感验收台链接，并说明“测试者回传三段内容 -> 主线程粘贴进验收台 -> 复制验收摘要”的回收路径。
- 2026-06-24 追加：试玩入口新增“复制回收清单”，面向主线程而不是测试者；清单固定要求回收《5分钟试玩记录》《音频听感复核》和游戏内“志”的本局复盘，并提示用反馈证据雷达检查五类证据。
- 复制失败时沿用现有手动复制兜底，不新增交互分支。
- QA 已补守门，避免后续误删该入口。

## 3. 使用方式

打开：

```text
GitHub资产区/03_WebDemo/prototype/试玩入口.html
```

点击“复制完整试玩包”，直接粘贴给测试者。测试者回传后，主线程打开包内的 P0 真人试玩验收台链接，把试玩记录、音频听感和本局复盘贴进去，勾选门槛后复制验收摘要。

主线程回收阶段点击“复制回收清单”，用于检查三段材料是否齐全，并把 P0 总验收报告、真人验收台、音频验收台三个入口一起发给负责收口的人。

## 4. 边界

- 这不是正式发行文案，只用于内部 P0 试玩邀请和反馈回收。
- 完整试玩包里仍明确要求测试者回传本局复盘文本和听感反馈。
- 回收清单只证明回收流程清楚，不替代真人手感和主观听感判断。
- 如果后续部署到线上，需要把复制出的链接换成线上地址；当前会自动使用打开入口页时的当前地址生成测试链接。
