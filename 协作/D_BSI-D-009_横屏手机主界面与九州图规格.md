# D_BSI-D-009 横屏手机主界面与九州图规格

> 子线程：D 视觉体验与交互氛围设计  
> 任务类型：产品体验与 UI 规格  
> 项目：《不思异：九州》Web Demo  
> 日期：2026-06-15  
> 边界：本文件只写体验规格，不修改 `index.html`、`styles.css`、`app.js`

## 0. 本轮结论

首版横屏手机体验应收成两个核心页面：

```text
地点停靠页：车队停下，处理当前地点、补给、剧情选择、反馈和继续上路。
九州地图页：打开羊皮纸舆图，比较路线、资源消耗、风险、雾中未知和锁定条件。
```

玩家不应该感觉自己在一个网页 dashboard 里来回点卡片，而应该感觉：

1. 我带着西行车队停在某个地点。
2. 我能看见车轴、粮草、神志的危险程度。
3. 我处理完当前地点后，打开九州图选择下一段路。
4. 上路后看到一小段横向行走图，途中可能停下处理路遇。
5. 抵达新城市后，地图雾散、城市显现、后续路线显现。

参考方向可以是“西行车队 / 地图远行感 / Darkest Dungeon 式假动态”，但只借鉴结构：横向旅途、剪影层次、短程推进、停下处理事件。不复制具体镜头、美术、角色站位、UI 框架、音效和文案。

## 1. 横屏手机主界面布局

目标设备先按手机横屏小尺寸设计，约 `16:9` 到 `19.5:9`。首版不做竖屏主体验。

### 1.1 总体区域

横屏主界面分为顶部 HUD + 中部视觉区 + 底部主选项区。不要把屏幕切成很多同权卡片。

```text
顶部 HUD：当前位置 / 三资源 / 当前目标 / 页面切换 / 行囊入口
中部视觉区：地点停靠页显示行走图；九州地图页显示羊皮纸舆图
底部主选项区：当前说明、补给、剧情选择、反馈、继续上路
```

在现有结构中可对应：

| UI 区域 | 现有结构 | 首版职责 |
| --- | --- | --- |
| 顶部 HUD | `.game-hud` | 当前地点、第几日、三资源、旅途/九州图切换、行囊入口 |
| 地点停靠页 | `.town-view` / `.journey-stage` / `.event-panel` | 行走图、地点事件、补给、选择和反馈 |
| 九州地图页 | `.map-view` / `.kyushu-map` / `.map-route-panel` | 选下一段路、看迷雾、看资源预览 |
| 行囊抽屉 | `.side-drawer` / `.inventory-grid` | 九宫格行囊、语言、困厄复查 |

### 1.2 顶部 HUD 左 / 中 / 右

顶部 HUD 是横屏手机最稳定的识别区，建议三段：

| 区域 | 内容 | 规格 |
| --- | --- | --- |
| 左区：当前位置 | 地点名、第几日、区域/路段状态 | 地点名最大；第几日小字；小横屏隐藏项目名和区域长文 |
| 中区：三资源 | 车轴、粮草、神志 | 每项显示名称、数值、细条；危险时变色和轻闪，不写长说明 |
| 右区：当前目标 | `旅途 / 九州图` 切换、距裂隙、下一步提示、行囊入口 | 目标文字必须告诉玩家现在该做什么；行囊入口可常驻小按钮 |

三资源显示建议：

1. `车轴` 用偏冷青灰或铁色，代表车队耐久。
2. `粮草` 用草木绿或土黄，代表补给。
3. `神志` 用金色偏白，低值时转暗红或污染色。
4. 数值必须比解释更重要；小横屏可以隐藏 `尚可远行` 这类提示。
5. 危险阈值用视觉状态表达：低于 35 明显警戒，低于 20 强警戒。

### 1.3 地点停靠页左 / 中 / 右

地点停靠页不是地图页，也不是日志页。它是“车队停在这里，我要处理完再走”的页面。

推荐横屏布局：

| 区域 | 内容 | 首版位置 |
| --- | --- | --- |
| 左区：到达与事件说明 | 当前地点名、到达介绍、事件正文、选择后反馈、最近日志一条 | 底部主选项区左侧；小横屏保留标题 + 2 行正文 |
| 中区：补给 / 整备 | 当前地点可搜集补给、已搜集状态、不可搜集原因 | 底部主选项区中侧；半途路遇时显示“无法补给” |
| 右区：主选择 | 剧情选择、困厄补救、继续上路入口 | 底部主选项区右侧；按钮最大，触控优先 |

中部视觉区 `journey-stage` 仍然是主视觉：车队、路、远景、云、污染、路标。底部三列只负责当前操作，不要抢掉行走图。

### 1.4 行囊九宫格放哪

行囊九宫格首版不要常驻主屏。原因是横屏手机空间太紧，行囊是复查信息，不是每秒行动。

规格：

1. 顶部 HUD 右区保留 `行囊` 按钮。
2. 点击后从右侧打开 `.side-drawer`。
3. 抽屉里第一屏显示 `.inventory-grid` 九宫格。
4. 九宫格下面或旁边显示语言、困厄、关键状态。
5. 抽屉不改变当前页面状态，关闭后回到原来的地点停靠页或九州地图页。
6. 小横屏抽屉宽度不要超过屏幕 42%，避免遮住全部主画面。

P0 不建议把行囊做成底部常驻栏。最多在 HUD 上放一个小数字或危险标记，例如“行囊 2 / 困厄 1”。

## 2. 两个页面的关系

两个页面的关系要清楚：

```text
地点停靠页 = 处理当前地点。
九州地图页 = 选择下一段路。
```

### 2.1 默认流程

```text
进入地点
-> 地点停靠页显示到达介绍
-> 玩家处理补给和剧情选择
-> 主选项区提示“打开九州图选择下一段路”
-> 九州地图页比较路线
-> 选择路线
-> 回到地点停靠页的行走图，播放一小段行进
-> 半途路遇或抵达新地点
```

### 2.2 页面切换

现有 `旅途 / 九州图` 切换可以保留，但视觉上要像游戏内模式按钮，不像网页 tab。

切换规则：

1. 玩家在地点停靠页处理事件和补给。
2. 玩家想出发时，点击 `九州图`。
3. 九州地图页只显示当前可走路线和少量雾中暗示。
4. 玩家选路线后自动回到地点停靠页，进入行走图状态。
5. 半途路遇时，九州图入口可保留，但路线列表提示“先处理路遇”。
6. 当前地点事件已处理后，地点停靠页必须明确提示“打开九州图选择下一段路”。

### 2.3 玩家如何知道当前目标

当前目标必须在两个位置同时出现：

1. HUD 右区：一句短目标。
2. 主选项区：具体可点行动。

建议目标文案状态：

| 状态 | HUD 目标 | 主选项区提示 |
| --- | --- | --- |
| 刚抵达 | `处理当前地点` | 显示到达介绍和剧情选择 |
| 有补给未取 | `可整备车队` | 补给区显示可搜集项 |
| 剧情未处理 | `作出选择` | 右区显示剧情选项 |
| 剧情已处理 | `选择下一段路` | 显示“打开九州图”提示 |
| 正在行走 | `车队在路上` | 行走图播放，选项暂时降权 |
| 半途路遇 | `处理半途路遇` | 补给不可用，右区显示路遇选择 |
| 濒死困厄 | `先救队伍` | 只显示补救选择，不让继续选路 |
| 无路可走 | `暂无显明路线` | 提示等待下一版困境/归返事件或保底路线 |

## 3. 羊皮纸九州图规格

九州地图页是战略选择页。它必须像一张逐步被走出来的羊皮纸山海舆图。

### 3.1 地图底

`.kyushu-map`：

1. 旧纸 / 帛书底色，偏暖。
2. 淡墨山水、河泽、折痕和污渍。
3. 边缘略暗，像旧图卷边。
4. 不用深色科技网格，不用星图背景。

`.map-grid`：

1. 改成弱经纬线、折痕、山川纹。
2. 只提供坐标感，不当主视觉。
3. 小横屏下透明度更低，避免糊住节点。

### 3.2 节点

节点状态：

| 状态 | 视觉 | 交互 |
| --- | --- | --- |
| 当前 | 朱砂印、车队小旗、清晰地名 | 锚点，不一定可点 |
| 已见 | 淡墨地名签、已录入地点志的小印 | 可复看，但不作为出发按钮 |
| 可达 | 金墨或朱砂描边，略亮 | 可点，或联动路线卡 |
| 雾中 | 模糊墨点、残缺地名、山影或水渍遮住 | 不可点，只给方向感 |
| 锁定 | 红封、断墨、压暗地名 | 不可走，路线卡显示条件 |
| 抵达中 | 墨晕显影，地名将出未出 | 半途状态，不可重复点 |

节点文字规格：

1. 桌面可显示地名 + 1 行状态。
2. 小横屏只显示地名或短名。
3. 不在节点里放完整消耗、风险、补给。
4. 不使用现代问号图标；雾中可以写“西北有影”“泽中未明”。

### 3.3 雾

雾不是黑遮罩，而是纸雾、墨晕、水渍、残卷缺页。

规格：

1. 雾中区域仍能看见一点地图质地。
2. 雾遮住完整路线关系，只露方向和影子。
3. 到达城市后，只散开当前城市周边，不揭全图。
4. 低神志时，雾可以有污染色或错字影，但不能影响基础可读。

### 3.4 路线

路线状态：

| 状态 | 视觉 | 信息 |
| --- | --- | --- |
| 已走路线 | 淡墨车辙、旧折痕、低亮 | 表示旅程记忆，不强调可点 |
| 可走路线 | 朱砂 / 金墨实线，略粗 | 当前可选，联动路线卡 |
| 雾中路线 | 断续虚线、残墨、被雾切断 | 只提示有方向，不给完整终点 |
| 锁定路线 | 红墨断线、封条、裂纹 | 路线存在但条件不足 |
| 抵达中路线 | 墨线流动或轻微晕开 | 车队正在这段路上 |

可走路线不需要箭头满屏飞。方向可以靠起终点位置、路线卡高亮、线尾淡出表达。

### 3.5 资源预览

资源预览主要放在 `.route-card`，地图上只做轻提示。

路线卡固定顺序：

```text
路线名
起点 -> 目的地
消耗：车轴 / 粮草 / 神志
风险：低 / 中 / 高 + 半途路遇提示
目的地可观察：补给倾向
提示或锁定原因
```

规格：

1. 三资源必须同一顺序：车轴、粮草、神志。
2. 负消耗用短文本，不要长解释。
3. 高风险或高消耗路线在卡片上有明显警戒。
4. 锁定卡仍显示诱惑和条件，不只是灰掉。
5. 小横屏卡片只保留核心：路线名、去向、消耗、风险、锁定条件。

## 4. 行走图规格

行走图负责让玩家感觉“车队真的走了一段”，不是点击路线后直接换地点。

### 4.1 假动态横屏长图

`journey-stage` 使用假动态横屏长图方式：

1. 背景长图或多层背景横向缓慢移动。
2. 车队常驻前景，从左向右推进一小段。
3. 远山、云雾、路面、车辙有不同速度的轻视差。
4. 路标显示起点、路线名、下一站。
5. 不做实时操控，不做跑酷，不做碰撞。

可以借鉴 Darkest Dungeon 式的“横向队伍推进 + 事件停顿”的节奏，但表达必须是西行车队、古道、山海异象，不复制其角色排布、战斗 UI、镜头和美术。

### 4.2 一小段路程怎么展示

玩家选路线后：

1. 自动切回地点停靠页。
2. 顶部目标变成 `车队在路上`。
3. `journey-stage` 进入 `.traveling` 状态。
4. 车队从左侧向右推进一段，背景轻微反向移动。
5. 底部主选项区暂时不显示新选择，或显示“行进中...”短状态。
6. 行进结束后进入半途路遇或抵达目的地。

首版行进时间建议 0.7-1.2 秒。太短没有路程感，太长会拖慢试玩。

### 4.3 何时中途停下进入事件

中途停下由路线数据或规则决定。视觉上要清楚表达“还没到站”：

1. 车队停在画面中段，而不是到达右端。
2. 路标显示 `起点 / 半途路遇 / 下一站`。
3. 中部视觉区加一点风沙、黑云、污染或路障。
4. 底部主选项区切到路遇事件。
5. 补给区显示“半途无法搜集城镇补给”。
6. 处理路遇后再抵达目的地，并触发地点停靠页。

半途状态必须避免误导玩家：不能让玩家以为已经到达新城市。

## 5. 主选项区状态规格

主选项区是最容易让玩家觉得“点了还卡在同一画面”的地方，必须按状态切换。

### 5.1 状态一：到达介绍

触发：进入新地点、抵达后、开局。

表现：

1. 左区标题显示地点名或事件名。
2. 左区正文显示 2-4 行到达介绍。
3. 中区显示可用补给。
4. 右区显示剧情选择。
5. HUD 目标为 `处理当前地点`。

### 5.2 状态二：补给

触发：玩家点击补给项。

表现：

1. 资源数值立即变化。
2. 补给卡显示 `已取`。
3. 最近反馈显示补给结果。
4. 不弹大窗口，不打断剧情选择。
5. 如果补给取完，中区显示简短已整备状态。

### 5.3 状态三：剧情选择

触发：地点事件或半途路遇未处理。

表现：

1. 右区显示 1-3 个主要选择。
2. 每个选择只写标签 + 一句后果暗示。
3. 选择按钮要大，适合横屏手机触摸。
4. 危险选择可用颜色或边框提示，不写长警告。

### 5.4 状态四：选择后反馈

触发：玩家完成剧情选择或路遇选择。

表现：

1. 左区标题变成 `已处理` 或事件结果。
2. 左区正文显示选择结果，不再显示原问题。
3. 右区原选择按钮消失，换成下一步。
4. 最近反馈同步写入一条日志。
5. 如果是地点事件完成，下一步为 `打开九州图`。
6. 如果是半途路遇完成，下一步为自动或手动 `继续抵达`。

这一步非常关键：玩家点了选择后，画面必须换状态，不能还停留在同一组选择上。

### 5.5 状态五：继续上路

地点事件已处理后，主选项区应明确把玩家推向九州图。

建议：

1. 右区显示一个明确按钮或提示：`打开九州图选路`。
2. HUD 目标也显示 `选择下一段路`。
3. 补给区仍可保留已取/未取状态，但不抢主动作。
4. 如果还有补给未取，可以提示“可先整备，也可直接上路”。
5. 玩家进入九州图后，路线卡成为主选择。

### 5.6 状态六：困厄 / 失败 / 结局

资源濒危或失败时，主选项区必须收窄：

1. 不允许继续普通选路。
2. 只显示补救选择或重开。
3. HUD 资源强警戒。
4. 行走图可压暗或污染化。
5. 文案直接说明当前阻塞原因。

## 6. 首版和后续版分级

### 6.1 P0 必须有

P0 是首版能玩、能理解、不会迷路的最低标准。

| 模块 | P0 规格 |
| --- | --- |
| 横屏框 | 手机横屏 16:9 可用，不出现纵向网页滚动主体验 |
| HUD | 地点、第几日、车轴、粮草、神志、当前目标、旅途/九州图切换、行囊入口可见 |
| 行囊 | 九宫格放右侧抽屉，不常驻挤压主界面 |
| 地点停靠页 | 中部行走图 + 底部左说明 / 中补给 / 右选择 |
| 页面关系 | 地点停靠页处理事件，九州地图页选择下一段路 |
| 当前目标 | HUD 和主选项区同时告诉玩家下一步 |
| 选择后反馈 | 剧情选择后按钮状态变化，显示结果和下一步 |
| 行走图 | 选路后车队推进一小段，半途能停下进入事件 |
| 九州图 | 羊皮纸底、当前/已见/可达/雾中/锁定节点、已走/可走/雾中/锁定路线 |
| 路线卡 | 显示去向、三资源消耗、风险、半途、补给预览、锁定原因 |
| 小横屏 | 地图节点、资源、主按钮可读可点，不依赖长说明 |
| 竞品边界 | 不复制星图、太空、科幻扫描、西部片马车 UI、Darkest Dungeon 具体 UI |

### 6.2 P1 有就加分

| 模块 | P1 规格 |
| --- | --- |
| 雾散城显 | 抵达新城市后有 1.2 秒内的雾散、城显、路显反馈 |
| 路线卡动效 | 新抽出的路线卡轻微进入，像路引签被摆出 |
| 地图联动 | 点路线卡时地图对应路线和节点高亮 |
| 行走气氛 | 日夜、低神志、地形影响行走图氛围 |
| 补给反馈 | 补给后资源条有轻微变化反馈 |

## 8. 验收问题

主线程验收时建议只问这些问题：

1. 横屏打开后，玩家是否 2 秒内知道当前位置、三资源和下一步？
2. 地点停靠页是否像“车队停在这里处理事情”，不是网页信息面板？
3. 剧情选择后，画面是否明确变成反馈和下一步，而不是还卡在原选择？
4. 九州地图页是否像羊皮纸远行地图，不像星图、后台流程图或现代导航？
5. 选路后是否能看到一小段行走反馈，半途事件是否清楚不是抵达？
6. 行囊九宫格是否可复查但不挤压主操作？
7. P0 不依赖 P1/P2 特效也能完整试玩吗？

## 9. 必须用户决策

本轮没有必须阻塞的用户决策。建议默认：

1. 首版固定横屏手机优先。
2. 行囊九宫格放右侧抽屉。
3. 地点停靠页和九州地图页分开，不做同屏复杂叠加。
4. 行走图使用假动态横向长图，不做实时操作。
5. P0 先保证流程清楚，再补 P1 的雾散城显和地图联动。
