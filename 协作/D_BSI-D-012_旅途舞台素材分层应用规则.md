## 0. 给 C 线程的首跑结论

本轮目标是让旅途页像“横屏手机游戏在行进”，不是做最终美术。C 线程可以先按下面这套默认组合接入：

| 层级 | 首跑默认素材 | WebDemo 相对路径 | 状态 | 接入要点 |
|---|---|---|---|---|

首跑层级顺序：

```text
BG-008 -> FG-004 -> CAR-005 -> OV-004 -> OV-008 -> UI 操作层
```

UI 安全规则：

- 所有舞台视觉层 `pointer-events: none`。
- 污染层只覆盖 `.journey-stage`，不覆盖资源、路线卡、事件文本、补给按钮、日志和行囊。
- 背景永远不带车队；车队只从 `CAR-*` 层进入。
- 绿幕车队不抠绿前不能直接贴进 Demo，否则会出现整块亮绿色背景。

## 1. 可直接映射的数据结构建议

C 线程可先按 `route.terrain || location.terrain` 取默认素材，再按当前 `locationId` 覆盖背景或地点显影层。

```js
stageAssets = {
  terrain: {
    road: { bg, fg, atmosphere, pollution, caravan },
    market: { bg, fg, atmosphere, pollution, caravan },
    water: { bg, fg, atmosphere, pollution, caravan },
    rift: { bg, fg, atmosphere, pollution, caravan }
  },
  locations: {
    central_post: { bg },
    abandoned_pass: { bg, loc },
    red_marsh: { bg, fg, pollution },
    kyushu_rift: { bg, pollution }
  }
}
```

路线行进中：

- 有 `pendingRoute` 或 `selectedRoute` 时，优先用 `route.terrain`。
- 抵达地点后，优先用 `locationId` 覆盖表。
- 半途路遇停下时，车队保持独立层，停在 30%-45% 横向位置；不要把车队烘焙进事件背景。
