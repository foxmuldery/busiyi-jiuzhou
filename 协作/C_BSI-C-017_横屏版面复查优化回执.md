# C_BSI-C-017 横屏版面复查优化回执

## 触发

用户反馈：版面还是有问题，请自行反复检查优化。

## 复查方式

1. 尝试 in-app browser 自动化，因本地插件环境异常未继续使用。
2. 改用本地 HTTP 预览，确认 `file://` 不是可靠验证方式。
3. 先错误地从 `prototype` 目录开服务，发现素材相对路径无法加载；随后改为从 `GitHub资产区` 根目录开服务，资源路径正常。
4. 使用 Chrome 无头截图检查：
   - 932 × 430 横屏旅途页
   - 932 × 430 展开当前遭遇
   - 932 × 430 九州图页
   - 844 × 390 更窄横屏旅途页
   - 844 × 390 展开当前遭遇
   - 844 × 390 九州图页

## 发现的问题

1. 底部右侧选择区过宽，左侧“地点志 / 当前遭遇”被压窄。
2. 普通横屏下地点志曾被隐藏，不符合“每到城镇先看地点志”的方向。
3. 小横屏默认显示地点图/事件图会挤压文字。
4. 当前遭遇展开时，空事件图框会被 CSS 强制显示，出现黑框。

## 已修复

1. 底部操作区改为更稳定的三列四行：
   - 左列：地点志 + 当前遭遇
   - 中列：补给
   - 右列：选择
   - 底部细条：行旅状态
2. 小横屏列宽重新分配，左侧叙事区从约 250px 提升到约 320px。
3. 地点图/事件图默认折叠隐藏，只在展开或有真实事件图时显示。
4. 修复 `hidden` 被展开态 CSS 覆盖导致的黑框。
5. 展开态确认可点击：
   - 932 × 430 下当前遭遇可从 328px 扩展到 555px。
   - 844 × 390 下当前遭遇可从 296px 扩展到 501px。

## 验证结果

- `app.js` 语法检查通过。
- `data.js` 语法检查通过。
- `git diff --check` 通过。
- 视觉资源引用检查通过：22 个引用，路径均存在。
- Chrome 截图复查通过：
  - `/private/tmp/busiyi-932x430-base.png`
  - `/private/tmp/busiyi-932x430-expanded.png`
  - `/private/tmp/busiyi-932x430-map.png`
  - `/private/tmp/busiyi-844x390-base.png`
  - `/private/tmp/busiyi-844x390-expanded.png`
  - `/private/tmp/busiyi-844x390-map.png`
- 932 × 430 与 844 × 390 下 `body.scrollWidth/scrollHeight` 均等于视口尺寸，没有整页溢出。
- 地图页 5 个节点、2 张路线卡在两种尺寸下均保持在可视区域内。

## 仍需用户肉眼复核
