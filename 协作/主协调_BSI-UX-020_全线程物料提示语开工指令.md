# 主协调交接单：BSI-UX-020 全线程物料提示语开工指令

> 项目：《不思异：九州》Web Demo  
> 日期：2026-06-19  
> 主线程目标：让 A-F 兄弟线程和当前会话子智能体同时开工，优先整理下一轮图像、视频、音频、文本物料提示语，供用户快速生成素材。  
> 当前原则：先让原型继续可玩，再逐步替换更好的物料。所有新素材默认只进入内部 Demo 候选，不得标为正式可商用。

## 1. 共同背景

当前 Web Demo 已经具备：

- 横屏手游式界面。
- 九州战略地图与迷雾显影。
- 横向旅途舞台。
- 地点志、事件弹窗、补给、路遇、结果弹窗。
- 三资源：车轴、粮草、神志。
- 基础音乐音效槽位。
- 全量试玩图可用，但不少为派生占位图，需要下一轮替换。

用户新的方向要求：

- 界面尽量简洁，参考 Out There 的数字/图标化信息表达。
- 事件、遭遇、详情和选项尽量居中弹窗显示。
- 后续每个地点、事件、路遇、危机、结局都应有对应图像或小动画。
- 遭遇图不必全部用宽屏，可以优先采用方图或 4:3 图，让弹窗视觉更集中。
- 背景长图不要带车队；车队如继续使用，必须作为前景独立资产。
- 网页生图不要要求透明背景，抠图素材优先用纯绿幕/高对比纯色背景，硬边缘、少羽化、无光晕、无投影、无运动模糊。

## 2. 本轮总任务

所有线程都要围绕“下一轮可生成物料提示语”推进，不要停留在泛泛方案。

主线程需要每个线程交付：

1. 本线程负责的物料清单。
2. 第一批优先级排序。
3. 可直接复制给 AI 工具的提示语。
4. 负面提示语或禁用清单。
5. 文件命名建议。
6. 接入或验收注意事项。
7. 需要用户决策的问题，集中列出，不要零散打断。

## 3. 分线程任务

### A 线程：规则与平衡

任务 ID：`BSI-A-010`

请输出：

- 第一章 5 分钟试玩路径中，哪些地点/事件/路遇最需要优先换图。
- 每类物料的玩法权重：主线必看、高频随机、低频惊喜、危机/失败。
- 图像是否需要直接表达资源变化，例如车轴损坏、粮草不足、神志污染。
- 为 B/D/F 提供“资源压力意象表”：车轴、粮草、神志各 8-12 个可视/可听意象。
- 不要修改代码，不要改数值表；只输出物料优先级与玩法约束。

建议交付文件：

```text
协作/A_BSI-A-010_物料优先级与资源压力意象表.md
```

### B 线程：世界观与事件文本

任务 ID：`BSI-B-020`

请输出：

- 下一轮必须配图的地点/事件/路遇/危机/结局清单。

- 每个事件弹窗对应的短诗/题记/异象文案方向。
- 神志高/中/低时，同一事件文本如何变形的 3 组样例。
- 不能直接复制《山海经》现代译文或百科表达；只保留公共领域神话意象并原创化。

建议交付文件：

```text

```

### C 线程：技术接入

- 当前 Web Demo 中哪些资产 ID 已经有槽位，哪些需要新增字段。

- 视频素材是否采用短循环、关键帧序列、还是静态图优先。

### D 线程：视觉交互与图像提示语

任务 ID：`BSI-D-030`

请输出：

- 统一美学规范 2.0：适配“弹窗方图/4:3 图 + 横向旅途长图”的版本。
- 第一批优先生图清单，至少 24 张，按 A/B/C 三组分配，方便用户用两个或三个生图线程并行。
- 每张图必须给出：资产 ID、用途、画幅、正向提示语、负面提示语、共同参考图建议。
- 必须包含方图/4:3 事件图提示语，不要全部沿用 16:9。
- 必须强调：无车队、无 UI、无文字、水印、现代物品；绿幕素材用纯绿色背景，硬边缘。

## 5. 禁止事项

- 不要把 AI 候选素材标为正式 cleared。
- 不要导入新素材到可用素材区，除非主线程验收。
- 不要删除旧素材。
- 不要复制竞品 UI、画面、文案、音效或命名。
- 不要生成带车队的背景长图。
- 不要要求透明棋盘格背景。
- 不要把本轮任务扩展到宣传片、商店页或正式发行。

## 6. 主线程验收标准

本轮不是以“想法好不好”验收，而是以“用户能不能立刻拿去生成素材”验收。

通过标准：

- 每个线程都有独立产物路径。
- D/F 至少提供可复制提示语。
- B 提供图像语义和神志文本方向。
- C 提供明确接入规格。
- A 提供优先级和玩法约束。
- E 提供可操作的风险红线。
- 所有待用户决策点集中，不零散打断。
