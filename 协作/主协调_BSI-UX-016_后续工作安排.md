## 下一轮目标

目标不是大改，而是让用户打开 Web Demo 后能顺畅完成一次：

```text
起点事件 -> 补给/选择 -> 打开九州图 -> 看资源消耗和风险 -> 选路
-> 半途路遇 -> 处理结果 -> 抵达城镇 -> 看地点志 -> 再补给/再选路
```

验收口径：

- 3-5 分钟能走完 3 到 4 段路线。
- 不因为随机事件密度太高而连续卡顿。
- 每到地点能看到短地点志和补给发现。
- 路线卡能一眼看出车轴、粮草、神志变化。
- 告急、补给、资源涨跌有清楚的视觉反馈，音频不抢文字。
