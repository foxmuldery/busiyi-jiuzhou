## 1. 资源芯片样式

资源芯片用于路线卡、选择按钮、补给按钮，目标是让玩家不读长句也能看懂资源变化。

| 项目 | 规格 |
|---|---|
| 建议 class | `.resource-delta-chip` |
| 适用位置 | 路线卡消耗行、选择卡代价行、补给卡收益行、事件结果预览 |
| 结构 | `图标 + 数字`，必要时加极短状态字 |
| 图标 | 车轴 / 粮草 / 神志沿用现有资源图标或现有资源卡图形 |
| 数字 | 增加用 `+8`，减少用 `-4`，不显示 `0` |
| 正向颜色 | 低饱和青绿，表示补给或恢复 |
| 负向颜色 | 暗朱砂红，表示消耗或损失 |
| 中性颜色 | 旧金 / 灰白，表示信息或线索 |
| 尺寸 | 桌面 24-28px 高；小横屏 20-22px 高 |
| 形状 | 小胶囊或小方角章，圆角不超过 6px |
| 背景 | 深色半透明底，细旧金边；不要纯亮色满底 |
| 字重 | 数字加粗，资源名不常驻显示 |
| 多资源排列 | 横向紧凑排列，间距 4-6px；小屏最多一行 3 枚 |
| Tooltip / title | 可保留完整说明：`车轴 -4`、`粮草 +8` |

推荐状态：

| 状态 | class | 视觉 |
|---|---|---|
| 获得 | `.gain` | 青绿数字，轻亮边 |
| 消耗 | `.loss` | 暗朱砂数字，细裂纹或暗红边 |
| 将危 | `.will-danger` | 暗朱砂边 + 小三角 / 叹号 |
| 禁用 | `.disabled` | 低透明灰，数字不高亮 |

显示例：

```text
[轴 -4] [粮 -5] [志 -2]
[粮 +8]
[轴 +6] [粮 -2]
[志 -28] [将危]
```

实现注意：

- 芯片只表达数值，不替代按钮文案。
- 路线卡的 `耗：车轴-4 粮草-4` 建议替换为芯片行。
- 选择按钮里优先显示芯片，其次显示短结果提示。
- 如果预计点击后资源低于警戒线，在芯片末尾加 `将危`，不要等结算后才提示。

## 2. 风险章样式

风险章用于路线卡、随机路遇预告、固定路遇预告和危机提示。

| 项目 | 规格 |
|---|---|
| 建议 class | `.risk-badge` |
| 适用位置 | 路线标题右侧、路线预感行、事件标签旁 |
| 结构 | `短字 + 等级 / 状态` |
| 形状 | 小朱砂章 / 龟甲裂纹章；不要大按钮样式 |
| 尺寸 | 桌面 24-28px 高；小横屏 20-22px 高 |
| 字数 | 1-3 字，避免长句 |
| 背景 | 深褐透明底 + 朱砂/旧金边 |
| 文本 | 旧金或浅米白，危险态用朱砂 |

风险章列表：

| badge | 含义 | 视觉 |
|---|---|---|
| `险低` | 低风险路线 | 旧金边，低对比 |
| `险中` | 中风险路线 | 黄褐边，轻朱砂点 |
| `险高` | 高风险路线 | 朱砂边，轻裂纹 |
| `定遇` | 固定半途路遇 | 朱砂短章 |
| `随机` | 有随机路遇池 | 墨点章 |
| `救` | 有补救候选 | 青绿小章，低调 |
| `锁` | 条件不足 | 灰暗锁章 |
| `已遇` | 该路遇已处理 | 低透明旧金章 |
| `将危` | 点击后会进入警戒 | 朱砂三角章 |

显示规则：

- 单张路线卡最多显示 3 枚风险章。
- 优先级：`锁 / 将危` > `险高` > `定遇` > `随机` > `救` > `已遇`。
- 小横屏只保留最重要 1-2 枚，其余放 title 或详情展开。
- 不再把“异象低 / 异象中 / 异象高”写成长文本，可压成 `随机` + 风险色。

## 3. 动作小章列表

动作小章用于底部选择按钮和补给按钮开头，目标是让玩家先看到“正在做什么”。

| 动作章 | 用途 | 常见按钮 | 图形建议 |
|---|---|---|---|
| `行` | 上路 / 前进 / 赶路 | 立刻上路、循路西行 | 小路标 / 车辙 |
| `取` | 搜集粮草 | 搜粮、采草、清点驿粮 | 粮袋 / 草束 |
| `修` | 修车 / 补轴 | 修轴、紧轴、垫轮 | 车轮 / 木楔 |
| `定` | 恢复神志 / 安神 | 定神、扎营、静夜 | 灯 / 香灰 |
| `问` | 问路 / 占路 | 献粟问路、问价、拓碑 | 牌 / 问路符 |
| `换` | 交易 | 旧铁换粮、粮换路引 | 天平 / 影价 |
| `避` | 绕行 / 回避风险 | 绕祠而过、卸重避云 | 断线 / 旁路 |
| `记` | 记录线索 / 古辞 | 记下鸟声、拓下反字 | 简牍 / 朱砂笔 |
| `渡` | 渡水 / 借桥 | 借甲过水、买羽寻渡 | 水纹 / 羽 |
| `焚` | 焚烧 / 祭问 | 焚壳辨烟、埋火等风停 | 火灰 |
| `歇` | 休整 / 点名 | 重新点名、短暂停车 | 帐点 / 名册 |
| `终` | 结局 / 结束选择 | 归返、入裂隙 | 裂纹 / 卷轴合拢 |

样式规格：

| 项目 | 规格 |
|---|---|
| 建议 class | `.action-mark` |
| 尺寸 | 桌面 24px；小横屏 20px |
| 形状 | 小方章或圆章，圆角不超过 6px |
| 字体 | 用当前 UI 字体即可；不要换书法字 |
| 颜色 | 默认旧金；危险动作朱砂；补给动作青绿点缀 |
| 位置 | 按钮左侧第一元素，后接动作名 |
| 小横屏 | 允许只显示动作章 + 资源芯片，长提示收起 |

按钮结构建议：

```text
[修] 停车紧轴     [轴 +7] [粮 -3]
[取] 采草煮汤     [粮 +10] [志 -3]
[行] 立刻上路     [轴 -4] [粮 -5]
[问] 拓下反字     [志 -4] [线索 +8]
```

禁用态：

- 动作章降低透明度。
- 资源芯片保持可读，但不高亮。
- 禁用原因保留小字，不占主视觉。

## 4. 三档注意点

### 4.1 844x390

这是最紧的小横屏规格，只保留最必要信息。

| 区域 | 注意点 |
|---|---|
| 资源芯片 | 高度 20-22px；每行最多 3 枚；资源名可缩成 `轴 / 粮 / 志` |
| 风险章 | 每张路线卡最多 2 枚，优先 `锁 / 将危 / 险高 / 定遇` |
| 动作小章 | 必须显示，但按钮正文最多 1 行；长结果提示折叠 |
| 路线卡 | 只显示地点名、芯片、风险章；诗句/长 hint 隐藏或进展开 |
| 补给按钮 | 显示 `动作章 + 动作名 + 芯片`，不要显示长解释 |
| 事件区 | 保持当前“志 / 遇”折叠思路，但按钮要有可展开状态 |
| 字号 | 不低于 12px；数字不低于 14px |

### 4.2 932x430

这是首版手机横屏的主要适配尺寸。

| 区域 | 注意点 |
|---|---|
| 资源芯片 | 可显示完整 `轴 / 粮 / 志`，芯片高度 22-24px |
| 风险章 | 每张路线卡最多 3 枚；`救` 可以显示 |
| 动作小章 | 保留动作章 + 两行按钮文案；资源芯片放第二行 |
| 路线卡 | 可显示短 hint，但不要和芯片同一行挤压 |
| 补给按钮 | 一行显示动作，第二行显示芯片 |
| 事件区 | 行动过程条可以显示短过程文案，例如 `搜集中 / 清点中 / 入车` |
| 字号 | 按钮名 13-14px，芯片数字 14px 左右 |

### 4.3 1280x720

桌面与大平板规格，目标是减少网页感。

| 区域 | 注意点 |
|---|---|
| 资源芯片 | 可用 24-28px 高，图标更清楚；但不要变成大装饰 |
| 风险章 | 允许 3 枚稳定显示；详情仍不要堆长句 |
| 动作小章 | 按钮可显示动作章、动作名、资源芯片、短提示 |
| 路线卡 | 仍以芯片和风险章为主，长说明降级为小字 |
| 补给按钮 | 图标动作按钮可更舒展，但不做卡中卡 |
| 事件区 | 可显示事件小图，但不把事件图做成整屏背景 |
| 字号 | 保持工具界面克制，不使用过大标题压迫舞台 |

## 5. 给 C 的最小落地顺序

1. 先做 `.resource-delta-chip`，替换路线卡和选择按钮里的资源变化文字。
2. 再做 `.risk-badge`，压缩路线风险和路遇预告。
3. 最后做 `.action-mark`，给补给和选择按钮加动作小章。
4. 三档适配先保证 844x390 不溢出，再微调 932x430 和 1280x720。
