# D_BSI-D-014 试玩版横屏界面与状态反馈复核

> 子线程：D，视觉体验与交互氛围设计  
> 更新时间：2026-06-16  
> 范围：横屏手游感、信息密度、状态反馈、路线卡 / 资源卡 / 底部选择区图标化建议。  
> 约束：不要求整体重做，不做宣传视觉，不新增大视频依赖。

## 结论

当前 Web Demo 已经从“普通网页信息面板”推进到“可试玩横屏框架”：固定 16:9 画幅、顶部 HUD、横向舞台、九州图切换、底部行动区、资源涨跌闪光、飘字、告急浮层和行动过程条都具备首版手游感基础。

最影响体验的不是大美术缺口，而是“动作前的风险读法”仍偏文字：路线消耗、补给收益、选择后果、资源告急大多要读短句才能判断。下一轮建议只做小修，把路线、资源、底部选择区统一成“图标 + 数字 + 状态色”的读法，让玩家在 1 秒内知道该不该点。

## 可直接进入 Demo 的内容

- C 线程已实现的资源涨跌闪光、资源飘字、资源首次跌破警戒线舞台浮层，可以继续进入 Demo。
- 当前横向山海旅途舞台、车队层、地点名、路线节点、底部事件 / 补给 / 选择区，已能支撑第一章 5 分钟试玩路径。
- 紧凑横屏下把地点志、当前遭遇收进小按钮的方向是对的，可以保留；但按钮需要更像“可展开信息入口”，不能像两个孤立字。
- 行动过程条已能缓冲“点一下立即结算”的生硬感，可以继续作为补给、上路、路遇、抵达的通用反馈层。

## 最影响手游感的问题

1. 路线卡仍然是“读说明”而不是“看风险”。`耗：车轴-4 粮草-4` 这类信息应转成资源小图标和红色负数，风险等级应转成朱砂章 / 裂纹章。
2. 资源卡的即时反馈已经有了，但常驻风险不够一眼读懂。闪光和飘字是结算后反馈，玩家点之前还需要看见“将告急 / 已告急”。
3. 底部选择区的动作参与感仍偏弱。行动过程条改善了生硬感，但选择按钮本身缺少“我正在做什么”的小过程提示，例如取粮、修轴、辨路、绕行。
4. 紧凑横屏下“志 / 遇”入口过于抽象。它节省空间，但新玩家可能不知道这里藏着地点志和当前事件文本。
5. 地图路线、舞台路线、底部行动三套信息还没有完全合并成一个视觉语法。玩家能玩，但会觉得自己在三个面板之间找信息，而不是一路推进。

## 路线卡 / 资源卡 / 底部选择区图标化建议

### 路线卡

- 路线标题左侧固定放“路标”图标，锁定路线放雾 / 锁图标。
- 消耗改成资源芯片：车轴、粮草、神志分别用小图标 + `-4`，负数用暗红，收益用青绿，0 不显示。
- 风险改成 `险 1-5` 的朱砂小章；3 以上加裂纹边，不再依赖长句解释。
- 路遇类型只保留一个短徽记：定、随、救、已遇；详细说明放 hover / 展开，不占主卡高度。
- 目的地有可补给时，在目的地名旁显示 `可补` + 对应资源小图标。

### 资源卡

- 数字仍保留为最大信息，但资源名旁加常驻图标：轴、粮、志三类一眼分清。
- 增加三档常驻状态：安、紧、危。正常低调；紧用黄边；危用朱砂边和轻微脉冲。
- 当选择或路线会让资源跌入警戒线时，在按钮上预显示“将危”，不要等结算后才告警。
- 飘字位置应靠近对应资源卡或行动发生区，不要漂到舞台重要视觉中心。

### 底部选择区

- 每个选择按钮开头加动作图标：行、取、修、问、避、记、险、终。
- 卡片正文拆成三段：动作名、资源变化芯片、短结果提示；不要让玩家在一句话里找代价。
- 补给按钮保持小图标方向，但建议只显示“一字动作 + 资源变化”，例如 `取粮 +8`、`修轴 +6 -粮2`。
- 禁用按钮用锁 / 雾 / 断线图标表达原因，文字原因保留为次级小字。

## 需要 C 线程实现的内容

以下为小修优先级，不超过 8 条，均不要求重构：

1. 建一个轻量 `resource-delta-chip` 样式，用于路线卡、选择卡、补给卡统一显示车轴 / 粮草 / 神志涨跌。
2. 给资源卡增加 `normal / warn / danger` 常驻状态样式，保留现有闪光和飘字作为瞬时反馈。
3. 路线卡把 `耗：...` 改成资源芯片行，风险改成 `险` 章，路遇徽记保留一行最多 3 个。
4. 底部选择按钮按动作类型加小图标或一字章，优先覆盖上路、补给、修整、询问、绕行、终局。
5. 在路线 / 选择点击前计算预计资源，如果会跌破警戒线，在对应卡上显示 `将危` 小标。
6. 紧凑横屏下把“志 / 遇”按钮改成带图标的圆角小按钮，并加可展开状态，不增加默认文字占屏。
7. 行动过程条与按钮联动：点击补给时显示“搜集中 / 清点中 / 入车”，点击上路时显示“束车 / 辨路 / 起辕”，用短文案替代通用等待感。
8. 地图页路线列表在 844x390 / 932x430 下限制单卡高度，超出时只滚路线列表，不挤压地图主体。

## 暂缓建议

- 暂缓整体 UI 重做；当前问题可以用图标化、状态色和层级压缩解决。
- 暂缓为每个事件加大图或短片；第一章核心仍是路线、资源和文字选择。
- 暂缓增加新系统入口，例如完整背包、人物面板、复杂成就页。
- 暂缓继续扩展装饰性边框和大面积纹理，先保证小横屏信息清楚。

## 风险 / 待确认

- 当前素材和字体授权不在本任务复核范围内，仍按 E 线程口径处理。
