# B_BSI-B-012 第一章随机事件扩展池

> 子线程：B，山海经世界观与叙事主创  
> 任务：BSI-B-012  
> 范围：在现有第一章地点事件、半途路遇和半随机显影基础上，补充“有趣随机事件”设计池  
> 边界：只写协作文档，不修改 C 代码，不改 A/C/D/E/F 交付文件，不决定最终数值公式，不复制现代译文、百科说明或商业设定

## 0. 本轮结论

随机事件应成为《不思异：九州》第一章的主要乐趣来源之一：玩家不是只在地点读固定事件，而是在每段横向旅途中不断遇见小异象、小交易、小误读和小补救。它们要让同一条路线每次都有一点不同，同时不把玩家随机压死。

首版建议先额外接入 8 个随机事件，作为 `routeEvents` 的补充池；后续再扩到 24 到 36 个。事件重点不是“扣资源”，而是让玩家每次都觉得自己刚刚处理了一个具体的山海小故事。

## 1. 随机事件设计口径

| 原则 | 说明 |
| --- | --- |
| 每个事件必须有一个画面 | 玩家要能记住“天上鱼雨”“路碑倒着走”“粮袋自己变重”这类瞬间。 |
| 每个事件必须有一个选择张力 | 不是单纯奖励或惩罚，至少在粮、车轴、神志、线索之间做交换。 |
| 低神志可以疯，但不能不可玩 | 文案可以不可信，但选项仍要保留主要风险方向。 |
| 随机不阻断主线 | 关键路线线索至少有两个来源，不能只靠某个随机事件。 |
| 连续坏运气要被托住 | 连续风险后，事件池应偏向补给、喘息、线索或低代价交易。 |
| 事件要适配横向旅途舞台 | 多发生在“抵达前、路途中、抵达后短停”，不是只在地点弹文字。 |
| 每个事件必须有山海经锚点 | 优先对应异兽、异国、山川、水泽、巫祝、天象、草木物产；不做无出处的泛怪谈。 |

## 5. 触发时机建议

| 时间点 | 适合事件 | 玩家体验 | C 线程接入备注 |
| --- | --- | --- | --- |
| 抵达地点前 | 雷泽云缝、巫咸反里、白羽渡价、半截骨桥 | 玩家还没到，就先感到目的地“在影响路” | 可挂在路线 `midEvent` 或后续随机 `midEventPool` |
| 路途中 | 鸣蛇入轴、祝余苦汤、泥咬轮、倒芦照人 | 横向舞台停一下，处理小选择，再继续走 | 最适合进入 `routeEvents` |
| 抵达后短停 | 巫咸点名、黑齿影价、灰中粟、墨又湿 | 抵达反馈后给一个小插曲，不抢主事件 | 可作为地点事件处理后、开图前的低概率插入 |
| 危机保护时 | 祝余苦汤、温石垫轮、巫咸点名、狐灯小市 | 玩家快死时得到“有代价的救命选择” | 需要 `rescue_candidate` 标签 |
| 低神志专属 | 夸父复迹、烛龙息图、皮下缺笔、死路归来 | 世界开始不可信，但选择仍有信息 | 需要 `sanityMax` 条件 |

## 6. 低神志随机事件表现

低神志不是“把信息藏起来”，而是让同一条信息换一种不可靠的外观。

| 正常信息 | 低神志外观 | 必须保留的可玩信息 |
| --- | --- | --- |
| 车轴有风险 | 鸣蛇入轴、轴木发热、轮影分裂 | 选项 hint 仍标明车轴变化方向 |
| 粮草可补 | 粮袋呼吸、祝余梦中发苦、赤粟倒长 | 仍告诉玩家是补粮还是耗粮 |
| 路线有线索 | 巫咸反里、夸父复迹、烛龙息图 | 仍给“可能得线索/路线提示” |
| 神志可恢复 | 点名、闭眼、献粟、休整 | 仍明确这是稳心神的选择 |
| 高风险捷径 | 云缝、骨桥、死路归来 | 仍标出会伤神志或车轴 |

## 7. 给 A 线程的风险标签

| 标签 | 用途 |
| --- | --- |
| `pool_risk` | 一般风险事件，总净损耗可控。 |
| `pool_supply` | 补给事件，可在粮低时加权。 |
| `pool_rest` | 喘息事件，可在神志低或连续坏事件后加权。 |
| `pool_clue` | 线索事件，可降低坏运气计数。 |
| `pool_item` | 器物/flag 事件，给后续路线或结局条件。 |
| `pool_trade` | 交易事件，适合资源交换。 |
| `pool_sanity` | 神志污染事件，不能连续高密度出现。 |
| `rescue_candidate` | 可作为坏运气保底候选。 |
| `low_sanity_only` | 只在神志低于阈值时抽取。 |
| `avoid_after_crisis` | 濒死补救后两步内避免抽取。 |
| `positive_tag` | 玩家至少获得补给、修复、神志恢复、线索或 flag。 |

## 8. 给 C 线程的数据字段建议

现有 `routeEvents` 已能承载首版随机事件。若要后续做真正随机池，可在不破坏现有结构的基础上增加外层元信息。

```js
randomRouteEvents: {
  rnd_bitter_grass_soup: {
    title: "祝余苦汤",
    tag: "路遇",
    pool: ["supply", "rest"],
    shjAnchor: {
      name: "祝余",
      type: "草木物产",
      mode: "直名引用"
    },
    risk: "low",
    routeTags: ["plain", "marsh", "low_pressure"],
    trigger: {
      phase: "mid_route",
      grainMax: 55,
      badLuckMin: 35
    },
    weight: 18,
    cooldown: 2,
    positiveTag: true,
    rescueCandidate: true,
    texts: {
      clear: "路旁祝余草煮后能压饥，只是汤色黑得像夜路。",
      uneasy: "苦味先在梦里醒来。草汤还没煮，粮袋已经轻了一点。",
      mad: "你喝下路。路在肚里发苦，仍替你们顶住一程饥。"
    },
    choices: [
      {
        label: "采草煮汤",
        hint: "粮草 +10，神志 -3",
        effect: { axle: 0, grain: 10, sanity: -3, badLuck: -8 },
        result: "苦汤入腹，队伍多撑了一程。夜里有人梦见草根在说路。"
      },
      {
        label: "只取草籽",
        hint: "粮草 +5，神志 +2",
        effect: { axle: 0, grain: 5, sanity: 2, badLuck: -5 },
        result: "草籽少，却干净。随从把剩下的苦味留在路边。"
      }
    ]
  }
}
```

首版若不想新增 `randomRouteEvents`，也可以先把 8 个优先事件直接合入 `routeEvents`，再由路线 `midEvent` 固定或半随机引用。

## 9. 随机抽取节奏建议

| 阶段 | 推荐频率 | 推荐池 |
| --- | --- | --- |
| 前 3 分钟教学 | 每 1 到 2 段路 1 次 | `supply`、`clue`、`rest`、低强度 `risk` |
| 中段转折 | 每段主路可有 1 次 | `risk`、`trade`、`sanity`、`item` |
| 高压后段 | 保持密度但提高保底 | `sanity`、`clue`、高压 `risk`，穿插 `rest` |
| 资源危急 | 强制补救或交换 | `rescue_candidate`、`supply`、`rest`、`trade` |
| 低神志 | 少量专属事件 | `low_sanity_only`，但不能连续 2 次以上 |

## 10. 版权与风格自检

- 本文件所有事件均有《山海经》锚点：异兽、异国、山川、水泽、巫祝、天象、草木物产或叙事母题。
- 本文件只借用公共古典神怪语汇和项目自有设定，不复制现代《山海经》译文、百科条目、插画设定或竞品事件句式。
- 所有事件均为 B 线程原创短结构，后续可继续压缩为 Demo 文案。
- 不把随机事件做成战斗、职业、卡牌构筑或载具驾驶，以免偏离《不思异：九州》的“远行、资源、神志、路途异闻”核心。
- 低神志文本保持可玩信息，不让玩家因纯文本欺骗失去判断。
