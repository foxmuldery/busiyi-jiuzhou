## 1. 一句话视觉方向

《不思异：九州》的 Web Demo 第一屏不是横版动作游戏，也不是普通网页地图，而是：

```text
假动态横屏长图 / 横向山海旅途舞台
```

玩家看到的是一段从左到右展开的小路程：前景有辇车、车队、草木、路面；后景有山影、云气、日月、异象、远处地貌；真实玩法仍是看资源、选路线、读事件、做选择。

界面关键词：

- 帛书
- 竹简
- 龟甲
- 朱砂
- 墨迹
- 山海远行
- 旧地图
- 神志污染
- 异闻档案

使用原则：这些关键词只能克制使用，不能堆成泛古风装饰。所有纹理、边框、雾气、裂纹都必须让位于资源、路线、事件文本和按钮可读性。

## 2. 当前已定产品目标

本阶段目标不是做最终美术，而是让 Web Demo 在 3-5 分钟试玩里成立：

1. 玩家知道现在在哪里。
2. 玩家知道还能去哪里。
3. 玩家知道去那里消耗什么。
4. 玩家知道车轴、粮草、神志是否危险。
5. 玩家能读到当前事件并做选择。
6. 每个动作有一点过程感，不是点完硬切结果。
7. 低神志时，世界本身开始不可信，而不是只改文字。

## 3. 第一屏信息架构

推荐结构：

```text
┌──────────────────────────────────────────────┐
│ 顶部 HUD：地点 / 日数 / 三资源 / 旅途进度 / 功能入口 │
├──────────────────────────────────────────────┤
│ 横向山海旅途舞台                              │
│ BG 后景：远山、荒泽、城关、云气、异兽远影       │
│ LOC 地点：废关、渡口、边市、裂隙等显影层        │
│ FG 前景：草木、路面、车辙、石块                 │
│ CAR 车队：辇车/车队独立层                      │
│ OV 污染：幻路、第二月亮、远山重影、赤水幻纹      │
├──────────────────────────────────────────────┤
│ 底部行动区：事件摘要 / 补给 / 路线或选择按钮 / 日志 │
└──────────────────────────────────────────────┘
```

第一版 UI 优先级：

```text
资源是否危险 -> 当前地点 -> 可选路线 -> 当前事件 -> 行动选择 -> 结果日志
```

移动横屏优先级：

```text
资源状态 -> 舞台视觉 -> 当前遭遇 -> 可点按钮
```

## 4. 舞台分层规则

固定层级：

```text
BG 背景长图
LOC 地点显影层
FG 前景视差层
CAR 车队层
OV 氛围/神志污染层
UI 操作层
```

严禁：

- 把车队烘焙进背景。
- 把资源条、事件文本、按钮、日志放进污染层。
- 让雾气、裂纹、地图纹理遮挡正文。
- 把横向舞台做成可操控角色动作。
- 为了展示整张长图牺牲小屏可读性。

允许的假动态：

- 云气缓慢横移。
- 前景轻微视差。
- 车队轻微起伏或位置推进。
- 抵达地点时 LOC 层淡入。
- 低神志时 OV 层逐渐增强。
- 行动时出现短过程：搜集中、清点中、束车、辨路、起辕。

不建议第一版做：

- 大段视频背景。
- 每个事件独立短片。
- 大规模 3D。
- 完整过场动画。
- 每条路线独立最终插画。

## 6. 已定字体方向

目标：有古韵、清晰、可辨认、商用风险低。

推荐组合：

| 用途 | 字体 |
|---|---|
| 正文 / UI 数字 / 长文本 | 思源宋体 SC |
| 地点名 / 少量标题 | 霞鹜文楷 |
| 片名副标题 / 章节名 | 站酷小薇体 |
| 极少量印章/题字点缀 | 马善政毛笔楷体 / 龙藏体 |

规则：

- 正文不能用难认书法字。
- UI 数字必须清楚，不要为了古韵牺牲可读性。
- 点缀字体只用于少量标题、印章、地点显影，不进入按钮正文。
- 字体授权必须按 E 线程规则记录，不要口头声称可商用。

## 7. 当前 UI 需要继续改善的核心问题

来自 D_BSI-D-014 复核：

1. 路线卡仍偏文字说明，应该改成图标 + 消耗芯片 + 风险章。
2. 资源卡有闪光和飘字，但常驻风险不够一眼懂。
3. 底部选择按钮缺少动作图标，参与感不足。
4. 小横屏下“志 / 遇”入口太抽象，需要更像可展开信息按钮。
5. 地图路线、舞台路线、底部行动还没有统一视觉语法。

新 agent 如果继续做 UI 资源，应优先补：

- 资源涨跌小芯片。
- 路线风险章。
- 行动类别小图标。
- 地图节点状态。
- 小横屏按钮背景。
- 行动过程提示条。

不要优先做：

- 大宣传主视觉。
- 商店封面。
- 大量结局图。
- 每个地点完整插画。

## 8. A/B/C 三组生成工作流

用户明确希望按 A、B、C 三组提示语并行生成，方便三个线程同时跑图。

后续所有图像提示语建议固定为：

```text
A 组：大背景 / 横向长图 / 地貌舞台
B 组：可拆层 / 前景 / 地点显影 / 神志污染层
C 组：UI 元件 / 车队 / 节点 / 资源图标 / 菜单按钮
```

### A 组职责

生成完整横向舞台背景，通常是 16:9 或更宽。

用途：

- 主舞台 BG。
- 地点背景。
- 大地图背景。
- 菜单或结局整图背景。

要求：

- 不带车队。
- 不带文字。
- 不带 UI。
- 中下部留出道路或地平面。
- 横向构图，适合裁切到 844x390、932x430、1280x720。

### B 组职责

生成可叠加的分层素材。

用途：

- 前景草木、路面、车辙。
- 雾气、云影、风沙。
- 低神志污染。
- 地点显影。
- 天空层、日月层、第二月亮。

要求：

- 背景简单或纯色，方便抠图/混合。
- 不要复杂主体挤满画面。
- 不要文字。
- 不要按钮。
- 叠加后不能遮 UI。

### C 组职责

生成可直接切入 UI 的小资源。

用途：

- 车队透明 PNG / 绿幕源。
- 资源图标。
- 路线节点。
- 风险章。
- 选择按钮底纹。
- 主菜单按钮。
- 地图标记。

要求：

- 单体清楚。
- 边缘干净。
- 纯绿色背景或透明背景。
- 不要投影、不要发光、不要大面积羽化。
- 尽量同一张图内多变体拼版，便于切片。

## 12. 前景与后景不搭的修正原则

用户已经指出过“前景和后景的图不搭”。后续生成或筛选要注意：

1. 光源一致：太阳/月亮方向、画面明暗不能冲突。
2. 线条密度一致：背景细密，前景不能突然写实厚涂；背景淡雅，前景不能太黑。
3. 透视一致：横向舞台是侧向/平视远行，不要把前景做成强俯视。
4. 颗粒一致：都要像旧帛书/矿物颜料，不要一个像照片、一个像插画。
5. 色温一致：赤水路段前景要偏湿冷/赤褐；废关路段前景要偏干土/暮色。
6. 车队落点一致：车轮必须压在路面或浅滩上，不能漂浮。

快速修正手段：

- 给前景层加统一旧纸噪点。
- 降低前景饱和度和对比度。
- 用 multiply/overlay 混合前景，不要整张硬贴。
- 背景和前景都加同一层轻薄云雾。
- 车队底部加极浅接触阴影，但不要烘焙大投影。

## 13. 动作过程感规范

用户明确要求：每个动作都应该有一个小过程，尤其补充资源。

所有补给/上路/选择建议分三步：

```text
点击 -> 过程提示 -> 结算反馈
```

例子：

| 动作 | 过程文案 | 结算反馈 |
|---|---|---|
| 搜粮 | `搜集中`、`清点中`、`入车` | `粮草 +8` |
| 修轴 | `卸辕`、`缚木`、`试轮` | `车轴 +6 / 粮草 -2` |
| 定神 | `闭目`、`焚草`、`记名` | `神志 +5` |
| 上路 | `束车`、`辨路`、`起辕` | `车轴 -3 / 粮草 -5` |
| 绕行 | `探路`、`避泽`、`绕远` | `粮草 -4 / 风险降低` |
| 问路 | `献物`、`询问`、`记号` | `路线显影` |

视觉表达：

- 行动按钮按下后短暂锁定，避免重复点击。
- 舞台上可以出现一条轻微进度纹，不要做大 loading。
- 对应资源卡闪光 + 飘字。
- 如果资源将告急，要在点击前预告，不只在结算后警告。
