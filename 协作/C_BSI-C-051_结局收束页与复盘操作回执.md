# C_BSI-C-051 结局收束页与复盘操作回执

日期：2026-06-18  
子线程：C 技术原型  
目标：把抵达终点后的“空收束”改成可复盘、可重开的试玩闭环。

## 1. 问题

C-050 已验证小横屏真实点击路径可以抵达 `九州裂隙`，但终点事件选择后，主选项区缺少明确的后续操作。玩家会知道自己到达了终点，却不容易完成“看结局 -> 复盘 -> 重开”的完整闭环。

这个问题会影响 P0 的“完整可玩”判断，因为短局生存探索游戏的结局/失败必须能复盘，而不是只停在一段结果文字。

## 2. 本轮改动

- 新增统一 `restartRun()`，重开时清理路线、抽屉、诗句 fallback、音频告警和视图状态。
- 新增结局标题、结局说明和本局摘要函数。
- 结局页改为两个明确操作：
  - `重开一局`
  - `查看日志`
- 结局文本追加本局摘要：天数、车轴、粮草、神志、见闻数、结局收集数。
- 失败页重开逻辑改为复用同一个 `restartRun()`，减少两套重置状态不同步的风险。
- QA 脚本新增 `ending recap action contract`，防止结局页重新退化为空选项区。
