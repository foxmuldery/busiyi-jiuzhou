# C_BSI-C-013 横屏界面返工小修回执

> 子线程：C，Web Demo 技术原型  
> 日期：2026-06-15  
> 触发：用户反馈“界面还是有问题”  
> 结论：已做三轮横屏手游感小修；真实截图仍受浏览器权限/额度阻断，需用户侧或权限恢复后复核。

## 1. 判断

当前问题不是玩法逻辑，而是小横屏下信息层级还像网页：

- 舞台、地点志、补给、事件和选项都存在，但底部操作区太矮。
- 844x390 一类尺寸下，旧规则会直接隐藏地点志，削弱《山海经》亮点。
- 选项和补给像网页卡片堆叠，缺少横屏手游一屏操作感。
- 地图页低高度时曾变成“上地图 / 下路线”堆叠，战略选择感不足，路线卡也不好点。
- 文字内容的摆放仍偏网页，应改为手机游戏常见的默认摘要、点按展开。

## 2. 已修改

- [styles.css](/Users/yuanzhe/Documents/game/GitHub资产区/03_WebDemo/prototype/styles.css)

具体改动：

- 小横屏底部操作区从 `112px / 34%` 逐步调整为 `148px / 42%`。
- 底部操作区固定为三栏：左侧地点志与事件文本，中间补给，右侧剧情选择。
- 小横屏不再隐藏地点志，改为两行压缩显示。
- 小横屏隐藏事件诗引，把有限空间让给地点志、事件正文和选择。
- 选项提示、补给提示加入两到三行截断，避免按钮互相挤压。
- 已处理事件后的“打开九州图”提示在小横屏中改为紧凑网格。
- 小横屏地图页从上下堆叠改为左侧九州图、右侧路线栏，保留“下一站”和半途提示。
- 小横屏路线栏从横向滚动卡片改为纵向滚动卡片，更接近战略选择面板。
- 补充 `--serif-font` 变量，避免诗引字体变量缺失时回退不可控。
- 地点志加入“展开 / 收起”按钮，默认两行摘要，展开后在原面板内滚动查看完整文本。
- 事件正文加入“当前遭遇”信息块和“展开 / 收起”按钮，默认隐藏事件诗引，只显示三行事件摘要。
- 切换路线、抵达新地点、进入半途路遇、重开时会自动恢复折叠状态。

如果用户仍觉得界面不像手机游戏，下一刀应从视觉结构入手：

- HUD 再减薄，隐藏标题副信息。

- 底部操作区做成更像手游的固定命令栏，而不是传统面板。
