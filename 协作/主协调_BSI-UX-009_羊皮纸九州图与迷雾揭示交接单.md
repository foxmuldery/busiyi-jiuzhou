# BSI-UX-009 羊皮纸九州图与迷雾揭示交接单

项目：不思异：九州  
主协调：Codex 主线程  
状态：交给 A/D/C 协同；主线程整理、校验、沟通  
日期：2026-06-14

## 一、用户新增意见

战略选择参考《Out There》的“大界面”路线决策感，但《不思异：九州》要用羊皮纸地图 / 山海舆图的调性。

探索方式：

1. 一开始远方只有节点、阴影或雾中标记。
2. 到达一个新的城市后，迷雾散开，该城市显现。
3. 只有当前城市后续可走的路线才显现出来。
4. 这种揭示可以服务随机性和抽卡。

## 二、核心判断

这不是普通“地图页美化”，而是战略层玩法和表达的核心。

推荐定义：

```text
羊皮纸九州图 = Out There 式路线决策压力 + 山海舆图视觉 + 迷雾揭示 + 可控随机路线池
```

目标体验：

- 玩家像在看一张逐步被描开的古地图。
- 当前所在城市是明确锚点。
- 未到达的远方不是完整可见地图，而是雾中节点、模糊地名、残缺道路。
- 到达新城市后，地图“显影”：该城、周边道路、可走路线、部分资源线索出现。
- 玩家每次选择路线，都像从一组抽出的前路里做资源和风险判断。

## 三、视觉方向

### 3.1 羊皮纸山海舆图

视觉基调：

- 羊皮纸 / 帛书 / 旧舆图。
- 墨线山川、淡色地形、朱砂路线、印记节点。
- 雾区用半透明纸雾、墨晕、残缺纹理表现。
- 城市显现时像墨迹晕开、朱印落下、地名被写出。

避免：

- 不做太空星图。
- 不做现代导航地图。
- 不复制 Out There 的星球节点、连线、图标和 UI 表达。
- 不把地图做成复杂 RTS 战略图。

### 3.2 节点状态

建议第一版用 5 种状态：

| 状态 | 含义 | 表现 |
|---|---|---|
| 当前 | 车队所在城市 | 朱印 / 车队小标 / 清晰地名 |
| 已见 | 已到达或已显现城市 | 清晰墨线、可读地名 |
| 可达 | 当前城市后续可走节点 | 半亮节点、路线线条显现、可点 |
| 雾中 | 远方未揭示节点 | 模糊点 / 无名影 / 只见轮廓 |
| 锁定 | 条件不足路线 | 断线 / 红墨封记 / 简短锁定原因 |

### 3.3 路线状态

建议第一版用 4 种路线：

- 已走路线：淡墨或旧车辙。
- 可走路线：朱砂线 / 金墨线，带消耗和风险卡。
- 雾中路线：若隐若现，只提示“西北有路”“泽中有影”。
- 锁定路线：断裂线或封印线，显示条件，如“需白羽语片”“需神志低于 35”。

## 四、玩法揭示规则

### 4.1 最小可运行揭示逻辑

第一版不需要完整程序生成世界，先做“可控随机 + 显影规则”。

基本规则：

1. 开局只显示当前城市、1-2 个可达方向，以及更远处的雾中节点。
2. 选择路线并抵达城市后：
   - 当前城市变为已见。
   - 新城市变为当前。
   - 从新城市出发的候选路线显现。
   - 远方新增 1-3 个雾中节点或道路影子。
3. 玩家不能提前看到全地图完整路线。
4. 路线卡仍显示 Out There 式决策信息：目的地、消耗、风险、半途路遇、可观察补给、锁定原因。

### 4.2 随机性和抽卡

可以把“后续路线显现”理解为抽卡：

- 地点池：山隘、泽国、墟市、祭所、禁地、异族边境、裂隙前哨。
- 路线池：古道、河渡、荒泽、山脊、幻路、暗市小径。
- 事件池：路遇、地点事件、资源事件、神志异象。
- 资源池：偏车轴、偏粮草、偏神志、混合补给、无补给高风险。

但抽卡必须有保底：

- 至少保留一条能继续推进主路径的路线。
- 不能连续多次只抽高消耗路线。
- 不能让当前资源状态必死，除非玩家此前已经多次高风险选择。
- 每次显现的路线至少有一个“可理解的取舍”：低消耗高神志风险 / 高车轴消耗但补粮 / 锁定高收益路线等。

## 五、与当前 Web Demo 的关系

当前已有：

- `locations[].map`：地点坐标和 step。
- `routes[]`：from/to/cost/risk/midEvent/lockedHint。
- `state.discoveredLocations`：已发现地点。
- `state.traveledRoutes`：已走路线。
- `renderMap()`：地图节点和路线线条。
- `renderRoutes()`：路线卡。

因此第一版不需要推翻现有结构，只需要：

1. 修改地图显隐逻辑：未发现远方不完整展示。
2. 增加“雾中节点/雾中道路”表现。
3. 抵达新地点后，将后续路线和雾中远方逐步加入可见范围。
4. 后续随机/抽卡先由数据规则模拟，不必立刻做大型程序生成。

## 九、主协调验收标准

- 九州图第一眼像羊皮纸山海舆图，而不是太空星图或网页地图。
- 玩家只看到当前已知区域和可走前路，不会提前看完整世界。
- 到达新城市后，有“雾散、城显、路显”的反馈。
- 路线卡仍能完成 Out There 式资源/风险决策。
- 随机路线和抽卡有保底，不产生直接死局。
- 不复制竞品的 UI、美术、命名、图标、宣传表述。

## 十、暂不需要用户拍板

先按以下默认推进：

- 战略地图采用羊皮纸山海舆图。
- 地图揭示采用“已见 / 可达 / 雾中 / 锁定”四层以上状态。
- 随机性先用可控地点/路线/事件池模拟，不立刻做复杂程序生成。
