# 《山野逸事》第三部志怪内容协作者资料交接包

日期：2026-06-24

适用对象：《山野逸事》第三部部分内容策划 / 志怪内容协作者 / 旧项目设定交接人  
接入项目：《不思异：九州》

如果接收人还不熟悉《不思异：九州》，请先看：

- `协作/山野逸事内容策划项目交接说明_给不熟悉不思异九州的协作者.md`

## 1. 这份文档解决什么问题

这不是让《山野逸事》志怪内容协作者来填游戏素材表，也不是让他学习游戏策划格式。

这位协作者只负责自己掌握的《山野逸事》第三部部分内容，不是《不思异：九州》的主策划，也不是整个旧项目资料库的唯一负责人。

内容协作者最有价值的工作是判断和补充：

- 哪些旧设定有灵魂，值得进入《不思异：九州》。
- 哪些意象、人物关系、禁忌和句子不能丢。
- 这些素材在世界观里是真相、谣言、风俗、骗局，还是古国遗痕。
- 玩家在这里为什么会被诱惑，为什么会害怕。
- 低神志时，这个素材应该怎样变得更恐怖、更荒诞或更不可信。
- 旧项目里哪些情节、典故来源、人物因果不能被 AI 改错。

AI 负责读资料、拆候选、追问、整理和转成游戏侧可用的素材包。内容协作者只做自己负责范围内的高价值判断和补充，不负责《不思异：九州》的整体玩法策划。

## 2. 一次交接的最小单位

每次只交一小批，避免一开始就做大规模迁移。

推荐第一批：

- 5 到 10 页旧资料；或
- 3 到 5 个旧设定；或
- 1 个地方 + 2 个怪事 + 1 个人物关系；或
- 1 集未拍成故事的完整设定。

资料可以是：

- 剧集大纲。
- 分集梗概。
- 人物小传。
- 古典典故摘录。
- 怪谈设定。
- 地点、古国、族群、禁忌、道具说明。
- 内容协作者口述补充。
- 原本删除但仍有价值的桥段。

## 3. 内容协作者提交资料时只需要附这 8 行

```markdown
## 本批资料说明

资料主题：
原本在《山野逸事》第三部中的用途：
我最想保留的气质：
绝对不能误读或删掉的东西：
可以大胆改编的部分：
我希望 AI 帮我重点判断的问题：
我只确定这些内容：
我不确定、可以让 AI 先猜再问我的内容：
```

示例：

```markdown
## 本批资料说明

资料主题：无灯墟市和影子交易
原本在《山野逸事》第三部中的用途：第三集的主事件，表现边地民俗与人口买卖的混合恐怖。
我最想保留的气质：一开始像鬼市，后来发现活人更可怕。
绝对不能误读或删掉的东西：灯不照脸，只照影子；货价不是钱，而是明日。
可以大胆改编的部分：人物可以换，地点可以并入九州旅途。
我希望 AI 帮我重点判断的问题：它更适合做城镇事件，还是半途路遇？
```

## 4. AI 首轮阅读 Prompt

把下面这段交给 AI，并附上内容策划资料。

```markdown
你现在是《不思异：九州》的志怪资料采编和副策划。

你要阅读我提供的《山野逸事》第三部旧资料，但不要直接改写成游戏剧情，也不要让我填写大量表格。

请你先做四件事：

1. 资料摘要
   - 这批资料主要讲什么。
   - 有哪些地点、人物、异象、禁忌、道具、典故来源。

2. 可游戏化候选
   - 哪些适合做地点志。
   - 哪些适合做城镇事件。
   - 哪些适合做半途路遇。
   - 哪些适合做补给、危机、行囊物或结局伏笔。

3. AI 初步判断
   - 每条候选用一句话说明它的故事核。
   - 标出 3 到 5 个不可忽视的视觉/听觉/动作意象。
   - 初步判断它可能关联车轴、粮草、神志中的哪一项。
   - 标出你不确定、不能擅自定论的地方。

4. 内容协作者追问清单
   - 最多问 8 个问题。
   - 每个问题必须指向具体素材。
   - 每个问题都要说明：内容协作者回答后会影响什么游戏设计判断。
   - 不要问“请完善世界观”这种大问题。

输出格式：

# AI 初读报告

## 资料摘要
## 可游戏化候选
## 不确定点
## 给内容协作者的追问
## 暂不建议进入游戏的内容
```

## 5. AI 追问应该长什么样

好的追问不是“请补充设定”，而是把分歧讲清楚。

```markdown
我读到“无灯墟市”这一段，目前判断它适合做成路遇/交易点。

关键分歧：
它到底是真实存在的鬼市，还是当地人用来掩护人口交易的谣言？

这会影响游戏设计：
- 如果是真鬼市，玩家可以用神志换粮草。
- 如果是人为骗局，玩家可以揭穿它，但可能损车轴、惹来追杀或影响后续路线。

请你判断它在原设定里更接近哪一种？有没有第三种答案？
```

内容协作者可以短答：

```markdown
这是骗局，但骗局背后借用了真实禁忌。当地人不知道自己正在重复古国祭仪。
不可丢的是“灯不照脸，只照影子”。
玩家最好先以为自己进了鬼市，后来才发现活人比鬼更可怕。
```

## 6. 内容协作者只需要回答的 7 类问题

每条重要素材最多回答这些，不需要写完整策划案。

| 问题 | 作用 |
| --- | --- |
| 这个东西为什么值得进入《不思异：九州》？ | 判断是否保留 |
| 玩家在这里被诱惑什么？ | 决定奖励、补给、交易或路线诱因 |
| 玩家害怕失去什么？ | 决定代价、失败和神志压力 |
| 它在世界观里是真相、谣言、禁忌、骗局、风俗还是古国遗痕？ | 决定叙事性质 |
| 哪个意象、动作、关系或句子绝不能删？ | 保留原作灵魂 |
| 它应该让玩家更想走下一站，还是更怕走下一站？ | 决定旅途节奏 |
| 低神志时它怎么变坏？ | 接入神志系统 |

时间紧时，只回答第 1、2、4、5、7 条。

## 7. AI 吸收内容协作者回答后的输出 Prompt

内容协作者回答后，把下面这段交给 AI。

```markdown
请你根据内容协作者回答，生成一份《内容策划确认版素材包》。

要求：

1. 不要丢掉内容协作者确认的原意。
2. 明确区分“内容策划确认”和“AI 推测”。
3. 不要直接定数值，只给资源倾向。
4. 不要把影视桥段硬塞成完整任务，要拆成可测试的游戏素材。
5. 每条素材都要说明它适合进入《不思异：九州》的哪一类模块。

输出格式：

# 山野逸事志怪内容确认素材包_批次_日期

## 本批资料一句话价值

## 可进入游戏转译的素材

### SYYS-001 素材名

- 原资料位置：
- AI 初步理解：
- 志怪内容协作者确认/修正：
- 世界观定性：
- 玩家欲望：
- 玩家恐惧：
- 不可丢意象：
- 低神志扭曲：
- 推荐游戏用途：地点志 / 城镇事件 / 半途路遇 / 补给 / 危机 / 行囊物 / 结局伏笔
- 资源倾向：车轴 / 粮草 / 神志 / 暂不确定
- 还需要二次追问：

## 暂缓素材

## 风险和注意事项
```

## 8. 游戏侧接手标准

一条素材进入《不思异：九州》的游戏转译前，不需要完整剧情，只需要满足：

- 内容协作者明确保留。
- 世界观性质明确。
- 玩家欲望明确。
- 玩家恐惧明确。
- 不可丢意象明确。
- 有低神志扭曲方向。
- 没有明显版权、表达或项目调性风险。

之后才由游戏侧转成：

- 地点志。
- 城镇事件。
- 半途路遇。
- 补给行动。
- 神志危机。
- 行囊物。
- 结局伏笔。

具体数值、概率、保底和 UI 文本不要求内容协作者处理。

## 9. 主线程验收口径

主线程只验收这些：

- AI 是否误读内容协作者确认的原意。
- 追问是否具体、节省内容协作者时间。
- 内容协作者回答是否被正确吸收。
- 是否保留志怪味道，而不是变成普通任务。
- 是否能形成 2 到 4 个玩家选择。
- 是否能关联车轴、粮草、神志。
- 是否能放进当前横屏弹窗与小地图流程。
- 是否值得进入 Web Demo 小批量测试。

## 10. 第一轮试运行建议

第一轮不要追求全量整理。

建议流程：

1. 内容协作者交 3 到 5 个旧设定、故事桥段或典故来源。
2. AI 生成初读报告和最多 8 个追问。
3. 内容协作者短答。
4. AI 生成内容策划确认版素材包。
5. 主线程挑 2 条转成游戏事件卡。
6. 只把 1 到 2 条接入 Web Demo 测试。

如果这套流程让内容协作者觉得省力，再扩大到每轮 20 到 30 条候选。
