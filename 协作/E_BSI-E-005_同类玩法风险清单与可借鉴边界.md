## 0. 总结判断

《不思异：九州》可以综合吸收三类同类玩法的抽象经验：

```text
Out There：大界面路线选择、资源压力、未知探索。
横屏远行游戏：横向旅途舞台、假动态、日夜、云、远近景、队伍在路上。
Darkest Dungeon 类横向行进：队伍压力、低神志、横向推进、事件打断、失败记忆点。
```

但必须把这些经验全部转译成《不思异：九州》自己的表达：

```text
羊皮纸山海舆图
+ 古道车队
+ 车轴 / 粮草 / 神志
+ 雾散、城显、路显
+ 山海地名与自研游记
+ 自研图标 / 自研音效 / 自研 UI
```

最高风险不在“同类玩法机制”，而在“可识别表达组合”：星图、太空、横向载具剖面、西部马车、哥特怪物远征、DD 式压力 UI、现代《山海经》插画/译文、以及宣传里绑定竞品名。

## 1. Out There 风险边界

### 1.1 可借鉴的抽象机制

| 可借鉴项 | 风险等级 | 九州转译 | 说明 |
|---|---|---|---|
| 大界面战略选择 | 低 | 九州舆图上决定下一段路 | “地图上选下一站”是通用结构 |
| 多资源压力 | 低 | 车轴、粮草、神志 | 资源管理可用，但资源语义必须彻底不同 |
| 路线代价比较 | 低 | 路引签显示消耗、风险、补给、锁定原因 | 信息结构可用，视觉和文案自研 |
| 未知探索 | 低 | 雾中地名、残卷缺页、远方影子 | 不提前摊开全地图 |
| 到达后显现 | 低 | 雾散、城显、路显 | 显影是本项目自己的舆图表达 |
| 短局失败复盘 | 低 | 荒外游记、残卷、车队遗物 | 失败记忆点可用，文本原创 |
| 逐渐理解世界 | 可控 | 巫文、地志、异族语片、神志阈值 | 不复制外星语言学习流程和 UI |

### 1.2 必须避开的具体表达

| 禁止对象 | 风险等级 | 替代方式 |
|---|---|---|
| 星图、星球节点、星系航线 | 禁止 | 羊皮纸舆图、地名签、朱印、墨线古道 |
| 太空背景、飞船、采矿、宇航员孤独叙事 | 禁止 | 古代车队、荒外路途、王命/失路/游记 |
| 燃料 / 氧气 / 船体对应三资源 | 禁止 | 车轴 / 粮草 / 神志 |
| 科幻 HUD、扫描圈、雷达点亮 | 高 | 墨晕、水渍、纸雾、朱印落下 |
| 竞品资源图标和颜色组合 | 高 | 自研车轴裂纹、粮袋、神志残纹 |
| 竞品路线卡排版和文案节奏 | 高 | 路引签 / 驿牒 / 残卷批注 |
| 外星语言、飞船设备槽、星球采集流程 | 高 | 巫文、地志线索、器物、地点事件 |
| “山海经版 Out There”等宣传 | 禁止 | “山海经题材远行生存叙事游戏” |

## 2. 西部横屏远行游戏可借鉴边界

这里的“西部横屏远行游戏”只作为内部体验参考：横向画幅、队伍在路上、远景移动、假动态、昼夜变化、天气和路途景观。可借鉴的是视听语法，不是题材、具体镜头或资产。

### 2.1 属于通用表达、可以借鉴

| 通用表达 | 可用等级 | 九州转译 |
|---|---|---|
| 横向旅途舞台 | 低 | 横向山海路途，车队从左到右行进 |
| 假动态推进 | 低 | 背景轻微视差、云气缓动、车辙或尘雾移动 |
| 日夜变化 | 低 | 日数、天色、低神志、异象叠加 |
| 云、风、雾、远山 | 低 | 山海云气、泽雾、纸雾、荒外远岭 |
| 路途景观层次 | 低 | 天空 / 远山 / 中景古道 / 前景车辙 |
| 队伍前景常驻 | 低 | 古代辇车/车队剪影，不做西部马车 |
| 半途停下 | 低 | 路遇打断，车队停在路线中段 |
| 路标 / 下一站提示 | 低 | 舆图路引、木牌、残卷旁注、地名短签 |
| 天气影响氛围 | 低 | 风沙、泽气、赤水、裂隙污染 |

### 2.2 容易形成相似风险，必须避开

| 高风险表达 | 风险等级 | 避让说明 |
|---|---|---|
| 西部马车、牛仔、枪、木轮车造型组合 | 禁止 | 本项目是山海古道车队，不是西部片 |
| 直接临摹横屏竞品的镜头构图 | 高 | 不按竞品截图摆车、地平线、天空和 UI |
| 横向载具剖面、内部操作站、锅炉/引擎管理 | 高 | 尤其接近 FAR 类载具游戏时必须避开 |
| 路障解谜、实时驾驶、碰撞、拉杆操作 | 高 | 本项目路线选择是策略/叙事，不是横版操作关卡 |
| 西部配色、字体、UI 木牌、酒馆式按钮 | 高 | 视觉回到帛书、旧纸、朱砂、墨线和山海器物 |
| 竞品云层/日夜/远景的具体节奏和图像 | 高 | 可用“云在动”，不能用同款云形、同款画幅节奏 |
| 西部片音效：马蹄、缰绳、枪声、口琴式主题 | 高 | F 线程应做古意、纸墨、车轮、风雾，不做西部声景 |
| 宣传中说“西部远行换成山海经” | 高 | 对外只说“横向山海旅途舞台” |

### 2.3 对当前 D/C 方案的安全口径

当前 D 线程提出的“假动态、日夜、云、路途景观、队伍前景”可以继续推进。验收时看三件事：

1. 车队是否是山海古道车队，而不是西部马车或 FAR 式独特载具。
2. 横向舞台是否只是表现层，核心决策仍是路线、资源、事件。
3. 截图是否能一眼看出九州荒外气质，不会被认为是某个横屏旅途游戏的换皮。

## 3. Darkest Dungeon 式横向行进感可借鉴边界

Darkest Dungeon 和类似作品的可借鉴点在“队伍承压、横向推进、黑暗氛围、事件打断、失败复盘”这些通用视听和玩法语法。不可复制的是哥特手绘风格、怪物和职业体系、压力 UI、叙事口吻、旁白节奏、战斗构图和商店话术。

### 3.1 属于通用视听语法、可以借鉴

| 通用语法 | 可用等级 | 九州转译 |
|---|---|---|
| 队伍从左向右推进 | 低 | 车队沿古道向西，或在舆图上推进 |
| 队伍状态压力 | 低 | 车轴、粮草、神志、随从恐惧、饥荒 |
| 黑暗 / 雾 / 未知压迫 | 低 | 纸雾、泽雾、裂隙污染、低神志文本 |
| 事件打断行进 | 低 | 半途路遇、车队停在路中 |
| 资源和精神共同压迫 | 低 | 神志影响文本可信度，不复制压力病症体系 |
| 失败后留下记忆点 | 低 | 荒外游记、残卷、前队遗物 |
| 队伍剪影与前景层次 | 可控 | 古代车队剪影，不做哥特英雄队列 |
| 低饱和危险气氛 | 可控 | 山海荒外的旧纸、赤水、黑土、泽气 |

### 3.2 不能碰的具体 UI / 美术 / 叙事节奏

| 禁止对象 | 风险等级 | 避让说明 |
|---|---|---|
| Darkest Dungeon 的哥特鸦羽线稿/硬阴影风格 | 禁止 | 本项目应是山海舆图、帛书、墨线，不做同款哥特手绘 |
| 英雄横向队列、职业头像、压力条布局 | 高 | 车队资源 HUD 自研，不复制英雄栏和状态图标 |
| 火把/光照压力 UI | 高 | 若有“明暗”，转译为神志/雾/日夜，不做火把系统 |
| 旁白式讽刺、庄严、失败评语节奏 | 高 | B 线程用游记、地志、路引口吻，不模仿旁白腔 |
| 怪物、职业、病症、怪癖、营地结构 | 禁止 | 不导入哥特怪物远征体系 |
| 战斗站位、回合 UI、技能格子 | 禁止 | 本项目首版不是战斗 RPG |
| 红黑高反差战斗界面与扭曲怪物设计 | 高 | 视觉危险应来自山海异象和认知污染 |
| “压力爆发/美德”式命名和机制组合 | 高 | 可有神志阈值，但命名和表现自研 |
| 商店页说“山海经 Darkest Dungeon” | 禁止 | 对外只讲远行生存叙事 |

### 3.3 对本项目的安全表达

安全目标不是“做山海经 Darkest Dungeon”，而是：

```text
车队在横向旅途中承受资源和认知压力；路遇打断前进；神志下降让文本和地图变得不可靠。
```

这句话可指导 C/D/B，但不能照搬 DD 的美术、UI 和语气。

## 4. 《山海经》公共领域与现代译注 / 图像版权边界

### 4.1 可以使用的部分

| 类型 | 可用等级 | 使用方式 |
|---|---|---|
| 《山海经》原典中的古代名称、地名、异兽名、神话母题 | 低 | 可作为灵感和命名基础 |
| 公共领域古籍原文 / 影印 | 可控 | 少量引用时记录来源；更推荐自研改写 |
| 古籍中可查的母题关系 | 低到可控 | 用于启发地点、路线、祭所、异族边境 |
| 自研翻译、自研解释、自研事件文本 | 低 | 推荐路线 |
| 自研图标、地图符号、抽象纹样 | 低 | 建立项目自有视觉体系 |

### 4.2 不能直接使用的部分

| 类型 | 风险等级 | 原因 |
|---|---|---|
| 现代白话译文 | 高 | 翻译本身通常是独立表达 |
| 现代注释、考据段落、百科条目 | 高 | 选择、编排、解释和文字表达可能受保护 |
| 现代插画、异兽图鉴、设定图 | 高 | 美术作品权利明确，不能“参考后改一点” |
| 现代设定书、影视、漫画、游戏中的山海经二创设定 | 高 | 很容易继承别人的原创设定 |
| 网络整理表格、公众号文章、百科图片 | 高 | 来源链不清，且可能混入现代作者表达 |
| AI 提示词中点名现代画师、现代图鉴、影视游戏风格 | 高 | 会把相似风险带入生成结果 |

### 4.3 本项目推荐工作法

1. B 线程只把原典和现代资料当理解辅助，正式文本全部自写。
2. 每个地点 / 异族 / 异象至少完成一次“玩法功能转译”：它在路线、资源、神志或事件中起什么作用。
3. D 线程不要用现代山海经图鉴构图；优先做地图符号、剪影、纹样和抽象器物。
4. 宣传页只说“以《山海经》神秘地理为灵感”，不说“忠实还原某译本 / 某图鉴”。
5. 重要文本和素材在商业发布前建立资料来源台账，区分“原典母题”“现代参考”“正式进入游戏的自研表达”。

## 5. 给 C/D/B 的可执行红线清单

### 5.1 C 线程：界面与实现红线

| 类别 | 红线 | 安全做法 |
|---|---|---|
| 地图显隐 | 不能提前展示全量节点和全量路线网 | 只渲染当前、已见、可达、锁定、雾中 |
| 星图风险 | 不能用星图、航线、扫描、雷达、jump 等外显命名和 UI | 用 `fogged / hinted / revealed / traveled` 等中性语义 |
| 路线卡 | 不能列出全局路线，不能让 UI 像太空生存面板 | 只列当前地点已显现路线，做路引签式信息 |
| 横屏舞台 | 不能引入实时驾驶、碰撞、路障解谜、载具内部操作站 | 舞台只表达旅途状态和半途打断 |
| DD 风险 | 不能做英雄队列、职业头像、火把压力条、战斗技能格 | 保持车队资源 HUD 和事件选择结构 |
| 文案来源 | 不能把现代译文/百科句子塞入数据字段 | 使用 B 线程自研 `fogName / revealText / routeHintText` |
| 资产来源 | 不能从竞品截图、现代图鉴、来源不明网图生成 UI | 临时素材标 Demo 临时；正式素材走台账 |
| 调试命名 | 内部变量可中性，不要公开 UI 出现竞品名 | 文档可写参考，产品 UI 不写竞品 |

### 5.2 D 线程：视觉资产红线

| 类别 | 红线 | 安全做法 |
|---|---|---|
| Out There | 不用星空、星球、发光航线、科幻 HUD | 羊皮纸、墨线、朱砂、地名签、纸雾 |
| 横屏远行 | 不用西部马车、枪、牛仔、酒馆木牌、西部配色 | 山海古道车队、辇车剪影、荒泽、废关、赤水 |
| FAR 类载具 | 不做大型独特载具剖面、锅炉/引擎内部操作 | 车队是前景剪影和状态符号，不是操作对象 |
| Darkest Dungeon | 不做哥特硬阴影、红黑怪物、英雄头像队列 | 山海墨线、旧舆图、低神志污染层 |
| 图标 | 不抄资源图标、职业图标、火把/压力图标 | 车轴裂纹、粮袋、神志残纹全部自研 |
| 动效 | 不做科幻扫描、手游抽卡爆光、DD 式压力爆发 | 雾散、墨晕、笔锋、朱印、纸纹显影 |
| 现代山海图像 | 不参考现代图鉴构图，不改一点就用 | 从古籍母题抽象成符号、纹样、剪影 |
| 临时 AI 素材 | 提示词不得点名竞品、画师、影视、现代图鉴 | 用“古舆图、墨线、纸雾、山海异象”等抽象词 |

### 5.3 B 线程：文案与叙事红线

| 类别 | 红线 | 安全做法 |
|---|---|---|
| Out There | 不写宇航员孤独、外星文明、飞船迷航、星域命运 | 写车队、荒外、王命/失路、地志和巫文 |
| 西部远行 | 不写牛仔、西部镇、赏金、枪械、驿站酒馆口吻 | 写古驿、关隘、泽国、墟市、祭所 |
| Darkest Dungeon | 不模仿旁白腔、失败嘲讽、哥特庄严句式 | 写残卷、地点志、路引、车队记事 |
| 《山海经》资料 | 不照搬现代白话译文、注释、百科段落 | 原典母题加自研转译和玩法功能 |
| 异兽 / 异族 | 不继承现代图鉴设定和二创族群设定 | 自定义其资源、语言、路线和神志作用 |
| 失败文本 | 不复制竞品失败结构和结局句式 | 写“荒外游记 / 残卷 / 前队遗物” |
| 宣传文案 | 不写“某某版”“同款”“精神续作” | 写玩法事实：选路、消耗、路遇、神志变文本 |
| 低神志文本 | 不误导真实规则，不把不可走写成可走 | 气质可怪，规则信息必须清楚 |

### 5.4 宣传红线

| 禁用表达 | 替代表达 |
|---|---|
| 山海经版 Out There | 山海经题材远行生存叙事游戏 |
| 中国版 Darkest Dungeon | 车队在荒外远行，资源和神志会被道路消耗 |
| 西部远行换皮山海经 | 横向山海旅途舞台 |
| 类 FAR 载具旅途 | 古道车队远行表现层 |
| Steam 爆款同款玩法 | 节点路线、文字事件、三资源压力 |
| 完全还原《山海经》 | 以《山海经》神秘地理为灵感的原创转译 |
| 使用某现代译本/某图鉴设定 | 自研山海异闻与地点志 |
| 克苏鲁山海经 | 认知污染 / 不可靠文本 / 荒外异闻 |

### 5.5 视觉资产红线

1. 不使用竞品截图、商店图、图标、角色、UI 截图作素材。
2. 不使用影视截图、现代插画、现代山海经图鉴、来源不明网图。
3. 不使用竞品音乐、音效、主题旋律、环境声组合。
4. 不把 AI 临时素材标成正式可用。
5. 不用点名竞品、画师、影视、品牌、角色的提示词生成正式候选。
6. 不使用来路不明的古风字体、书法字体包。
7. 不把未授权用户上传素材放进商店页、宣传图或正式发布包。
8. 所有正式素材进入发布前，必须能回溯来源和授权状态。

## 6. 需要未来找律师确认的点

以下不是“已经法律确认”，只是 E 线程建议未来正式商业化前请律师或版权顾问复核：
