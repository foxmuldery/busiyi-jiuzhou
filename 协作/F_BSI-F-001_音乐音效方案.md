# F_BSI-F-001 音乐音效方向与资产规范

## 1. 任务目标

为《不思异：九州》Web Demo 建立第一版音频方向，重点不是做完整配乐，而是定义最小可用的音乐、环境声、交互音效和神志污染音频规则。

第一版音频应服务三个目标：

1. 让横向旅途舞台更有远行感。
2. 让车轴、粮草、神志的变化有可感知反馈。
3. 让神志降低时，玩家不仅“看见”世界变了，也“听见”世界不可靠。

## 2. 音频总原则

- 声音要空，不要满。
- 音乐要低存在感，不要压文本。
- 环境声比旋律更重要。
- 恐怖感来自错位、远近不对、声源不明，不靠尖叫和强惊吓。
- 所有音频素材必须原创、可商用或授权明确。
- 不模仿任何具体竞品、影视、游戏的标志性旋律或音效。

## 3. 第一版音频分层

| 层级 | 作用 | 示例 | 第一版优先级 |
|---|---|---|---|
| 音乐底层 | 提供整体情绪和区域气质 | 低频弦、埙/箫质感、缓慢鼓点、无人声吟唱底色 | 中 |
| 环境声层 | 让路段有空间 | 风、远雷、水声、草响、空市低响 | 高 |
| 行旅声层 | 让车队“正在走” | 木轮、车轴吱呀、绳索、脚步、马/牲口低响 | 高 |
| 交互音效层 | 反馈玩家操作 | 选项确认、资源变化、路线解锁、事件出现 | 高 |
| 神志污染层 | 表现认知不可靠 | 倒放低语、错位鼓点、水下人声、重复轮声 | 高 |
| 告警层 | 提示资源危机 | 断轴重响、饥荒空腹低鸣、神志耳鸣 | 中 |

## 4. 三资源音频语言

| 资源 | 听觉隐喻 | 正常状态 | 危急状态 |
|---|---|---|---|
| 车轴 | 木、绳、轮、轴 | 低而规律的木轮声 | 断续、偏斜、尖锐摩擦 |
| 粮草 | 腹、袋、火、队伍 | 布袋摩擦、微弱炊火、低声整理行囊 | 空袋声、火灭、队伍沉默 |
| 神志 | 耳、心跳、远声、错位 | 稳定风声和清晰环境 | 远鼓错拍、低语倒放、水声从错误方向传来 |

## 5. 地点与路段音频标签

| 场景 | 环境声 | 音乐倾向 | 神志污染变化 |
|---|---|---|---|
| 中原驿 | 远处人声、布幡、木门 | 几乎无音乐，留人间秩序 | 人声逐渐听不清 |
| 故王道 | 风穿旧道、车辙声 | 稳定低频 | 脚步与车轮不同步 |

## 7. AI 音频生成提示语原则

生成音乐时：

- 不写任何具体作曲家、影视、游戏、厂牌或竞品名称。
- 不要求“像某某作品”。
- 强调古代荒野、低频风、木轮、远鼓、埙箫质感、留白。
- 要求无主旋律或弱主旋律，便于循环。
- 避免强节拍、现代鼓组、电子舞曲、史诗管弦。

生成音效时：

- 单个音效尽量干净，避免自带混响过重。
- 行旅循环必须能无缝循环。
- 告警音不要刺耳，避免廉价恐怖跳吓。
- 神志污染可用倒放、错位、轻微失真，但不要不可控噪音。

## 8. 第一批生成提示语草案

### MUS-001 荒外低频底乐

```text
生成一段适合中国古代神秘九州远行游戏的低存在感背景音乐，时长 45 秒，可无缝循环。氛围空旷、克制、荒凉，有低频风声般的持续底色，少量埙、箫、远鼓、弦鸣质感，但不要明显主旋律。整体像凡人车队穿过不可理解的荒外。

负面要求：不要歌词，不要清晰人声，不要人声唱词，不要史诗化，不要仙侠热血，不要现代电子节拍，不要流行歌结构，不要模仿任何具体游戏、影视、音乐作品，不要现代电子舞曲，不要史诗大片感，不要西部片马车配乐，不要跳吓，不要刺耳爆音。声音要空，留白，不能压过文字阅读。
```

### TRV-001 木轮稳定行进

```text
生成一段 12 秒可无缝循环的古代木轮车队行进音效。包含木轮压过土路、车轴轻微吱呀、绳索和行囊轻晃、远处微风。节奏稳定但不要太机械。

负面要求：不要歌词，不要清晰人声，不要现代车辆声，不要马车西部片风格，不要音乐旋律，不要模仿具体游戏、影视、音乐作品，不要现代电子舞曲，不要史诗大片感，不要仙侠网游感，不要跳吓，不要刺耳爆音。声音要空，留白，不能压过文字阅读。
```

### SAN-002 神志重度污染

```text
生成一段 15 秒可循环的低神志认知污染环境声。声音应诡异、克制、遥远：水声从不正确的方向传来，远鼓节拍轻微错位，低语像倒放但不可辨认，木轮声短暂重复。

负面要求：不要歌词，不要清晰人声，不要尖叫，不要恐怖片跳吓，不要清晰语言，不要强烈噪音墙，不要模仿具体游戏、影视、音乐作品，不要现代电子舞曲，不要史诗大片感，不要仙侠网游感，不要西部片马车配乐，不要刺耳爆音。声音要空，留白，不能压过文字阅读。
```

## 9. 文件格式与实现建议

- Web Demo 优先使用 `.ogg` 和 `.mp3` 两套导出，浏览器兼容更稳。
- 循环音乐和环境声目标体积尽量控制在 1 MB 以内。
- UI 音效可使用短 `.mp3` 或 `.ogg`，单个尽量小于 100 KB。
- 音量默认偏低，给用户静音按钮。
- 移动端要考虑浏览器自动播放限制，首次点击后再启动音频。

## 10. 音量与混音基准

Web Demo 是文字阅读型游戏，音频的默认职责是托住氛围，不是抢占注意力。第一版建议以“可听见但可忽略”为标准。

| 类型 | 默认播放音量 | 响度目标 | 峰值余量 | 备注 |
|---|---:|---|---|---|
| 音乐底层 | 0.18-0.28 | 约 -24 LUFS | 低于 -2 dBFS | 用户主动开启后播放，或默认极低音量 |
| 环境声层 | 0.25-0.40 | 约 -24 到 -21 LUFS | 低于 -2 dBFS | 可作为第一版主要氛围 |
| 行旅声层 | 0.18-0.32 | 约 -24 到 -20 LUFS | 低于 -2 dBFS | 与车队移动状态绑定 |
| UI 音效 | 0.35-0.55 | 不做强制 LUFS | 低于 -3 dBFS | 短促、轻，不要刺耳 |
| 资源告警 | 0.40-0.65 | 不做强制 LUFS | 低于 -2 dBFS | 只在危急阈值触发 |
| 神志污染层 | 0.08-0.30 | 约 -30 到 -22 LUFS | 低于 -3 dBFS | 按神志下降逐步叠加 |

第一版不建议做复杂动态混音，只需要三档：

| 神志档位 | 音频变化 |
|---|---|
| 高 | 环境声清晰，行旅声稳定，音乐可选 |
| 中 | 神志轻污染层淡入，环境声略低通，偶发错位铃声 |
| 低 | 神志重污染层增强，行旅声短暂重复，部分 UI 音效变钝，但仍不能影响读字 |

## 11. 授权与来源记录

任何音频进入 `05_可用素材` 前，必须记录以下字段：

| 字段 | 说明 |
|---|---|
| 资产 ID | 例如 `AMB-001`、`TRV-001`、`MUS-CAND-002` |
| 类型 | 音乐、环境声、行旅声、UI、告警、神志层 |
| 文件路径 | 原始文件、剪辑版、Web Demo 版分别记录 |
| 来源工具 | AI 工具、素材库、实录、人工制作 |
| 生成日期 | 年月日 |
| 提示语/检索词 | AI 生成提示语或素材库检索词 |
| 授权状态 | 待确认、内部 Demo 可用、可商用、禁止商用 |
| 修改记录 | 剪辑、降噪、响度、格式转换、循环点 |
| 主观复核 | 是否太满、太现代、太恐怖、太像现成作品 |
| 使用状态 | 待复核、Demo 临时、可用、弃用 |

未经授权确认的音乐、采样包和网上下载音效，只能放入待复核区，不能写入“可用素材”。

## 13. 验收标准

F 线程交付音频方案时，主线程应按以下标准验收：

1. 是否有清晰的最小音频包，不依赖大体积音乐库。
2. 是否区分音乐、环境声、行旅声、UI、告警、神志层。
3. 是否明确默认静音/低音量和浏览器自动播放策略。
4. 是否有授权字段，且未把未授权素材标为可用。
5. 是否有可直接交给用户生成音频的提示语。
6. 是否有可直接交给 C 线程接入的文件命名、循环、音量和失败回退建议。
