# BSI-UX-007 横屏手游界面修正单

项目：不思异：九州  
主协调：Codex 主线程  
状态：修正 BSI-UX-006；交给 D 线程重出体验方案，C 线程重出技术预案后再执行  
日期：2026-06-14

## 一、用户修正意见

“更像手机游戏”不是竖屏手机壳，而应该是横屏游戏界面。

本项目的核心气质更适合横屏：

- 横向山海旅途舞台是第一视觉。
- 车队沿画面从左向右行进，远景、云、日夜、异象都服务于旅途感。
- 交互更接近横屏手游、Darkest Dungeon 式远行呈现、以及西部横向旅行游戏，而不是竖屏卡牌列表。
- Out There 式路线选择仍保留，但地图选择也应放在横屏游戏 UI 内，不做网页仪表盘。

## 二、立即修正

BSI-UX-006 中“竖屏优先、桌面居中手机预览、底部四导航”的默认决定作废。

新的默认决定：

- 第一版采用横屏优先，推荐基准比例 16:9 或接近 19.5:9。
- 桌面浏览器预览时显示一个横屏游戏画幅，不铺满成网页工作台。
- 手机端默认提示或适配横屏，不再按竖屏主流程设计。
- 主要体验围绕“横向旅途舞台 + 游戏 HUD + 行动面板”组织。

## 三、目标界面结构

推荐结构：

```text
┌────────────────────────────────────────────┐
│ 顶部轻 HUD：地点 / 日数 / 车轴 / 粮草 / 神志 / 距终点 │
├────────────────────────────────────────────┤
│                                            │
│       横向山海旅途舞台：车队、远景、路面、异象       │
│                                            │
├──────────────────────────┬─────────────────┤
│ 底部主行动区：事件文本 / 选项 / 补给 / 路线卡       │ 右侧轻状态：
│                          │ 行囊/日志/设置折叠 │
└──────────────────────────┴─────────────────┘
```

重点：

- 舞台占主面积，事件和路线是贴在舞台下方的游戏 UI，不是网页卡片墙。
- 路线选择可以切换到“九州图模式”，但仍在同一个横屏游戏框内完成。
- 行囊九宫格作为弹层或右侧抽屉出现，不常驻挤压主舞台。
- 日志默认显示最近一条反馈，完整日志折叠。
- 音乐、动态层、设置类开关收进设置，不作为主界面文字块。

## 四、地点页横屏逻辑

地点页应像“车队停靠或正在行进的一幕”：

1. 顶部 HUD 显示当前位置、日数、三资源、终点距离。
2. 中央大舞台显示当前地点/路途状态。
3. 下方行动区显示地点志摘要、补给、当前事件选择。
4. 已处理地点事件后，下方行动区提示“打开九州图选择下一段路”。
5. 半途路遇时，舞台车队停在中段，下方行动区切换成路遇事件。

## 五、九州图横屏逻辑

九州图不是独立网页页，而是“战略地图模式”：

- 舞台区域切换为横屏九州图。
- 地图左/中部显示路线节点和当前点。
- 底部或右侧显示可选路线卡：目的地、消耗、风险、可观察补给、锁定原因。
- 点击路线后回到旅途舞台，车队从左向右移动。
- 不能让玩家只看到路线文字列表；要保留“我在地图上选择前路”的感觉。

## 八、主协调验收标准

- 第一眼是横屏游戏画面，不是网页信息面板。
- 中央横向旅途舞台占主视觉。
- 三资源、日数、当前位置、终点距离常驻但不沉重。
- 事件选择和补给在下方行动区完成，按钮适合横屏触控。
- 九州图模式能清楚比较路线消耗、风险、可观察资源。
- 行囊九宫格可打开查看，但不常驻挤压主舞台。
- 日志可复盘，但默认不抢主界面。
- 横屏小尺寸下无文字严重遮挡，无横向滚动导致游戏框破裂。
- 语法检查和核心流程 smoke 通过。

## 九、暂不需要用户拍板

先按横屏默认推进。后续再集中让用户判断：

- 横屏比例更接近 16:9 还是更宽的手机横屏。
- 右侧状态抽屉是否常驻一小列。
- 九州图模式是全屏切换还是覆盖在舞台上。
- 行囊九宫格从右侧抽屉打开还是从底部行动区打开。
