# D_BSI-D-003 第二批素材应用与舞台分层方案

> 子线程：D 视觉体验与交互氛围设计师  
> 任务：第二批素材应用与舞台分层方案  
> 阶段：Web Demo 测试原型  
> 边界：不生成新素材，不写代码，不修改其它线程交付文件，不让视觉遮挡资源、路线、事件文本、按钮和日志

## 1. 目标

本方案把第二批已整理素材转成 C 线程可直接实现的横向山海旅途舞台组合。第一版目标是验证“假动态横屏长图”的信息层级，而不是制作最终美术。

核心原则：

- 背景服务地点气氛，不直接承载正文。
- 车队保持独立主行动层，不烘焙进背景。
- 神志污染只影响舞台，不盖住 UI 操作层。
- 路线、事件、按钮、资源和日志保持独立、稳定、可读。

## 3. 通用舞台层级

C 线程可按以下顺序实现：

```text
BG 后景底图
LOC 中景/地点显影层
FG 前景路面层
CAR 车队主行动层
OV 云雾/神志叠层
UI 操作层
```

实现边界：

- `BG/LOC/FG/CAR/OV` 都应 `pointer-events: none`。
- `UI 操作层` 独立于舞台，z-index 永远高于视觉层。
- 事件文本、行动按钮、资源条、路线卡、日志不进入 OV 污染层。
- 舞台内只允许放少量节点提示和方向提示；真实点击仍在路线卡或按钮中完成。

## 4. 三套可实现舞台组合

### 4.1 组合一：废关古道

用途：适合开局或前段路程，风险中等，强调暮色、旧关、荒路和行旅感。

| 层级 | 素材 | 建议设置 | 玩法作用 |
|---|---|---|---|
| BG 后景 | `BG-006_废关古道暮色.png` | 作为底图，桌面端居中裁切，移动端优先保留中左部旧关轮廓 | 明确“废关古道”地点气质 |
| LOC 中景/地点 | 可不启用；需要地点显影时可低透明叠 `LOC-001` 左侧区域 | 透明度 0%-18%，只在抵达/发现时淡入 | 表示废关、祭所、墟市等地点浮现 |
| FG 前景 | `FG-003_荒外路面泥痕车辙前景层.png` | 固定底部，透明度 85%-100% | 强化道路、车辙和车队所在平面 |
| CAR 车队 | `CAR-005_古代辇车队伍抠图初版.png` | 桌面宽度约 34%-40%，底部 6%-10%，横向居中偏左 | 表示玩家队伍正在路上 |
| OV 氛围 | `OV-004_云雾风沙气流轻叠层.png` | 透明度 10%-18%，可低速横移 | 形成假动态风沙和暮色流动 |
| OV 神志 | `OV-003_低神志山海幻路血星.png` | 高神志关闭；中神志 8%-12%；低神志 18%-28% | 低神志时出现幻路和异象 |

UI 遮挡安全区：

- 资源条放舞台外顶部，不覆盖画面。
- 路线卡和事件文本放舞台下方或右侧，不压在旧关主体上。
- 舞台内如放当前节点，建议放在下方 58%-72% 高度范围，不超过 2 个标记。
- 日志放底部独立容器，不使用背景图作为日志底。

### 4.2 组合二：赤水骨滩

用途：适合危险水域、中后段风险节点，强调赤水、骨滩、荒泽和神志不稳。

| 层级 | 素材 | 建议设置 | 玩法作用 |
|---|---|---|---|
| BG 后景 | `BG-005_赤水骨滩荒泽.png` | 作为底图，桌面端保留赤水和骨滩横向展开；移动端优先保留中部水泽 | 明确赤水危险路段 |
| LOC 中景/地点 | 不默认使用 `LOC-001` | 若需要地点事件，可只在路线卡中表达 | 避免地点层和赤水画面互相抢焦点 |
| FG 前景 | `FG-003_荒外路面泥痕车辙前景层.png` | 底部叠加，透明度 70%-90%，可轻微降低对比 | 让车队有可行走路面，不像漂在水上 |
| CAR 车队 | `CAR-005_古代辇车队伍抠图初版.png` | 桌面宽度 32%-38%，底部 8%-12%，靠近画面下方安全路段 | 表示“沿水泽边缘行进” |
| OV 氛围 | `OV-004_云雾风沙气流轻叠层.png` | 透明度 12%-22%，可作为水汽/风沙层 | 增强湿冷、荒泽、远雾 |
| OV 神志 | `OV-003_低神志山海幻路血星.png` | 中神志 10%-16%；低神志 22%-32% | 赤水路段可更早出现幻路和血星 |

UI 遮挡安全区：

- 赤水画面信息量高，事件文本不要浮在舞台上。
- 低神志叠层仅覆盖舞台，不进入右侧事件面板。
- 路线消耗必须放在路线卡内，例如“车 -1 / 粮 -1 / 神 -12”，不要只靠画面暗示。
- 行动按钮不使用红色满底，避免和赤水/血星混淆。

### 4.3 组合三：地点显影

用途：适合发现废关祭所、异族墟市、未知地点或隐藏路线时使用，强调“地点从云雾中被辨认出来”。

| 层级 | 素材 | 建议设置 | 玩法作用 |
|---|---|---|---|
| BG 后景 | `BG-007_九州边境天象天空层.png` | 作为天空/远景底图，亮度略降，留出地点层 | 提供较干净的显影背景 |
| LOC 中景/地点 | `LOC-001_废关祭所异族墟市地点层.png` | 透明度 70%-100%，可按事件裁切左/中/右区域 | 表达地点出现、确认、可进入 |
| FG 前景 | `FG-003_荒外路面泥痕车辙前景层.png` | 底部叠加，透明度 80%-100% | 保持车队与地点之间的路面关系 |
| CAR 车队 | `CAR-005_古代辇车队伍抠图初版.png` | 桌面宽度 30%-36%，底部 6%-10%，避开地点主体 | 让玩家感到队伍抵达地点外缘 |
| OV 氛围 | `OV-004_云雾风沙气流轻叠层.png` | 透明度 8%-16%，可在地点显影前后变化 | 表达云雾散开或地貌浮现 |
| OV 神志 | `OV-003_低神志山海幻路血星.png` | 仅在隐藏/异常地点启用，低神志 16%-26% | 表示地点是否可信 |

UI 遮挡安全区：

- 地点名称可以在舞台内短暂出现，但正文说明必须在事件面板。
- 若 LOC-001 做左/中/右裁切，点击区域仍由路线卡承担，避免小屏误点。
- 显影动画不应遮住顶部资源变化。
- 日志只记录“发现地点 / 进入地点 / 错认地点”等结果，不承担地图说明。

## 5. 桌面端高度、裁切、缩放建议

适用宽度：`1366px` 及以上。

| 项目 | 建议值 | 说明 |
|---|---:|---|
| 舞台容器高度 | 260-300px，最高 320px | 保留足够气氛，同时不挤压事件文本 |
| 舞台比例 | 4:1 为主 | 第二批 BG/FG/OV/LOC 均适配 |
| 背景缩放 | 宽度铺满，`object-fit: cover` | 保证无空边 |
| 背景裁切 | 默认 `center center` | 废关可偏左，赤水可居中，地点显影按地点裁切 |
| 车队缩放 | 舞台宽度的 30%-40% | 不要大到像角色操作游戏 |
| 车队位置 | 底部 6%-12%，横向中部偏左 | 保持“正在路上”的稳定感 |
| OV 透明度 | 常态 8%-18%，低神志 18%-32% | 以可读性为上限 |
| UI 面板 | 舞台外布局 | 事件、按钮、日志不进入舞台图层 |

桌面端推荐布局：

```text
顶部资源栏 64-88px
横向舞台 260-300px
下方左侧路线卡 30%-34%
下方右侧事件文本与行动按钮 66%-70%
底部日志 96-140px
```

## 6. 移动端高度、裁切、缩放建议

适用宽度：`360px` 到 `430px` 竖屏。

| 项目 | 建议值 | 说明 |
|---|---:|---|
| 舞台容器高度 | 132-168px，最高 180px | 只做氛围缩略，不抢首屏正文 |
| 背景缩放 | `object-fit: cover` | 保持满宽 |
| 背景裁切 | 废关偏中左、赤水居中、地点显影按事件主体裁切 | 小屏只保留一个主要视觉焦点 |
| 车队缩放 | 舞台宽度的 52%-68% | 小屏中车队需可辨认，但不遮住主体地点 |
| LOC 使用 | 优先局部裁切或降低透明度 | 避免整张地点层细节过密 |
| OV 使用 | 常态 0%-10%，低神志 12%-22% | 移动端污染更克制 |
| 日志 | 默认折叠，只露最近一条 | 不挤压事件与按钮 |

移动端推荐顺序：

```text
顶部三资源紧凑条
当前地点 / 日数
横向舞台缩略层
路线卡片
事件文本
行动按钮
折叠日志
```

移动端特别规则：

- 不在舞台内放多个节点标签。
- 不让污染叠层影响按钮对比度。
- 不强制完整展示 2508px 长图，允许为可读性裁切。
- 横向舞台不横向滚动，避免和页面纵向滚动冲突。

## 7. 给 C 线程的实现顺序建议

1. 先实现 `BG-006 + FG-003 + CAR-005 + OV-004` 的废关古道组合。
2. 再换底图实现赤水骨滩组合，验证同一层级是否可复用。
3. 最后加入 `LOC-001` 做地点显影组合，验证地点层淡入和裁切。
4. 神志高/中/低只先改变 OV 透明度和背景轻微色调，不做复杂动效。
5. 所有组合都先保证资源、路线、事件文本、按钮、日志不被遮挡，再调氛围。

## 8. 风险点

- `CAR-005` 是 Demo 临时可用抠图，边缘仍需复核，正式版应替换为更干净绿幕/透明素材。
- `OV-003` 氛围强，低神志可以使用，但透明度过高会压暗画面并干扰文本。
- `BG-005` 赤水骨滩色彩和信息量较重，不适合把按钮或正文压在舞台内部。
- `LOC-001` 细节较多，移动端整张使用会显得拥挤，建议裁切或只在地点事件中出现。
- 第二批素材可以验证舞台分层，但还不是最终美术风格定稿。
