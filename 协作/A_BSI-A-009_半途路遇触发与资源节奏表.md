## 1. 本轮结论

半途路遇第一版采用：

```text
关键路线显式挂载 midEvent + 路遇冷却 + 资源安全地板
```

目标不是每条路都出事，而是每 1-2 段关键推进一定有一次半途停顿，并且停顿真实改变车轴、粮草、神志或旗标。

当前 `data.js` 有 33 条路线，其中 10 条已有 `midEvent`。本轮建议 P0 再补 6 条必覆盖路线，使主线、青丘支线、无名祠支线、黑齿市支线、白羽淖支线都能稳定体验半途路遇。

P0 最小落地口径：

| 项 | 结论 |
|---|---|
| 路遇密度 | 关键推进每 1-2 段至少 1 次；普通缓冲/回撤路线不强制 |
| 第一段体验 | 从中原驿出发的 3 条路线都应能遇到半途路遇 |
| 单次路遇强度 | 非主动高危选择，单项损耗不超过 8 |
| 高风险污染 | 高风险路线可重压，但不能连续 2 次高污染必触发 |
| 死局保护 | 资源到危急线时先保护或危机补救，不抽普通高损耗路遇 |
| C 线程最小实现 | 先加 `midEvent` 覆盖；若来得及，加冷却与资源安全检查 |

## 2. 三资源节奏口径

沿用既有全局阈值，不另起一套数值。

| 资源 | 上限 | 初始 | 低值预警 | 危急线 | 归零处理 |
|---|---:|---:|---:|---:|---|
| 车轴 | 100 | 80 | 30 | 12 | 断轴补救 |
| 粮草 | 100 | 90 | 35 | 15 | 粮尽补救 |
| 神志 | 100 | 85 | 45 | 20 | 神志崩线补救 |

三资源分工：

| 资源 | 半途路遇中的职责 | 节奏要求 |
|---|---|---|
| 车轴 | 代表车队能不能继续走；常在坏路、泽水、强行穿越中受压 | 可以被低风险路遇小修，但不能在连续两次路遇中都被重扣 |
| 粮草 | 代表队伍撑多久；常作为停车、祭拜、避险的代价 | 是最常见交换货币，但粮草 <24 时不能再强制粮草大扣 |
| 神志 | 代表能不能看清路、读懂文本；污染路遇主要压神志 | 神志低可换幻路或线索，但神志 <30 时不能连续强推高污染 |

推荐试玩节奏：

| 阶段 | 推荐体验 | 资源目标 |
|---|---|---|
| 第 1-2 段 | 低/中风险路遇教学，让玩家知道半途会停 | 三资源一般保持在 65 以上 |
| 第 3-5 段 | 中风险抉择和一次高污染，迫使玩家补给或绕路 | 至少 1 项进入预警附近，但不应归零 |
| 赤水前后 | 神志压力上升，路线锁定和低神志幻路成立 | 神志可低到 30-45，但仍能看懂主消耗 |
| 裂隙前 | 高压收束，少抽普通随机，更多依靠地点事件和结局选择 | 不因最后一段普通路遇直接失败 |

## 3. 三类半途路遇

### 3.1 低风险提醒

用途：路线教学、轻微征兆、缓冲段打断。让玩家知道“路上也会发生事”，但不造成死局。

| 项 | 建议 |
|---|---|
| 适用风险 | 低风险路线，或连续事件后的缓冲路线 |
| 适用地貌 | `road`、少量 `market` |
| 当前可复用事件 | `wheel_omen`；没有新事件时也可低强度复用 `roadside_shrine` |
| 触发频率 | 低风险关键路线首次 100%；普通低风险 25%-30% |
| 选择数量 | 2 个即可 |
| 必须有的安全选项 | 一个小修/小绕/小安神选项，最大单项损耗不超过 4 |

资源范围：

| 效果类型 | 车轴 | 粮草 | 神志 | 备注 |
|---|---:|---:|---:|---|
| 停车整备 | +3 到 +6 | -1 到 -4 | 0 到 +2 | 用粮草换车轴或稳定感 |
| 不停赶路 | -2 到 0 | -1 到 -3 | -2 到 -5 | 让玩家感到有代价，但不杀人 |
| 轻微线索 | 0 到 +3 | -2 到 -4 | -2 到 +3 | 可给小提示，不给关键锁定 |

硬限制：

```text
低风险提醒不得让任一资源单项损耗超过 -5；
不得在玩家任一资源低于危急线时继续扣同资源；
不得成为同一路线反复刷收益的来源。
```

### 3.2 中风险抉择

用途：让玩家在车轴、粮草、神志之间做真实交换，适合作为第一章主要路遇密度。

| 项 | 建议 |
|---|---|
| 适用风险 | 中风险路线；高风险后的一次降压路线 |
| 适用地貌 | `market`、`road`、部分 `water` |
| 当前可复用事件 | `roadside_shrine`；水路可临时复用 `black_cloud` 的低压分支 |
| 触发频率 | 中风险关键路线首次 100%；普通中风险 45%-55% |
| 选择数量 | 2-3 个 |
| 必须有的安全选项 | 至少一个选项不扣当前最低资源，或用当前较高资源换补救 |

资源范围：

| 效果类型 | 车轴 | 粮草 | 神志 | 备注 |
|---|---:|---:|---:|---|
| 献粮问路 / 安神 | 0 到 +2 | -3 到 -7 | +3 到 +7 | 粮草换神志、方向或安全感 |
| 绕行避异 | -3 到 -7 | 0 到 -3 | 0 到 +3 | 车轴换少受污染 |
| 交易/祭祀线索 | -2 到 0 | -4 到 -8 | -2 到 +4 | 可给 flag/language/路线提示 |
| 抢行 | -4 到 -8 | 0 到 -3 | -3 到 -7 | 不给大收益，只换时间或少耗粮 |

硬限制：

```text
中风险抉择的非自愿单项损耗上限为 -7；
如果给 flag/language，允许总净损耗到 -10，但必须明确提示；
粮草 <24 或神志 <30 时，中风险事件不能只给“继续重扣粮草/神志”的选择。
```

### 3.3 高风险污染

用途：赤水、泽水、裂隙、梦图等高压段。它应当制造“世界不可信”的体验，但不能把随机半途停顿变成无解惩罚。

| 项 | 建议 |
|---|---|
| 适用风险 | 高风险路线；中后段关键推进 |
| 适用地貌 | `water`、`rift` |
| 当前可复用事件 | `black_cloud` |
| 触发频率 | 高风险关键路线首次 100%，但受冷却和安全地板限制 |
| 选择数量 | 2-3 个 |
| 必须有的安全选项 | 一个“避险/卸重/等待”选项，代价明确，不能同时重扣两项危急资源 |

资源范围：

| 效果类型 | 车轴 | 粮草 | 神志 | 备注 |
|---|---:|---:|---:|---|
| 卸重避险 | +0 到 +5 | -5 到 -8 | 0 到 -3 | 适合 `black_cloud`，用粮草换车轴安全 |
| 强行穿越 | -5 到 -9 | 0 到 -4 | -4 到 -8 | 可快行，但压车轴/神志 |
| 低神志幻路 | -2 到 -6 | 0 到 -4 | -8 到 -12 | 必须给 flag、捷径或终局线索 |
| 污染线索 | 0 到 +2 | -3 到 -6 | -7 到 -12 | 只在玩家明确选择凝视/听从/入梦时允许 |

硬限制：

```text
普通高风险路遇的非主动单项损耗上限为 -8；
主动禁忌、低神志、幻路选择可以到 -12，但必须给路线解锁、flag 或结局推进；
高风险污染不得连续两段都必触发；
濒死补救后的 2 段内禁止抽同资源高损耗污染。
```

## 4. 触发频率与冷却规则

### 4.1 基础触发率

| 路线风险 | 首次关键路线 | 非关键路线 | 已触发过同路线 | 备注 |
|---|---:|---:|---:|---|
| 低 | 100% 或 P0 必触发 | 25%-30% | 10%-15% | 教学和缓冲为主 |
| 中 | 100% 或 P0 必触发 | 45%-55% | 20%-30% | 第一章主要密度 |
| 高 | 100%，但受保护规则限制 | 65%-80% | 35%-45% | 不可连续高污染 |
| 裂隙终点段 | 不建议普通随机 | 0%-40% | 0%-20% | 优先交给地点/结局事件 |

### 4.2 连续触发限制

C 线程若实现触发器，建议按下面顺序判断：

```text
1. 如果当前有 pendingRoute 未处理，不允许再开新路遇。
2. 如果任一资源 <= 0，进入 crisisEvents，不触发普通路遇。
3. 如果刚完成 crisisEvents 后还在 2 段保护期，禁止同资源高损耗路遇。
4. 如果上一段已经触发高风险污染，本段高风险路遇降为 35% 或替换为低风险提醒。
5. 如果连续 2 段已经有半途路遇，第 3 段只允许低风险提醒或跳过。
6. 如果距上次半途路遇已经 2 段，且当前路线是关键推进，则本段提高到必触发。
```

### 4.3 资源安全地板

路遇显示前先做一次“至少有一个可承受选择”的检查。

| 检查 | 安全标准 | 不通过时 |
|---|---|---|
| 归零检查 | 路线成本结算后没有资源 <=0 | 直接进入对应危机补救，不开普通路遇 |
| 危急检查 | 车轴 >12、粮草 >15、神志 >20 | 只允许低风险提醒或危机补救 |
| 低值检查 | 车轴 >=20、粮草 >=24、神志 >=30 | 允许中风险；禁止高污染重扣当前最低资源 |
| 选择检查 | 至少 1 个选择不会让任何资源 <=0 | 替换为保底路遇或跳过半途 |
| 连续坏事检查 | 连续 2 个坏事件后，本次必须有资源恢复、flag、language 或低风险出口 | 插入补救/喘息/线索 |

坏事件定义沿用 A-004/A-006：

```text
没有获得资源、旗标、语言、路线线索或结局进展，
且总资源净损耗 >= 8，
或任一资源单项净损耗 >= 6。
```

## 7. 可直接给 C 的触发伪代码

如果 C 线程愿意在 `app.js` 里做最小保护，可以按这个顺序写：

```js
function shouldTriggerMidEvent(route, state) {
  if (!route.midEvent) return false;
  if (state.pendingRoute) return false;

  const afterRoute = previewResources(state.resources, route.cost);
  if (afterRoute.axle <= 0 || afterRoute.grain <= 0 || afterRoute.sanity <= 0) {
    return false; // 交给 crisisEvents
  }

  const last = state.lastMidEventSummary;
  const isHigh = route.risk === "高";
  const lowFloor = afterRoute.axle < 20 || afterRoute.grain < 24 || afterRoute.sanity < 30;

  if (lowFloor && isHigh) return false;
  if (last?.tier === "high" && isHigh) return false;
  if (state.recentMidEventCount >= 2 && route.risk !== "低") return false;

  return true;
}
```

如果本轮只改数据、不改触发器，则按第 6.1 的 P0 新增清单执行，暂时不要给第 6.2 的路线批量挂高风险事件。

## 8. 死局保护规则

### 8.1 绝对禁止

| 禁止项 | 原因 |
|---|---|
| 连续 2 次高风险污染必触发 | 会把高压变成随机处刑 |
| 玩家任一资源在危急线附近时，继续强制同资源大扣 | 会让补救规则失效 |
| 高风险路线后只剩高风险出口 | 玩家没有真实选择 |
| 终点路线同时有高成本、锁定、高污染、无补给 | 容易让玩家在终点前无解 |
| 低神志隐藏所有真实资源方向 | 失败会显得不公平 |

### 8.2 至少保留一个可活选择

每个半途路遇在显示时，都必须满足：

```text
至少 1 个选择不会让任何资源 <= 0；
至少 1 个选择不扣当前最低资源，或能恢复当前最低资源；
如果所有选择都会进入危机，则本次不显示普通路遇，改进 crisisEvents。
```

### 8.3 高风险后的降压

高风险路遇后，下一段按以下规则降压：

| 上一段结果 | 下一段规则 |
|---|---|
| 选择导致任一资源单项 -8 或更低 | 下一段不触发高风险污染 |
| 触发 `black_cloud` 且选择强行穿越 | 下一段优先低/中风险或地点补给 |
| 进入或刚完成 crisisEvents | 2 段保护期，不抽同资源高损耗 |
| 连续 2 段都有半途路遇 | 第 3 段跳过，除非是低风险提醒 |
