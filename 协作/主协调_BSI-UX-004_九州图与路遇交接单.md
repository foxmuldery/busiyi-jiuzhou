## 一、最新产品判断

当前 WebDemo 的核心体验应拆成两个层级：

1. 九州图页面：负责战略选择，参考 Out There 的星图式路线决策。
2. 地点页面：负责停靠体验，展示横向旅途大图、地点志、补给、剧情选择，吸收西部横向旅行游戏的路上体感。

玩家主循环应为：

九州图观察路线与资源 -> 选择下一站 -> 横向行程动画 -> 可能半途停下触发路遇 -> 处理路遇 -> 抵达城镇 -> 看地点志 -> 搜集补给 -> 处理地点剧情 -> 回到九州图。

## 二、必须保留的设计原则

- 路线选择像 Out There：每次前进都要看到消耗、风险、目的地可观察资源。
- 地点体验像横向旅行：车队沿长图移动，日夜、云、远景、前景体现远行感。
- 地图不是装饰：玩家必须知道自己在九州哪一段、离九州裂隙还差多远。
- 城镇不是纯剧情：每个地点有差异化补给，且补给只可搜集一次。
- 路途中可以停下：不是每次路线都直接到达，可以在半路触发路遇事件。
- 路遇第一版应可控，不做完全随机，避免测试不可复现；后续再扩展为带权重的随机池。
