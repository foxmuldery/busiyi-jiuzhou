## 结论

当前第一章 Web Demo 已经具备继续试玩的基础：三资源初始值 `车轴 80 / 粮草 90 / 神志 85` 可用，地点补给覆盖较足，固定路遇和 8 个随机路遇已经能形成“选路 -> 半途 -> 抵达 -> 补给”的循环。

主要风险不是“太松”，而是 5 分钟试玩里半途路遇密度偏高：多数路线已有固定 `midEvent`，同时随机路遇还有基础 28% 抽取率，资源告急时会升到更高。虽然系统一次路线只会选一个路遇，但玩家几乎每段都可能被打断，容易觉得“刚选路就又弹事件”。

数值强度总体可接受，局部偏紧：

| 项 | 判断 | 原因 |
|---|---|---|
| 车轴 | 合适略紧 | 高风险路线与水路多，修车补给较多，暂不必大改 |
| 粮草 | 压力合适 | 初始 90，地点补粮很多，但高风险路线粮耗可到 -11 |
| 神志 | 中后段偏紧 | 赤水、断碑、裂隙路线连续扣神志，且有极端 `神志 -75` 选项 |
| 随机路遇密度 | 偏密 | 固定路遇覆盖高，随机基础概率仍有 28% |
| 补给强度 | 略慷慨但可接受 | 当前目标是 5 分钟跑顺，补给慷慨优先于首局死局 |
| 死局防护 | 基本有，但需补规则门槛 | 已有危机事件和补救倾向；缺少“高压连续段”的明确降密建议 |

本轮建议：不重构、不加系统，只做小数值微调和两条体验门槛。先让 5 分钟试玩顺，再谈完整平衡。

## 可直接进入 Demo 的内容

1. 保留当前初始资源：车轴 80、粮草 90、神志 85。
2. 保留当前危机事件：断轴、粮尽、神志崩线均有 2 个补救选择。
3. 保留随机路遇 8 个事件池；它们的资源奖惩范围大体健康。
4. 保留资源告急反馈：资源卡闪光、飘字、告急浮层对数值理解有帮助。
5. 保留地点补给偏慷慨的口径；试玩版先防卡死。

## 5 分钟试玩路径

### 路径 1：保守主线，最适合验收核心循环

```text
中原驿 -> 故王道 -> 雷泽浅畔 -> 赤水外滩 -> 巫咸断碑
```

| 段落 | 期望体验 | 数值目标 |
|---|---|---|
| 中原驿 -> 故王道 | 低风险教学，第一次半途路遇 | 三资源仍在 70 以上 |
| 故王道 -> 雷泽浅畔 | 中风险水路，开始有资源压力 | 至少 1 项接近 60 |
| 雷泽浅畔 -> 赤水外滩 | 高风险污染感出现 | 神志可到 45-60，但不应低于 35 |
| 赤水外滩 -> 巫咸断碑 | 回到中风险解释路线 | 玩家仍能补神志或看懂路线 |

验收标准：不连续触发两个高压污染；抵达断碑前至少还剩 1 个可补给地点选择。

### 路径 2：青丘支线，验收低压绕行与神志恢复

```text
中原驿 -> 青丘外邑 -> 无名祠 -> 雷泽浅畔
```

| 段落 | 期望体验 | 数值目标 |
|---|---|---|
| 中原驿 -> 青丘外邑 | 边邑低压，狐灯稳定神志 | 神志不应掉破 70 |
| 青丘外邑 -> 无名祠 | 中风险祭所，粮草换神志 | 粮草有消耗但可控 |
| 无名祠 -> 雷泽浅畔 | 进入水路前给一次选择压力 | 至少一项资源接近预警但不进危机 |

验收标准：青丘线应比直冲赤水更安全，让玩家感到绕路有意义。

### 路径 3：黑齿市支线，验收补给强度和交易代价

```text
中原驿 -> 废关 -> 黑齿市 -> 巫咸断碑
```

| 段落 | 期望体验 | 数值目标 |
|---|---|---|
| 中原驿 -> 废关 | 中风险旧关，车轴可补 | 车轴压力被展示但能修 |
| 废关 -> 黑齿市 | 粮草压力出现，市场可交换 | 粮草掉到 55-70 合理 |
| 黑齿市 -> 巫咸断碑 | 用交易换路线推进 | 神志可能到 45-60，不应直接崩线 |

验收标准：交易线不能变成免费补给；至少让玩家用车轴或神志换粮草。

## 数值风险

### 资源节奏

| 资源 | 当前判断 | 建议 |
|---|---|---|
| 车轴 | 合适略紧 | 不改初始值；只避免连续水路/高风险都重扣车轴 |
| 粮草 | 合适略松 | 不削地点补给；只把高粮耗路线的连续出现控制住 |
| 神志 | 中后段偏紧 | 优先处理极端 `神志 -75` 和连续裂隙高压 |

### 随机路遇密度

当前密度偏密，但不是必须立刻大改。原因：

- 固定 `midEvent` 已覆盖多数路线，第一遍走路线基本会停。
- 随机路遇基础概率为 28%，资源压力和坏运会继续抬高。
- 固定与随机不会叠加成两个事件，但随机可能替换固定事件，导致玩家感觉每段都有意外。

试玩版建议目标：

| 路线段数 | 期望半途路遇 |
|---:|---:|
| 3 段 | 1-2 次 |
| 4 段 | 2 次左右 |
| 5 段 | 2-3 次 |

超过这个密度，5 分钟试玩会从“旅途有变”变成“不断弹选择”。

### 补给强度

补给略慷慨但应保留。理由：

- 当前目标是顺畅试玩，不是高难生存。
- 地点补给能防止随机路遇造成早期死局。
- 资源闪光和告急反馈刚接入，需要让玩家有机会看到“掉了还能补”的循环。

只需避免两种情况：

1. 补给无代价连续出现，让路线选择变浅。
2. 低资源时仍连续推高污染，让补给显得无效。

## 死局防护最低规则

这些规则是进入下一轮试玩前的最低线：

1. 任一资源归零时，先进入对应危机事件，不直接失败。
2. 任一资源低于告急线后，下一次随机路遇优先抽补给、休整、线索或救援候选。
3. 每个半途路遇至少有 1 个选择不会让任何资源立刻归零。
4. 高风险路遇后，下一段不得继续高风险污染必触发。
5. 连续 2 段触发半途路遇后，第 3 段必须跳过，或只允许低风险补救。
6. 神志 <= 30 时，不再出现会单次扣神志超过 8 的普通随机路遇。
7. 赤水、断碑、裂隙前的终点路线，不叠加额外普通随机惩罚。
8. 低神志文本可以污染，但路线卡必须保留至少一个真实资源方向。

## 需要 C 线程实现的内容

C 线程不需要做大模拟器。本轮只建议落地以下小改：

1. 把随机路遇基础概率从 28% 下调到 20%-22%。
2. 把随机路遇上限从 74% 下调到 60%-65%。
3. 加“连续 2 段有路遇，第 3 段跳过或低风险化”的保护。
4. 加“高风险路遇后下一段不出高风险污染”的保护。
5. 临时隐藏或削弱 `red_bones` 的 `神志 -75` 选项。
