# BSI-B-030 《山野逸事》第三部志怪素材接入规范

日期：2026-06-24

## 1. 目的

《山野逸事》第三部里已有大量古典志怪文学典故、人物、异象、民俗和故事设定。它们不应被当成“完整游戏剧情”硬塞进《不思异：九州》，而应先由 AI 做批量初拆，再交给掌握这部分旧内容的志怪内容协作者确认，最后转成可筛选、可转译、可测试的游戏事件。

本规范的目标不是让志怪内容协作者做“素材卡录入员”，也不是让他承担《不思异：九州》的主策划、系统策划或总编剧职责，而是让他把自己对旧项目部分内容的判断用在最高价值的位置：

- AI 负责：批量初拆、来源归并、意象标签、初步玩法映射、风险初标。
- 志怪内容协作者负责：旧项目设定解释、取舍建议、世界观定性、情绪功能、玩家欲望、不可丢的意象、关键文本。
- 游戏策划负责：玩法类型、资源代价、选项结构、路线影响、失败保护。
- 主线程负责：审核边界、转译质量、接入优先级、进入 Web Demo 的验收。

一句话原则：

> AI 先把旧设定拆成候选“志怪素材卡”，志怪内容协作者只做判断、解释和批注，再由游戏侧转成“地点 / 路遇 / 补给 / 危机 / 结局”。

## 2. 接入分层

### 第一层：AI 初拆素材卡

填写人：AI / 资料整理，可由主线程批量生成。

用途：把旧剧集设定、典故、人物、异象、民俗先拆成可浏览候选，不要求一次准确。

主要回答：

- 这个典故来自哪里？
- 它最有意思的冲突是什么？
- 有哪些可以被玩家记住的意象？
- 适合放在九州世界的哪一类地点或遭遇里？
- 有没有一句可以直接进入游戏的短文本？

AI 初拆允许不完美，但必须保留来源线索和不确定备注，不能把猜测写成定论。

### 第二层：志怪内容协作者确认

填写人：《山野逸事》第三部志怪内容协作者 / 原项目内容设定交接人。

用途：发挥人的策划判断，不做批量录入。

志怪内容协作者只需要回答：

- 这个素材是否值得进入《不思异：九州》？
- 它在九州里是真相、谣言、禁忌、骗局，还是地方风俗？
- 它要制造哪种情绪：敬畏、恐惧、怜悯、诱惑、荒诞，还是成就感？
- 玩家在这里被诱惑什么？
- 玩家害怕失去什么？
- 哪个意象绝不能删？
- 低神志时它如何变得更恐怖或更疯狂？

禁止在这一层要求填写：

- 车轴 / 粮草 / 神志具体数值。
- 概率、权重、保底规则。
- 事件 ID、代码字段。
- 完整关卡结构。

### 第三层：游戏转译卡

填写人：游戏策划、主线程、可由 AI 先草拟。

用途：把文学素材转成可选择、可结算、可被测试的游戏事件。

主要回答：

- 这是地点志、城镇事件、半途路遇、补给、危机还是结局？
- 玩家面前出现什么选择？
- 选择带来什么代价和收益？
- 对车轴、粮草、神志哪一项最有关系？
- 是否会影响后续路线、图鉴、语言、结局或资源保底？

### 第四层：可接入事件卡

填写人：游戏策划、技术接入、主线程审核。

用途：进入 Web Demo 的数据化内容。

主要回答：

- 标题、诗引、正文、选项、结果文本是否齐全？
- 选项数量是否不超过 4 个？
- 资源变化是否能跑通，不会把玩家直接推入死局？
- 是否有图像需求、音效需求、神志变体文本？
- 是否符合当前 UI 一屏显示预算？

## 3. 工作流程

### 步骤 1：AI 批量初拆

从《山野逸事》第三部中由 AI 先拆出素材卡。建议第一批先拆 30 张，不追求全量。

每张卡只保留一个核心：

- 一个怪物。
- 一个仪式。
- 一个地方。
- 一个禁忌。
- 一个交易。
- 一个梦。
- 一个灾异。
- 一个失败结局。

不要把一整集剧情塞进一张卡。

AI 初拆每张卡必须带：

- 原始出处或旧项目位置。
- 一句话素材核。
- 3 到 5 个意象标签。
- 推荐玩法用途。
- 初步风险备注。
- “需要内容策划判断的问题”。

### 步骤 2：志怪内容协作者确认

志怪内容协作者不逐项重填素材卡，只看 AI 初拆候选清单，做内容判断、旧设定解释和简短批注。

建议评审节奏：

- AI 一次提交 20 到 30 张候选。
- 志怪内容协作者每轮只选出 6 到 8 张最值得进游戏的内容。
- 每张被选中的内容只写 3 到 7 条批注。
- 没被选中的内容只标 `暂缓`、`丢弃` 或 `二阶段再看`。

志怪内容协作者批注优先级：

1. 为什么值得进九州。
2. 不可丢的意象。
3. 玩家欲望和玩家恐惧。
4. 它在世界观中的性质。
5. 低神志版本的扭曲方向。
6. 一句最有味道、可进入游戏的文本。

### 步骤 3：主线程初筛

每张素材卡进入三色状态：

| 状态 | 含义 | 处理 |
| --- | --- | --- |
| `green` | 很适合《不思异：九州》 | 进入游戏转译 |
| `yellow` | 气质好，但需要改写或拆分 | 主线程给改写建议 |
| `red` | 与玩法不合、风险高、太现代、太影视化 | 暂存，不进入 Demo |

初筛标准：

- 是否有明确的玩家行动？
- 是否能形成 2 到 4 个选择？
- 是否能影响车轴、粮草、神志之一？
- 是否有强山海/志怪意象？
- 是否不会破坏当前九州主线？

### 步骤 4：游戏化转译

把 green / yellow 素材转成玩法用途：

| 原始素材倾向 | 推荐转译 |
| --- | --- |
| 地方、墟市、祭坛、废关 | 地点志 / 城镇事件 / 补给点 |
| 怪物、异兽、鬼魅、路上异象 | 半途路遇 / 随机事件 |
| 禁忌、诅咒、误食、失路 | 神志危机 / 资源危机 |
| 古器、残卷、巫文、旧名 | 行囊物 / 语言线索 / 结局钥匙 |
| 大型传说、宿命、献祭 | 章节结局 / 裂隙前事件 |

### 步骤 5：主线程验收

主线程检查：

- 文学气质是否保留。
- 玩家是否看得懂下一步。
- 是否有明确资源压力。
- 是否有探索成就感。
- 是否适合小横屏弹窗。
- 是否需要图像、音效、动画素材。

验收通过后才进入 Web Demo 或下一轮实现单。

### 步骤 6：小批量接入

不要一次导入大量旧设定。建议每轮只接：

- 2 个地点。
- 4 个地点事件。
- 4 个半途路遇。
- 1 个危机。
- 1 个结局或后续伏笔。

每轮接入后必须跑试玩，而不是只看文本。

## 4. 分工边界

### AI / 资料整理

负责：

- 批量拆旧资料。
- 归并重复来源。
- 标注典故、意象、人物、地点、禁忌。
- 生成初步素材卡。
- 给出可能玩法用途。
- 提醒来源、表达、清权风险。

不负责：

- 最终取舍。
- 世界观定性。
- 关键文本定稿。
- 玩法数值定稿。

### 《山野逸事》志怪内容协作者

负责：

- 从自己掌握的《山野逸事》情节设定、故事和典故中，选择哪些素材值得进入《不思异：九州》。
- 提炼故事核。
- 判断世界观功能：真相、谣言、禁忌、骗局、地方风俗、古国遗痕。
- 判断情绪功能：恐惧、敬畏、诱惑、怜悯、荒诞、成就感。
- 定义玩家欲望与玩家代价。
- 指出不可丢的意象、动作、关系或句子。
- 写或批准 1 到 3 句关键文本。

不负责：

- 批量拆卡。
- 《不思异：九州》整体方向。
- 游戏主线结构。
- 数值平衡。
- 事件概率。
- 路线池。
- 资源保底。
- UI 一屏预算。

### 游戏策划 / 主线程

负责：

- 判断玩法类型。
- 设计选项。
- 设置资源变化。
- 拆分为地点、路遇、补给、危机、结局。
- 保证不死局。
- 控制第一章节奏。

### 技术接入

负责：

- 数据字段合法。
- 文本不溢出。
- 图片、音频、图鉴、复盘能显示。
- QA 脚本覆盖。

### 风险复核

负责：

- 不照搬现代译文、百科、影视剧表达。
- 不直接复用旧项目中未清权的美术、音乐、剧本文本。
- 不把“内部参考”标成正式可商用素材。

## 5. 进入 Demo 的最低标准

一条素材进入 Web Demo 前，至少要满足：

- 有一句话故事核。
- 有 1 个明确意象。
- 有 1 个玩法类型。
- 有 2 到 4 个玩家选择。
- 每个选择有结果文本。
- 至少影响车轴 / 粮草 / 神志之一，或给出明确原因为什么不影响。
- 有推荐配图类型。
- 有神志低时可变恐怖或错乱的方向。
- 有来源记录和风险备注。

## 6. 推荐批次

第一批建议不要追求最宏大的设定，而选最容易进入玩法的内容：

1. 路上能遇到的异兽。
2. 可以交易或献祭的墟市。
3. 会改变粮草或神志的食物/药草。
4. 会损车轴的地形或机关。
5. 会让人误认路标、名字、亲人的异象。
6. 可以放进图鉴的地方风俗。

宏大世界观、主线宿命、人物群像先暂缓，等玩法稳定后再进入第二阶段。

## 7. 交付格式

每批素材先由 AI 交一份《AI 初拆候选清单》，文件命名建议：

```text
山野逸事AI初拆候选_批次_日期.md
```

例如：

```text
山野逸事AI初拆候选_第一批_20260624.md
```

内容策划确认后，再形成《内容策划确认表》：

```text
山野逸事内容策划确认_批次_日期.md
```

单张素材如果进入游戏转译，再生成：

```text
山野逸事素材卡_序号_短名.md
```

例如：

```text
山野逸事素材卡_001_食梦井.md
山野逸事素材卡_002_无面渡卒.md
山野逸事素材卡_003_赤犬借粮.md
```

主线程通过后，再生成：

```text
山野逸事转译卡_序号_短名.md
```

最终进入 Web Demo 的内容，再由 C 线或主线程转入数据文件。

## 8. 当前结论

最适合团队的办法不是让编剧学习完整游戏策划，而是建立“文学素材到游戏事件”的中间层。

这样他们可以继续做自己擅长的事：找典故、写气氛、提炼志怪意味；游戏侧再负责把它们变成可选择、可结算、可复盘的互动内容。
