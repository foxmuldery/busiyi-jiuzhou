# 《不思异：九州》突发 / 随机事件视觉一致性生成方案

> 用途：修正当前突发事件、随机路遇生成图风格不一致的问题。  
> 日期：2026-06-16  
> 结论：不要继续为每个事件单独生成完整插图；先生成统一事件模板和可替换符号层。  
> 范围：固定路遇 `routeEvents`、随机路遇 `randomRouteEvents`、危机事件 `crisisEvents` 的视觉规则。

## 1. 当前问题

现在事件生成效果不一致，主要不是提示词不够“古风”，而是生成对象选错了：

- 每个事件都像独立海报，构图、光源、笔触、主体比例不统一。
- 有的事件图像太具象，有的太抽象，放进同一个事件面板会跳风格。
- 突发/随机事件本质是“半途流程”，不是地点插画；如果每条路遇都换完整大图，会破坏横向旅途舞台连续性。
- 低神志版本不应该换成另一张完全不同的图，而应在同一事件底板上叠加污染层。

正确方向：

```text
统一事件底板 + 事件类型符号 + 地貌色温 + 神志污染叠层 + 资源变化图标
```

更推荐的生产方式是“统一底板 + 内容图二次加工”：

```text
先定 6 张统一事件底板
-> 再生成事件内容图 / 主符号
-> 把内容图统一转成暗帛书、低饱和、朱砂墨线风
-> 最后叠回同一套事件面板
```

这样即使不同事件的原始内容来自不同批次，也能通过后处理统一纸纹、边缘、色温和构图重心。

## 2. 事件视觉系统总规则

事件视觉分 5 层：

```text
EV-BG 事件底板
EV-SYM 事件主符号
EV-FX 氛围/动作效果
OV-SAN 神志污染
UI-CHIP 资源/选择图标
```

### 2.1 EV-BG 事件底板

同一类事件共享底板，不为每个事件重画大背景。

底板只表达“事件发生的环境类型”：

- 路上
- 水泽
- 市集
- 祭祠/碑文
- 裂隙/梦图
- 危机/濒死

### 2.2 EV-SYM 事件主符号

主符号是事件的识别点，可以是：

- 车轮 / 轴心
- 草束 / 粮袋
- 里程牌 / 断碑
- 黑齿价码 / 影子摊
- 异兽剪影
- 第二月亮 / 地图裂纹

主符号不要做成完整角色大插画，应像古图符号、异闻标记、旧地图上的标识。

### 2.3 EV-FX 氛围/动作效果

用于表现正在发生的小过程：

- 鸣声线
- 风痕
- 水纹
- 尘土
- 朱砂线
- 墨迹扩散
- 纸面呼吸
- 地图翻卷

### 2.4 OV-SAN 神志污染

同一张事件图在神志高/中/低时不换主体，只改变叠层：

| 神志 | 表现 |
---|---|
| 高 | 底板清楚，主符号稳定，少量云雾 |
| 中 | 符号有重影，边缘出现朱砂裂纹 |
| 低 | 出现重复符号、第二路线、反字、错位阴影 |

### 2.5 UI-CHIP 资源/选择图标

事件图不承担数值说明。资源变化仍由 UI 芯片显示：

- 车轴变化
- 粮草变化
- 神志变化
- 巫咸古辞 / 线索
- 路线显影

### 2.6 二次加工层

如果需要让已有事件内容图保持一致，不要直接把原图放进 Demo。先做统一二次加工：

| 加工项 | 目标 |
---|---|
| 裁切 | 统一为横向小预览，主体居中略偏左，右侧留给文字 |
| 去色 | 降低饱和度，压掉彩色漂移 |
| 纸纹 | 加同一张暗帛书/旧纸纹理 |
| 边缘 | 四周加轻微墨迹褪边，避免硬切 |
| 线条 | 用墨线/朱砂线增强主符号轮廓 |
| 色温 | road 偏土黄，water 偏灰蓝/赤褐，market 偏黑石，rift 偏暗蓝 |
| 污染 | 中低神志只加 OV 层，不换整张图 |

二次加工后的图片只作为“事件小图”，不作为整屏背景。

## 4.6 推荐生产流水线

```text
Step 1 生成 EVBG 统一底板
Step 2 生成 EVSYM 主符号拼版
Step 3 从拼版中切出符号或让模型生成单个内容图
Step 4 用 EVPOST 模板统一二次加工
Step 5 输出事件小图 EVENTART-xxx
Step 6 接入 Demo 时仍保留前端标题、正文、按钮和资源芯片
```

命名示例：

```text
EVENTART-001_鸣蛇入轴_荒外底板_车轴符号_清醒.png
EVENTART-001_鸣蛇入轴_荒外底板_车轴符号_低神志.png
EVENTART-002_祝余苦汤_荒外底板_草束符号_清醒.png
EVENTART-003_巫咸反里_祭祠底板_反里程牌_低神志.png
```

每个事件最多保留两版：

- `清醒版`
- `低神志版`

不要再为高/中/低神志各生成完全不同的三张图。中神志通过叠层过渡即可。

## 7. 生成后筛选标准

可用：

- 6 张底板风格统一，光源、纸纹、色彩一致。
- 主符号能在 80px 内识别。
- 叠层能用 screen/lighten，不会洗白画面。
- 不含可读文字、水印、现代元素。
- 事件图放入小横屏时不会抢掉事件正文。

禁用：

- 单张像独立海报，无法和其他事件统一。
- 画面出现完整人物大主体。
- 出现真实可读文字。
- 过度恐怖、血腥、怪物正脸。
- 亮绿色背景抠不干净或边缘严重发光。

## 8. 后续实现建议

### 8.1 C 线程接入方式

建议给事件增加视觉字段，而不是给每个事件写死一张图：

```js
visual: {
  template: "EVBG-001",
  symbol: "snake_axle",
  tone: "road",
  pollution: "EVOV-003",
  icon: "repair"
}
```

随机事件可以按 `pool` 和 `routeTags` 自动选择模板：

```text
pool supply -> EVBG-001/002 + grain symbol
pool trade -> EVBG-003 + trade symbol
pool clue -> EVBG-004/005 + clue symbol
pool sanity -> EVBG-004/005 + sanity pollution
pool risk -> EVBG-001/002/005 + risk symbol
```

### 8.2 UI 呈现方式

事件面板里只显示小预览：

```text
左侧或上方小图：事件底板 + 符号 + 叠层
右侧/下方：标题、标签、事件文本、选择按钮
```

不要让事件插图变成整屏背景。

### 8.3 事件内容图二次加工接入方式

如果后续已经有单张事件图，C 线程不必全部弃用，可以按以下规则接：

```text
原事件图
-> 裁成固定比例小图
-> 加统一 EVPOST 纸纹/暗角/朱砂线
-> 根据事件 visual.template 叠底板色
-> 根据 sanityBand 叠污染层
-> 输出到 eventArt
```

建议前端字段：

```js
eventArt: {
  base: "EVBG-001",
  subject: "EVENT-SUBJECT-snake_axle",
  post: "EVPOST-001",
  pollution: "EVOV-003",
  crop: "center-left"
}
```

这样可以保留“内容图”的识别度，同时用统一底板和统一后处理把它们收进同一个视觉系统。

## 10. 给用户的直接判断

后续最该生成的不是“更多随机事件图”，而是：

```text
事件底板、事件符号、事件污染层、行动过程图标、二次加工模板。
```

先统一视觉语法，再扩充具体事件。否则随机事件越多，风格越散，Demo 越不像同一个游戏。
