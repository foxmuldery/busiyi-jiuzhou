## 4. 暂不建议接入

- AI 生成的“九州”字形候选：存在字形误读风险，不能当正式标题。
- 复杂角花和大纹样：容易让小横屏拥挤。
- 绿边残留明显的单个切片：试装后再逐个修，不批量上屏。
- 任何未登记来源和授权状态的素材：只能当内部 Demo 候选。

## 5. 给 C 线程的执行口径

下一步只做“高频 UI 小修”，不做整体重构：

1. 资源芯片：给车轴、粮草、神志补图标，维持现有数字和进度条。
2. 动作按钮：补给、修轴、搜粮、休整等按钮加小图标，不改变按钮数量。
3. 地图节点：优先替换当前/可达/未知/危险/目的节点，路线只做轻量贴图或 CSS 背景。
4. 风险章：事件卡和路线卡统一用低/中/高风险章。
5. 面板底板：只在事件弹窗、选择按钮、日志条上试一版，不影响文字可读。
6. 小横屏验收：必须看 `844x390 / 932x430 / 1280x720` 三档。

## 6. 验收标准

- 资源与风险能不看说明就识别。
- 事件文字不被底纹遮挡。
- 小横屏按钮不挤、不糊、不误触。
- 地图节点状态比现在更清楚。
- 页面仍然是“试玩可用界面”，不是宣传图或纯装饰页面。

## 7. 风险与边界

- 本批素材为 AI 生成候选，授权状态仍需后续统一登记和复核。
- 透明清理已做一轮，但少量图标边缘可能仍需人工微调。
- UI 试装要以玩法信息为先，不能为了装饰牺牲资源、路线、事件文本和按钮可读性。
