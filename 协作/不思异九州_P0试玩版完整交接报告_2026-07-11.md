## 2. 产品目标

### 2.1 核心定位

《不思异：九州》是一款以《山海经》气质为灵感的横屏远行生存叙事游戏。玩家扮演王朝派出的采风使，率领车队进入神秘九州，在有限资源和不可靠见闻中选择路线、处理异象、搜集补给、记录风土与异闻。

当前 P0 目标不是做完整游戏，而是验证以下核心体验是否成立：

1. 横屏手机优先的“远行感”是否比纯节点地图更有沉浸感。
2. `车轴 / 粮草 / 神志` 三资源是否能支撑清晰的风险决策。
3. `九州图战略选择 + 横向旅途舞台 + 地点事件弹窗` 的层级是否能跑通。
4. 山海经式地点志、古诗词诗引、异象事件、低神志污染文本是否有吸引力。
5. 玩家是否愿意继续多走一站，并愿意回传 5 分钟试玩反馈。

### 2.2 体验关键词

- 山海经
- 九州远行
- 车队生存
- 资源压力
- 神志污染
- 节点路线选择
- 采风入册
- 古诗词诗引
- 假动态横向风景
- 手机横屏 UI

### 2.3 当前不做的内容

- 不做横版动作战斗。
- 不做开放世界自由移动。
- 不做复杂角色养成。
- 不做正式账号系统。
- 不做正式商业发行包。
- 不把当前音频和临时素材标为最终可商用资产。

## 3. 当前功能清单

### 3.1 核心循环

当前试玩版已跑通以下主循环：

1. 从公开入口或本地入口开始干净新局。
2. 在地点页阅读地点志、诗引和当前遭遇。
3. 处理地点事件。
4. 选择采风写法，将地方见闻写入采风册。
5. 执行一次性补给。
6. 切到九州图，查看可走路线、路线消耗、补给倾向和半途风险。
7. 选择路线并启程。
8. 半途触发路遇事件。
9. 抵达下一地点，继续循环。
10. 最终抵达九州裂隙并进入结局 / 复盘。

### 3.2 三资源系统

正式资源名：

- `车轴`：对应车队行进与交通损耗，接近原型参照中的燃料逻辑。
- `粮草`：对应队伍续航与生存压力。
- `神志`：对应心智稳定、异象污染和文本变化。

当前机制：

- 路线会消耗资源。
- 地点补给会回复或转换资源。
- 事件选择会造成资源变化。
- 神志低时，地点志和遭遇文本会变得更不可靠、更恐怖。
- 资源触底后进入最小危机补救，而不是立即硬死。
- 连续补救过多或坏运过高会进入硬失败。

### 3.3 地图与路线

当前采用两层空间表达：

- 九州图：参考 Out There 的战略路线选择，显示节点、迷雾、可走路线、路线消耗和目的地倾向。
- 横向旅途舞台：参考横屏西行 / Darkest Dungeon 式“假动态远行”，以风景滚动、云雾、污染层、阶段提示表达行进感。

当前数据规模：

- 地点：14 个
- 路线：33 条
- 地点事件：14 个
- 半途路遇事件：13 个
- 随机路遇事件：8 个
- 危机事件：3 个
- 结局定义：2 个
- 地形类型：`road / market / water / rift`

### 3.4 地点志与山海经内容

每个地点都有：

- 地点名称
- 短标签
- 地点志
- 抵达句
- 补给发现句
- 专属地点事件
- 地点插图
- 采风题材
- 采风线索
- 一次性补给

当前地点包括中原驿、故王道、废关、鸟鼠夹道、无名祭所、雷泽、羽民渡、黑齿市、赤水、断碑、归墟裂隙等。

### 3.5 采风机制

采风是当前版本后期新增但已经接入主循环的核心机制。玩家在地点事件后选择写法，将当前地点的见闻写入采风册。

当前采风写法：

- 颂
- 讽
- 哀
- 谶

当前已实现：

- 每个地点有 `fieldNote` 题材层。
- 每个地点有采风类别、题材名、线索和资源取舍。
- 采风选择卡以类别圆标 + 题材芯片显示，减少手机横屏文字负担。
- 采风结果进入日志、地点详情、采风册回看、本局复盘。
- 采风写法会产生王朝案牍式后果短句。

### 3.6 UI 与交互

当前 UI 目标是“更像横屏手机游戏，少用长文字面板，多用图标、数字、弹窗和压缩信息”。

5. 事件选择是否有取舍，而不是纯文本阅读。
6. 横向风景是否比纯地图更有远行感。
7. 弹窗、按钮、图标、资源数字是否适合手机横屏。

### 8.2 技术审阅重点

建议重点看：

1. `app.js` 单文件过大，后续是否需要拆分状态机、渲染、音频、反馈工具。
2. `data.js` 数据结构是否适合内容策划长期维护。
3. `qa-check.js` 是否覆盖了后续最容易回归的交互。
4. 线上部署包和源码目录是否需要纳入 Git。
5. 发布流程是否需要从手工打包升级为可重复构建。
6. localStorage 存档和 URL 调试参数是否需要版本迁移策略。
7. 密码门是否只用于内部测试，正式测试是否需要更可靠的访问控制。

### 8.3 内容审阅重点

建议重点看：

1. 地点志是否每个地点都有差异。
2. 低神志文本是否只是恐怖化，还是会影响判断。
3. 采风写法“颂 / 讽 / 哀 / 谶”是否足够有选择感。
4. 王朝案牍后果是否能带来“我写下的东西会影响世界”的感觉。
5. 山海经元素是否避免堆名词，是否能转化成可玩的风险、风俗和异象。

### 8.4 视觉审阅重点

建议重点看：

1. 横向长图是否和 UI 合在一起。
2. 地点图、事件图、路遇图是否会混淆。
3. 方图 / 横图在弹窗里的使用是否合理。
4. 低神志污染层是否有效，但不过度遮挡文字。
5. 车队层是否保留，或是否应彻底弱化为风景主导。

### 8.5 音频审阅重点

建议重点看：

1. 默认静音后首次操作启声是否自然。
2. 地形音乐轮换是否有区分。
3. 神志污染层是否有效。
4. 短音效是否有反馈但不烦。
5. 哪些音乐应保留、待改、弃用。
